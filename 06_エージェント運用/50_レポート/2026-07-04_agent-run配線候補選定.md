## 推薦する実装設計（次ブリーフ下書き）

### Goal

`corpus-collect` の本番挙動を変えず、送信しないdry-run入口でClaude通知レッグを `AGENT_VENDOR=claude agent-run` 経由にできることを確認する。

### 触るファイル

本番実装時に触る候補は1本だけ:

- `~/.claude/scripts/run_corpus_collect.sh`

作業ログと台帳更新:

- `01_プロジェクト/AI自動化/導入済み.md`
- `06_エージェント運用/00_司令塔/作業ログ_ツバキとあおい.md`

### 差分イメージ

現状:

```bash
claude -p "次の内容を一字一句そのままDiscord #youtube ... ${MSG}" \
  --permission-mode auto \
  --allowedTools "mcp__plugin_discord_discord__reply" >> "$LOG" 2>&1
```

推奨:

```bash
AGENT_RUN="/Users/kojinn/agent-adapters/bin/agent-run"
CORPUS_COLLECT_AGENT_VENDOR="${CORPUS_COLLECT_AGENT_VENDOR:-claude}"
CORPUS_COLLECT_DRY_RUN="${CORPUS_COLLECT_DRY_RUN:-0}"
CORPUS_COLLECT_NO_POST="${CORPUS_COLLECT_NO_POST:-0}"

if [ "$CORPUS_COLLECT_DRY_RUN" = "1" ] || [ "$CORPUS_COLLECT_NO_POST" = "1" ]; then
  echo "$MSG" >> "$LOG"
  echo "DRY_RUN_NO_POST: would post corpus summary"
else
  AGENT_VENDOR="$CORPUS_COLLECT_AGENT_VENDOR" "$AGENT_RUN" -p "次の内容を一字一句そのままDiscord #youtube ... ${MSG}" \
    --permission-mode auto \
    --allowedTools "mcp__plugin_discord_discord__reply" >> "$LOG" 2>&1
fi
```

ただし、上記は設計イメージ。実装時は引用・改行・exit code・ログ検証を丁寧に詰める。

### 検証方法

Phase 1（安全入口だけ）:

- `bash -n ~/.claude/scripts/run_corpus_collect.sh`
- `CORPUS_COLLECT_DRY_RUN=1 CORPUS_COLLECT_NO_POST=1 ...` の手動dry-run設計を作る
- dry-runではDiscord投稿なし
- dry-runでは本番キュー/コーパス/収集済みURLを書き換えないよう、必要なら `/tmp` 複製で検証

Phase 2（本番wrapper反映）:

- `.bak-20260704-agent-run-corpus` を作成
- wrapper 1本だけ差し替え
- `bash -n` OK
- no-post dry-run OK
- 本番launchdは未変更
- 本番定期実行はClaude固定のまま

Phase 3（本番実行の扱い）:

- 収集・Discord投稿を伴うため別GO
- いきなり日曜21:00の定期本番で判定しない
- `/tmp` でメッセージ生成まで確認してから、人間GOで実送信を1回だけ行うか判断

### 戻し方

- launchd plistは触らないので、戻しはwrapper復元だけ
