# 06_エージェント運用

Obsidian x Claude / Codex を24時間に近い形で動かすための管制塔。

ここは作業成果物の置き場ではなく、AIが「今の状態」「未処理」「確認待ち」「健康状態」「判断ログ」を見るための運用レイヤー。

## 使い方（2026-06-28 実態に整理）

当初は 10_Inbox → 20_実行キュー → 30_ヘルスチェック → 40_判断ログ → 50_レポート の6段パイプラインを想定していたが、
実運用では Discord受付 + 司令塔の少数ファイルに集約された。**新しいAIエージェントは、以下の現役の一次面だけ見ればよい。**

### 現役の一次面（ここを見る・ここに書く）

1. `00_司令塔/NOW.md` — 今の状態・最優先（最初に必ず読む）
2. `00_司令塔/タスクボード.md`＋`期日タスク.md` — タスク（操作は `~/.claude/scripts/task_cli.py` 経由）
3. `00_司令塔/作業ログ_ツバキとあおい.md` — 実行ログ・申し送り
4. `30_ヘルスチェック/ヘルスチェックログ.md` — 自動化ジョブ・同期・障害の稼働記録
5. `40_判断ログ/決定事項.md` — 重要な決定（保留は `保留判断.md`）

未処理・確認待ちの受付は Discord（#一般 / #メモ / #レポート）が入口で、そこからデイリーノート（05_日誌）や
タスクボードに流れる。自動化ジョブの台帳の正本は `01_プロジェクト/AI自動化/導入済み.md`。

### 凍結（参照のみ・書かない）

下記は初期構成の名残で、現在は使っていない（参照切れ防止のため移動・削除はせず注記のみ）：
`10_Inbox/未処理.md`・`20_実行キュー/完了.md`・`30_ヘルスチェック/障害メモ.md`・`自動化監査.md`・
`50_レポート/日次AIレポート.md`・`週次AIレポート.md`。各ファイル冒頭の【凍結】注記を参照。

## 保存しないもの

- トークン
- Cookie
- セッション
- 認証情報
- 生ログ
- キャッシュ
- 外部サービスの秘密情報

生データではなく、判断に使える要約だけを残す。
