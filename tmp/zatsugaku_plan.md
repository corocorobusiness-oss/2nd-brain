# 平安時代雑学 zatsugaku_plan

## 0. このファイルの役割

このファイルは、平安時代雑学回の設計図である。  
台本本文、レス順の完全指定、manifestの代替にはしない。

下流では、創作スレ生成時にこのファイルを読み、会話の自然さと雑学回の構成を両立させる。

参照元:

## 2. 全体の主軸

主軸:

> 平安時代、雅どころか想像以上にカオスだった

この回は、安倍晴明の続編ではない。  
ラストで陰陽師・怨霊・祈祷へ接続するが、全編を陰陽師回に寄せない。

## 3. 前半/後半の役割

### 前半

役割:

- 入りやすい雑学で引っ張る
- 笑える、変、意外、教科書イメージと違う話を優先する
- 政治史から入らない

前半で残したい印象:

> 平安人、雅というよりクセが強すぎる

前半レスは綺麗に締めなくてよい。  
雑に終わり、前置きパートで後半へ空気を変える。

### 後半

役割:

- 深い、怖い、重い話で回収する
- 前半で笑ったあとに「普通に生きるのキツい」と感じさせる
- 安倍晴明単体ではなく、平安社会の不安として見せる

後半で使う重さの候補:

- 都と治安
- 病気/死生観
- 祈祷依存
- 怨霊観
- 陰陽師で回収

注意:

- 台本上では分類名を出さない
- この順番は固定しない
- 「次は治安」「次は病気」のように見出し化しない
- 質問と反応で自然に重くする
- 後半のどこかで一度、貴族生活や日常ネタへ戻して蛇行させる
- 祈祷、怨霊、陰陽師は連続させすぎず、茶化しや日常ネタを挟む

## 4. ブロック設計

### 前半ブロック候補

- 候補A: 平安時代の入口
   - 雅なイメージに対して、実際はかなりクセが強そうという偏見から入る
   - 視聴者の「平安って優雅なだけじゃないの？」を崩す

- 候補B: 暮らしのズレ
   - 食事、住居、寒さ、服装、衛生感覚など
   - 現代基準のツッコミで入りやすくする
   - 断定しすぎず、階層差や時代差を残す

- 候補C: 恋愛と結婚
   - 通い婚、和歌、貴族社会の面倒さ
   - 優雅ではなく、手続きや気遣いが多い世界として見せる
   - 性別や婚姻を現代価値観で雑に裁かない

- 候補D: 貴族と庶民のギャップ
   - 宮廷文化と庶民生活を同じ平安として扱いすぎない
   - 「貴族でも大変、庶民はもっと別世界」という温度で前半を終える

前半の終点:

> 平安貴族、優雅というよりクセが強すぎる

### 前置きパート

役割:

- 前半の軽い雑学から、後半の少し重い雑学へ空気を変える
- 雑学専用の型は作らず、既存の前置き型を使う
- 新しい雑学や長い解説は入れない

前置きに渡す情報:

- ここまで: 平安時代の貴族文化や暮らしのズレ
- 次: 実際の平安京の暮らしやすさ、夜の都、病気、不安
- 疑問: 実際の平安京や当時の暮らしは、どれくらい不安が多かったのか

注意:

- 前置きは賢くまとめすぎない
- 「華やかな文化の一方で」のような説明臭い対比を連発しない
- 既存の前置き型を守り、後半への空気転換だけにする

### 後半ブロック候補

- 候補E: 都と治安
   - 夜の都、都の端、検非違使、右京の開発されにくさなど
   - まず生活レベルの怖さに落とす
   - 右京は「荒廃」より「開発が伸びにくく、左京へ重心が寄った」程度に弱める

- 候補F: 病気と死生観
   - 現代医療とは違う時代の不安、疫病、出産、死との近さ
   - 「病気になったら祈るしかない」で単純化しすぎない
   - 医療、薬、祈祷が併存していた温度で扱う

- 候補G: 祈祷と怨霊
   - 災害や病を、怨霊や物の怪と結びつけて解釈することもあった感覚
   - すべてを怨霊のせいにしない
   - 御霊信仰は後半の解説候補

- 候補H: 陰陽師で回収
   - 安倍晴明本人の続編にしない
   - 陰陽師はファンタジー職ではなく、当時の不安を処理する役割として見せる
   - 陰陽寮、暦、占い、方角、儀礼などの背景へ広げる
   - 最後は「雅と不気味さが同居する平安」として余韻を残す

後半生成時の崩し:

- 候補E-Hをこの順番通りに並べなくてよい
- 重い話の直後に短い茶々を挟む
- 後半の途中で一度、貴族生活や日常ネタへ戻す
- 陰陽師の名前は終盤まで出しすぎない

### 5-2. 限定表現の目安

以下は台詞例ではなく、断言の強さの目安。  
そのままレス本文へ流用せず、創作スレ生成側で自然な会話に崩す。

- 通い婚/和歌: 平安貴族の恋愛は、和歌や男性が女性側へ通う婚姻形態が関わり、現代感覚ではかなり面倒に見える
- 貴族/庶民: 貴族文化として見える平安と、庶民の暮らしは分けて考える
- 検非違使: 京の警備や取り締まりに関わった役職として扱う
- 右京: 地形条件などから開発が伸びにくく、次第に左京へ重心が寄った程度に留める
- 病気/祈祷: 現代医療とは違い、医療・薬・祈祷が併存していた温度にする
- 御霊信仰: 疫病や災いを怨霊と結びつけて解釈することもあった、という表現に留める
- 陰陽師: 暦や占い、吉凶判断に関わる官人として扱い、妖怪退治職にしない

後半候補:

- 御霊信仰

- 検非違使

優先案:

- 前半: 通い婚

安倍晴明本人の人物解説には寄せない。

理由:

- `通い婚` は前半の恋愛/貴族文化のズレを理解させやすい
- `陰陽道` は後半の祈祷・怨霊・陰陽師を、安倍晴明本人ではなく時代背景として受け止めさせやすい
- `御霊信仰` はスレ本文の山場で触れ、必要なら補助候補として残す

## 6.5. 雑学モード尺ゲート

この案件は大カテゴリ雑学回なので、通常の単体人物/事件型よりスレ本文の量を厚くする。

ハードゲート:

- スレ①: 最低2,500字
- スレ②: 最低2,500字
- スレ本文合計: 最低5,000字

対象はスレ本文のみ。  
冒頭、前置き、解説、締めでの水増しは不可。

`creative-thread-gen/qa_check.py` の `本文字数: 2500 PASS` は、単体スレの最低QAであり、雑学回の完成尺ではない。  
雑学回では、スレ①とスレ②の両方がそれぞれ2,500字を超えるまで、manifest化・組み立てへ進まない。

manifest には雑学専用として以下を入れる:

```json
{
  "content_mode": "zatsugaku",
  "length_policy": {
    "profile": "zatsugaku",
    "min_thread1_chars": 2500,
    "min_thread2_chars": 2500,
    "min_total_thread_chars": 5000
  }
}
```

## 7. AI臭防止ルール

基本骨格は裏に持つ。

```text
偏見レス
茶々・ツッコミ
知識レスで反転
リアクション
次の雑学
```

ただし、綺麗に完走させるのは半分から6割までを目安にする。  
数字を機械的に守るより、読んだ時に型が見えないことを優先する。
連続2回以上、綺麗に完走させない。

入れる崩し:

- いきなり知識レスから入る
- ツッコミだけで終わる小ネタ
- 勘違いレス
- 未回収の疑問
- 日常ネタの脱線
- 短い `>>` 返信
- 話題の急カーブ
- 「諸説」「地域差」「時代差」「全部がそうではない」
- 後半途中で日常ネタへ戻る蛇行
- 重い話の直後の短い茶々

禁止:

- 長文3連続
- 同じ語尾3連続
- 役割固定
- 毎回の正解発表
- 3ネタ連続の同じ話運び
- レスが便利に説明を進めすぎること
- 後半を候補E-Hの順番通りにきれいに並べること

最低限残すもの:

- 冒頭付近の軸
- 話題切替前の短い整理
- 後半入りと終盤の空気変化

## 8. 表現リスク

注意する話題:

- 身分
- 性別
- 病気
- 死
- 貧困
- 衛生

運用:

- 当時の制度や価値観として扱う
- 現代価値観で「バカ」「終わってる」だけにしない
- 驚きやツッコミはOK、見下しはNG
- 蔑称や病気いじりを伏字で残さない
- 断言できないものは「とされる」「伝わる」「一説には」に落とす

## 9. 軽ゲート

`zatsugaku_plan.md` 後の軽ゲート結果:

| 項目 | 判定 | メモ |
|---|---|---|
| 雑学回として入口が広いか | PASS | 平安時代全体を扱うため入口は広い |
| 前半/後半の役割が分かれているか | PASS | 前半は暮らし/恋愛、後半は治安/病気/祈祷/怨霊 |
| 雑学モード尺ゲートが定義されているか | PASS | スレ①2,500字以上、スレ②2,500字以上、スレ合計5,000字以上をハードゲート化 |
| 通常の2ch歴史トーンからズレてないか | PASS | ナレーションは既存スキル型を使用 |
| AIっぽい固定ループになってないか | MINOR | 生成時に崩しルールを強制する |
| 史実・規約で危なそうな話題がないか | PASS/MINOR | 断言レベルは整理済み。生成時に強い言い切りへ戻さない |

軽ゲート結論:

> MINOR。構成は通してよい。ファクト候補の断言レベルは整理済み。創作スレ生成では、強い断言・現代価値観の見下し・晴明万能化を避ける。
