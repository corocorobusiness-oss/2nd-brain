# スレッド評価ルール

検索で見つけたスレッドの中身を確認し、台本に使えるかどうかを判断するためのルール。

---

## 1. Chromeでの効率的な中身確認手順

### まとめサイトの場合
```
1. navigate でページにアクセス
2. get_page_text または javascript_tool で本文を取得
3. 以下を確認：
   - コメント数（多いほど良い。目安は20コメント以上）
   - 議論の方向性（テーマに沿っているか）
   - 転載元URL（重複チェック用）
```

### 元サイト（5ch/open2ch）の場合
```
1. navigate でスレッドにアクセス
2. get_page_text で全コメントを取得
3. コメントの質と量を確認
```

---

## 2. スレッドの評価基準

### A評価（メインで使える）
- コメント数が30以上
- テーマに沿った議論が活発
- 疑問文のコメントが複数ある（視聴者のコメント誘導に使える）
- 返信の連鎖がある（議論が深まっている証拠）
- 知識系コメント（解説の種になる）が含まれている

### B評価（補助で使える）
- コメント数が15〜30
- テーマに部分的に沿っている
- 面白いコメントが数個ある
- 他のスレッドと組み合わせれば使える

### C評価（使いにくい）
- コメント数が15未満
- タイトルと中身にズレがある（タイトル詐欺）
- 議論が脱線している
- 差別的コメントが大半を占める
- コメントが短すぎる（「草」「それな」ばかり）

---

## 3. 台本の構成に影響する要素の確認

各スレッドの中身を確認する際、以下の要素を把握しておく。
これが構成提案（前半/後半の振り分け）の判断材料になる。

### 確認項目

| 項目 | 確認すること | 構成への影響 |
|---|---|---|
| 話題の方向性 | 笑い寄りか、考察寄りか | 笑い→前半、考察→後半が基本 |
| 感情の温度 | 盛り上がっているか、冷静か | 盛り上がり→前半、冷静→後半 |
| 結論の有無 | 議論に結論が出ているか | 結論あり→前半（if仮想型の伏線に） |
| 人物論の有無 | 特定人物の深掘りがあるか | あり→後半（人物深掘り型に） |
| 逆視点の有無 | 反対意見や裏テーマがあるか | あり→後半（逆視点型に） |

## 4. 候補提案時のフォーマット

ユーザーに提案する際は、以下のフォーマットで提示する。

```
【提案スレッド】
| # | まとめサイト | タイトル | URL | 元スレ | 評価 | 特徴 |
|---|---|---|---|---|---|---|
| ① | ガバガバ歴史速報 | [タイトル] | [URL] | [元スレID] | A | 議論活発、コメント50+ |
| ② | なんJ歴史部 | [タイトル] | [URL] | [元スレID] | B | 面白いツッコミ多数 |
...

おすすめは①②④です。理由：[簡潔に]

推奨構成パターン：[パターン名]
  前半：①② → [方向性の説明]
  前置き：[転換のポイント]
  後半：④⑤ → [方向性の説明]
```

---

## 5. 差別コメントの事前チェック

スレッドの中身を確認する段階で、以下に該当するコメントが多いスレッドは警告する。

- 特定民族・国籍への蔑称を含むコメント
- 障害者を揶揄する表現を含むコメント
- 性差別・性的指向への蔑称を含むコメント

**理由：** このチャンネルは過去2回のガイドライン違反を受けており、
3回目でチャンネル停止のリスクがある。
差別コメントが大半を占めるスレッドは、使えるコメントが少なくなるため
候補から外すことを推奨する。
