# YouTube台本パーツ生成スキル

2ch掲示板風×歴史YouTubeチャンネルの台本における「冒頭パート」「解説パート」「前置きパート」「締めパート」の4種類を、テーマに合わせてゼロから生成するスキルです。

## 対応パート一覧

| パート | 役割 | 文字数目安 | リファレンス |
|---|---|---|---|
| 冒頭パート | 動画冒頭で視聴者の興味を引く | 200〜350文字 | `references/bouto-guide.md` |
| 解説パート | 用語・人物・事柄の教育的解説 | 100〜250文字 | `references/kaisetsu-guide.md` |
| 前置きパート | スレッド切替時の導入・橋渡し | 100〜200文字 | `references/maeoki-guide.md` |
| 締めパート | 動画の締めくくり・行動促進 | 最大280文字 | `references/shime-guide.md` |

## 手順

### ステップ1：パートの特定

ユーザーのリクエストから、どのパートを生成するか特定する。
明示されていない場合は「どのパートを作成しますか？（冒頭・解説・前置き・締め）」と確認する。
複数パートの同時生成も可能。

### ステップ2：リファレンスの読み込み

該当パートのリファレンスファイルを `Read` ツールで読み込む。

- 冒頭パート → `references/bouto-guide.md` を読む
- 解説パート → `references/kaisetsu-guide.md` を読む
- 前置きパート → `references/maeoki-guide.md` を読む
- 締めパート → `references/shime-guide.md` を読む

**CRITICAL**: リファレンスには作成ルール・構成パターン・ゴールデンサンプルがすべて含まれている。必ず読み込んでから生成すること。

### ステップ3：テーマの確認

生成に必要な情報をユーザーから確認する。

- **冒頭パート**: テーマ（武将名・事象など）
- **解説パート**: 解説対象（人物・用語・戦い・地名など）
- **前置きパート**: これまでの議論内容と、次に展開するテーマ
- **締めパート**: 動画全体のテーマと議論の要約

### ステップ4：生成と出力

リファレンスに記載されたルール・構成パターン・フォーマットに**厳密に従って**本文を生成する。

**共通の絶対ルール**:
- 余計な前置きや後書き（「以下のように作成しました」等）は一切出力せず、本文**のみ**を出力すること
- 各パートのリファレンスに記載されたゴールデンサンプルと完全に同じトーン・リズム・文末表現で生成すること
- 1行が長くなりすぎないよう、テンポ良く意味の切れ目で改行を入れること

## トラブルシューティング

- **トーンが合わない場合**: リファレンス内のゴールデンサンプルを再読し、文末表現やリズムを細かく模倣する
- **文字数オーバーの場合**: 各パートの文字数制限を厳守。超過した場合は情報を絞って再生成する
- **どのパートかわからない場合**: ユーザーに確認を取る。「台本のどの部分？」と聞くだけでOK
