**やること：**

4. **勝ちフォーマット観察**: 伸びてる競合のタイトル・尺をJ列メモに記録

### 手順（各ネタ候補につき実行）

1. **WebSearchで実URL検索**（最低3クエリ）
   ```
   {ネタの主要キーワード} 2ch まとめ
   {ネタの関連人物/事件} なんJ 歴史
   {ネタのキーワード} site:gabareki.blog.jp OR site:nanjhistory.2chblog.jp

2. **検索結果から実在URLを抽出**
   - まとめ記事タイトルとネタの関連性を確認
   - 関連性が低いもの（キーワードが偶然一致しただけ）は除外
3. **元スレ番号で重複排除**
   - 同じ元スレの転載は1本としてカウント
4. **収集結果で判定**

### 判定基準（実URL数ベース）
| 記号 | 実URL数 | 意味 | アクション |
|------|---------|------|-----------|
| ◎ | 6本以上（異なる元スレ） | 鉄板、即台本化可能 | 予測×1.1 |
| ○ | 4〜5本 | 安定、通常制作 | そのまま |
| △ | 3本 | 工夫すれば行ける | 予測×0.8 |
| ✕ | **2本以下** | **台本化不可** | **スプシに書き込まない** |

### 除外ルール（厳格化）
- **実URL 3本未満 → ✕ 判定 → スプシ書き込み禁止**（例外なし）
- URL単位でネタの中身を確認し、タイトルだけのヒットは除外する
- 関連性が薄いスレ（例：「信長」で検索してヒットしたが信長と無関係）は除外する

### 収集したURLの記録
集めたURLはスプシの **Q列（候補スレURL）** に改行区切りで記録する。最大5本まで記録。
P列（スレ可用性コメント）には元スレの簡潔なメモ（「信長公記系のスレ多数」等）を書く。

### thread-research-planner への引き継ぎ
台本化決定時、thread-research-planner はまずスプシのQ列に記録された候補URLを起点に評価を開始する。
追加で検索する場合もあるが、**「ゼロから探して見つからなかった」問題はここで予防する**。

```

### 特徴量の内訳
`feature_db.json` に以下の集計が入っている：
- **by_era**: 時代別平均再生数（縄文=79k鉄板、平安=122k、戦国=9k飽和）
- **by_category**: カテゴリ別（闇・裏話=51k、文化制度=34k、戦争合戦=33k）
- **by_title_pattern**: タイトル型別（「〜の真実/闇」=204k、「雑学シリーズ」=40k）
- **by_era_category**: 時代×カテゴリクロス
- **by_hype_word**: 煽りワード（「本当」「実は」「絶」が高評価）
- **by_subject**: 人物・事件別（2026-06-04追加。最強の単独予測因子。縄文人=79k、秀吉=11k/75%、関ヶ原ヒット実績）

### 2026-06-04 精度改良（バックテスト 22%→60%+）
5月公開10本で検証した結果を反映：
- **人物コンポーネント**（weight 0.40）: by_subjectにデータがある人物は実績平均を強く反映
- **知名度フォールバック**: データ無し人物は FAME_S（信長/秀吉/卑弥呼等）×1.25、マイナー×0.75
- **死筋ペナルティ**: 一度も1万超えのないカテゴリ/型（打線・ランキング・if系）に該当 → 予測×0.45、確率上限15%
- **教訓**: 視聴維持率は全動画41〜52%でほぼ一定。当たり外れは「ネタ（人物の知名度×切り口）」でほぼ決まる

### 予測式
```
予測再生数 = 重み付き平均(era×cat 45%, era 30%, cat 20%, pattern 25%, hype 10%)
           × 大河連動ブースター(×1.6)
           × トレンドブースター(×1.3)
           × 季節ブースター(×1.2)
1万超え確率 = 特徴量別の実績1万超え率を平均 + 予測絶対値による補正
自動ランク = 予測≥50k or (20k+50%以上)→S / 15k+ or 50%+→A / 7k+ or 30%+→B / その他C
```

### 自動除外（REJECT）
タイトルに以下NGキーワードが含まれるネタは **自動で除外**:
- 近代戦争: 太平洋戦争、日露、日清、ノルマンディー、硫黄島、特攻、栗林、東郷、乃木、203高地など
- 世界史: 三国志、ローマ、エジプト、モンゴル、シュメール、ナポレオンなど

### 優先順位の決め方
1. **予測再生数 ≥ 30,000 & スレ可用性 ◎ or ○** → 最優先で制作
2. **大河ドラマ連動 & 予測 ≥ 15,000** → 放送1〜3日前に投稿タイミング合わせて優先
3. 予測再生数 < 10,000 のネタは **スプシに書き込まない**（絶対伸びる基準）
4. スレ可用性 ✕ のネタは **スプシに書き込まない**（台本化できない）
5. 時代・カテゴリの偏りを避ける（過去109本分析でif系=33本飽和、縄文=4本全部伸び）

### モデル再学習
チャンネルの再生数が変化したら `scripts/fetch_channel_stats.py`（YouTube APIで全動画取得）→ `build_feature_db.py` を再実行。
**週次自動更新は稼働中**: launchd `com.claude.neta-retrain`（毎週月曜9:10、ログ: `~/.codex/scripts/neta-retrain.log`）

### 予測vs実績の自動答え合わせ（2026-06-04稼働）
`scripts/verify_predictions.py` が週次再学習の直後に自動実行される：
- スプシの予測つきネタを公開動画とタイトル照合（類似度0.45閾値、語尾改変も追跡可）
- 公開7日以上経過した動画の実績を **R列（実績再生数）/S列（計測日）/T列（的中判定）** に記録
- 累計的中率を `data/prediction_log.json` に蓄積 → モデル精度の推移を追える
- リサーチ時はこのログを見て「最近の的中率」をユーザー報告に含めること

### ネタ名の命名ガイド

ユーザー（池田祐馬）のチャンネル「2chで世界史学ぶ民」の実際のタイトルから抽出した命名パターン。
スプシの **ネタ名** 列は、動画タイトルの「本体部分」（`【2ch歴史】` プレフィックスと `2ちゃんねるの歴史オタクたちの見解がおもしろい【ゆっくり解説】` サフィックスを除いたコア）をそのまま入れる。

**よく使う型：**

| 型 | 例 |
|----|-----|
| `〜www` / `〜wwww`（煽り・草生やし） | 毛利元就が天下を狙わなかった理由www |
| `〇〇とかいう△△www` | 神武天皇とかいう初代天皇wwwwww／戦国時代の天皇とかいうガチで極貧生活だった人 |
| `〇〇って△△なんじゃね？` / `〜って結局△△？` | 家康が最も恐れた男って結局誰なんや？／上杉謙信って実は過大評価なんじゃね？ |
| `〜した理由www` | 明智光秀が本能寺の変を起こした理由www |
| `〜が〇〇し過ぎたwww` / `〜がヤバすぎる` | 浅井長政の裏切りがヤバすぎた件www／江戸時代の雑学がカオスすぎて草 |
| `〇〇←こいつが△△した理由` | 豊臣秀吉←こいつが天下取れた理由www |
| `〇〇の奇行・逸話打線www` | 伊達政宗の奇行・逸話打線www（なんJ打線系） |
| `もし〜` | もし今も徳川幕府が続いていたら…… |
| `〜の真実` / `〜の闇` | 源頼朝の闇が深すぎるwwww |
| `歴史オタクの雑学で〇〇が面白くなり過ぎたwww` | 定番シリーズ形式 |

**ルール：**
- 語尾に `www`・`wwww`・`草` などを付けて2ch／なんJっぽい煽り口調にする
- 「ガチで」「ヤバすぎる」「〇〇過ぎ」など誇張表現OK
- 疑問形（`〜なんや？` `〜じゃね？`）も定番
- 堅い書き方（「〜について考察する」など）は避ける
- 1タイトル30文字前後が目安（プレフィックス・サフィックス付きで全体70文字程度に収まるように）
