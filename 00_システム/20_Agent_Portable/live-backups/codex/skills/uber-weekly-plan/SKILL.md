# Uber週次プランナー

来週（月〜日）の最適な稼働プランを、クエスト型＋天気予報＋実績データから生成して #お金 に出す。
スクショ不要（クエスト型は保存済み、天気は自動取得、実績はデータ化済み）。

## データ
`data/patterns.md` を読む（クエストの型・天気影響・曜日/時間の実績・金額配分）。

## 手順

### 1. 来週の日付範囲を出す
今日から見て次の月曜〜日曜を対象にする。

### 2. 天気予報を取得（自動）
Bashで Open-Meteo の予報を取得（長岡 緯度37.45 経度138.85）：
```bash
curl -s "https://api.open-meteo.com/v1/forecast?latitude=37.45&longitude=138.85&daily=precipitation_sum,weathercode&timezone=Asia%2FTokyo&start_date=<来週月>&end_date=<来週日>"
```
降水3mm以上の日＝「雨＝稼ぎ時」とマーク。

### 3. クエスト型の鮮度チェック（自己修正）
直近週の完了クエストを drivers.uber.com の getWebActivityFeed（履歴）で確認できる。
もし保存済みの型と大きく違っていたら、プランにその旨を注記し「クエスト変わったみたいなので最新スクショ送って」と添える。
（ここは任意。簡易運用なら保存済み型をそのまま使う）

### 4. 最適化ロジック
patterns.md を踏まえて来週の各日を設計：
- **月〜木**: 大型40配達クエスト(¥5,790)を取りに行く＝4日で平均10配達/日
- **時間枠クエストの窓を必ず入れる**: 火＝夜17-22 / 水＝昼10:30-14:30 / 木＝通し
- **雨予報の日を最優先で厚く**（1.6倍）。曜日より天気を上に置く
- 金〜日は10件クエストを軽く回収
- 金額は基礎配分（木¥10,000>水¥9,000>月¥5,000>金¥4,500>火¥4,000）を土台に、雨予報で上振れ調整
- イベント休み（誕生日・遠足等）は目標と計画.md の備考を尊重

### 5. Discord #お金 に投稿
`mcp__plugin_discord_discord__reply` chat_id=1512911463180800120 に、この形で：
```
📅 来週(M/D〜M/D)の最適稼働プラン

天気: 月☀ 火☔ 水☀ 木☔ 金☀ …（雨マーク）

| 日 | 天気 | 狙う時間 | 配達目安 | 目標額 | ポイント |
| 月 | ☀ | 昼 | 8 | ¥5,000 | 大型クエスト開始 |
| 火 | ☔ | 夜17-22 | 9 | ¥4,000+雨 | 夜クエスト¥1,200 |
| 水 | ☀ | 昼10:30-14:30 | 6+ | ¥9,000 | 昼クエスト¥700 |
| 木 | ☀ | 朝〜夜通し | 12 | ¥10,000 | 最強日＋40達成 |
| 金土日 | - | 軽め | 10 | ¥4,500等 | 小クエスト回収 |

💰 目標額は基礎配分（木¥10,000>水¥9,000>月¥5,000>金¥4,500>火¥4,000）。
　 雨予報の日は+1.6倍見込みで上振れ目標を添える。
🎯 今週の狙い: 大型40配達(¥5,790)クリア＋雨の◯曜を厚く
📊 週合計目標: ¥◯◯（うちクエスト¥5,790込み）／ 想定上振れ +◯◯円（雨）
```

## 安全運用
- サイト/予報の取得は週1回のみ（低頻度＝規約セーフ）
- 不可逆操作なし（提案を出すだけ）
