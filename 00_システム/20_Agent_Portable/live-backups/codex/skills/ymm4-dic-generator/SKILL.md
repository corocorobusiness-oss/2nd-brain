# YMM4 読み仮名辞書ジェネレーター

歴史系YouTube台本のテキストから、TTS（音声合成）が正しく読めない可能性のある
漢字・用語の読み仮名を抽出し、YMM4の単語辞書(.dic)形式で出力するスキル。

---

## 基本方針

- **毎回書き換え方式**：台本が送られるたびに、その台本の用語だけでdicファイルを新規作成する
- 前の台本のエントリは残さない（上書き）
- 出力先：ユーザーのワークスペースフォルダ内の `ymm4_user.dic`

### 完全網羅の原則（漏れゼロ）※最重要

このスキルの目的は「有名な人名・地名を拾うこと」ではなく、
**TTS（ゆっくり音声）が誤読する可能性のある語を1つ残らず登録すること**。

「名前っぽい／地名っぽい／用語っぽい」という型に当てはまらない語こそ漏れやすい。
過去に漏れた実例とその教訓：

| 漏れた語 | 正しい読み | 漏れた理由 | 教訓 |
|---|---|---|---|
| 高望王 | たかもちおう | 「王号」でフルネーム人名の型に当てはまらない | 王・親王・天皇・院など**皇族の称号付き名**も人名として拾う（カテゴリA） |
| 良馬 | りょうば | 一般熟語に見えて名前候補から外れた | 文脈で読みが割れる**熟語**も拾う（カテゴリE） |
| 2回 | にかい | 数字＋助数詞でどのカテゴリにも無かった | **数詞＋助数詞**を独立カテゴリで拾う（カテゴリF） |
| 落とし所 | おとしどころ | 送り仮名つき訓読み熟語で見落とし | **訓読み複合語・送り仮名語**を拾う（カテゴリE） |

→ カテゴリ抽出だけに頼らず、**ステップ2.5の全文スイープ**で台本を頭から末尾まで走査し、
漢字を含む語で「ゆっくりが誤読しうるもの」を機械的に拾い切ること。
迷ったら「登録しない」ではなく「登録する」側に倒す（過剰登録は害が小さい）。

---

## 手順

### ステップ1：台本の受け取りと確認

1. ユーザーから台本テキストを受け取る
2. ユーザーから追加の指示があれば確認する（例：名前の分割指定、特殊な読みの指定）

### ステップ2：用語の抽出と分類

台本テキストから以下のカテゴリの用語を網羅的に抽出する：

**カテゴリA：人名（最優先）**
- フルネーム（例：伊達政宗→だてまさむね）
- 名前のみ（例：政宗→まさむね）
- フルネーム（苗字＋名前）しかない場合は、名前単体も必ず別エントリで登録する
  （例：上杉謙信 がある場合 → 謙信 も登録）
- ユーザーが分割指定した場合、フルネームと名前の両方を登録
- 氏族名（例：伊達→だて、伊達家→だてけ、伊達軍→だてぐん）
- 通称・あだ名
- **皇族・王号・称号付きの名（漏れやすい！）**
  - 〇〇王（例：高望王→たかもちおう、護良親王→もりよししんのう）
  - 〇〇天皇・〇〇上皇・〇〇院（例：後醍醐天皇→ごだいごてんのう、白河院→しらかわいん）
  - 「王」「親王」「天皇」「院」で終わる固有名は、人名として必ず1エントリで登録する

CRITICAL: 名前単体エントリの自動追加時、以下の誤検知に注意すること。
「苗字＋名前」のパターンと「固有名詞＋一般語」を混同しないこと。

**名前単体として登録すべきもの（人名の下の名前）：**
- 上杉謙信 → 「謙信」を追加 ✓
- 毛利元就 → 「元就」を追加 ✓
- 明智光秀 → 「光秀」を追加 ✓

**名前単体として登録してはいけないもの（一般語・誤分割）：**
- 小田城 → 「城」は一般語であり名前ではない ✗
- 武田軍 → 「軍」は一般語であり名前ではない ✗
- 藤原家 → 「家」は一般語であり名前ではない ✗
- 足利将軍 → 「将軍」は役職名であり名前ではない ✗
- 伊達領 → 「領」は一般語であり名前ではない ✗
- 伊達姓 → 「姓」は一般語であり名前ではない ✗
- 小田原討伐 → 「小田原」を「小田＋原」に分割するのは誤り ✗
- 龍造寺四天王 → 「四天王」は称号であり名前ではない ✗
- 蜂須賀党 → 「党」は一般語であり名前ではない ✗
- 徳川宗家 → 「宗家」は一般語であり名前ではない ✗

**判定基準：**
分割後の「名前」部分が以下に該当する場合は登録しない：
1. 一文字の一般語（城、軍、家、領、姓、党、氏 など）
2. 一般的な役職名・称号（将軍、守、太守、四天王 など）
3. 一般的な名詞（宗家、重臣、討伐 など）
4. 分割自体が不自然（小田原→小田＋原 のように地名を壊す）

**カテゴリB：地名・城名・建造物**
- 城名（例：米沢城→よねざわじょう）
- 地名（例：仙台平野→せんだいへいや）
- 河川名（例：北上川→きたかみがわ）

**カテゴリC：歴史用語・専門用語**
- 役職名（例：藩主→はんしゅ、陪臣→ばいしん）
- 戦・事件名（例：関ヶ原→せきがはら、戊辰戦争→ぼしんせんそう）
- 制度・概念（例：石高→こくだか、慶長遣欧使節→けいちょうけんおうしせつ）

CRITICAL: 助詞「の」や複数語を含む表現でも、必ず1つのエントリとして登録すること。
TTSが個別の単語に分解して誤読するのを防ぐため、フレーズ全体を1エントリにする。

例：
- 三方ヶ原の戦い → みかたがはらのたたかい（1エントリ）
- 海道一の弓取り → かいどういちのゆみとり（1エントリ）
- 建武の新政 → けんむのしんせい（1エントリ）
- 錦の御旗 → にしきのみはた（1エントリ）
- 慶長遣欧使節団 → けいちょうけんおうしせつだん（1エントリ）

バラバラに登録すると「三方ヶ原」「の」「戦い」が別々に読まれてしまい、
全体としての正しい読みが保証されない。

**カテゴリD：打線表記（台本に含まれる場合）**
打線が含まれている場合、以下の固定ルールで読みを付ける。
打線エントリは半角数字＋全角括弧を含むため、カラム2を `1`（正規表現有効）にすること。

| 表記 | 読み |
|---|---|
| 1（遊） | 1ばんしょーと |
| 2（二） | 2ばんせかんど |
| 3（三） | 3ばんさーど |
| 4（捕） | 4ばんきゃっちゃー |
| 5（中） | 5ばんせんたー |
| 6（左） | 6ばんれふと |
| 7（右） | 7ばんらいと |
| 8（一） | 8ばんふぁーすと |
| 9（投） | 9ばんぴっちゃー |
| （DH） | でぃーえいち |
| （指） | でぃーえいち |

※ 打順とポジションの組み合わせは台本ごとに異なる。上の表は一例。
台本に書かれている実際の組み合わせに従って読みを付けること。
例：台本に「3（投）」とあれば → `3ばんぴっちゃー`

**カテゴリE：その他TTSが誤読しやすい語**
- 難読漢字（例：瓢箪→ひょうたん、甲冑→かっちゅう）
- 文脈依存の読み（例：最上→もがみ（人名時）、大御所→おおごしょ）
- 作品名・書名（例：独眼竜政宗→どくがんりゅうまさむね）
- 2ch → にちゃんねる など
- **文脈で読みが割れる熟語（漏れやすい！）**
  - 良馬→りょうば、上手→じょうず/うわて、大勢→おおぜい/たいせい
  - 一般語に見えても、台本の文脈で誤読されうるなら登録する
- **訓読み複合語・送り仮名つきの語（漏れやすい！）**
  - 落とし所→おとしどころ、立ち位置→たちいち、後ろ盾→うしろだて
  - 駆け引き→かけひき、見せ場→みせば、捨て駒→すてごま
  - 「所」「場」「位置」などを含む複合語はTTSが音読み（しょ・じょう 等）で誤読しやすい

**カテゴリF：数詞＋助数詞（漏れやすい！）**
TTSが数字＋カウンターを誤読・不自然読みするものを登録する。
- 2回→にかい、3回→さんかい、1回→いっかい
- 一騎→いっき、二手→ふたて、三度→さんど、数十年→すうじゅうねん
- 半角／全角どちらの数字表記も台本に出た形で登録する
- ※打線表記（カテゴリD）とは別。打線以外の通常の数え語はカラム2=`0`でよい

### ステップ2.5：全文スイープ（漏れゼロ確認）※必須

カテゴリ抽出（ステップ2）で拾った語は「型に当てはまったもの」だけ。
ここで台本を**頭から末尾まで1行ずつ走査**し、抽出リストに無い語で
ゆっくりが誤読しうるものを機械的に追加する。

走査時に必ずチェックする観点：
1. **漢字を含む語を全部見る** — 有名/無名で判断しない。良馬・高望王のような語を逃さない
2. **王・親王・天皇・院・公・卿で終わる語** → 皇族・称号付き人名（カテゴリA）
3. **数字＋助数詞（〇回・〇騎・〇手・〇度 など）** → カテゴリF
4. **送り仮名つき・「所/場/位置/盾/際」等を含む複合語** → カテゴリE（落とし所など）
5. **音読み/訓読みが割れる二字熟語** → 文脈の読みでカテゴリE
6. **既に抽出済みの人名の「下の名前」単体**が漏れていないか（カテゴリAのルール）

判断に迷ったら登録する。スイープ後、「この台本で誤読されうる語は全部入った」と
言い切れる状態にしてから次へ進む。

### ステップ3：文脈に応じた読みの判定

CRITICAL: 以下の点に特に注意すること。歴史系台本では文脈依存の読みが非常に多い。

1. **人名の特殊読み**を必ずWeb検索で確認する
   - 最上義光 → もがみ**よしあき**（よしみつではない）
   - 愛姫 → **めご**ひめ（あいひめではない）
   - 支倉常長 → **はせくら**つねなが
   - このような特殊読みは必ずWeb検索で裏取りすること

2. **同じ漢字でも文脈で読みが変わるもの**
   - 最上 → もがみ（人名・氏族名）vs さいじょう（一般語）
   - 上杉 → うえすぎ（人名）
   - 石高 → こくだか（歴史用語）vs いしだか（一般語）

3. **確信が持てない読みは必ずWeb検索で確認する**
   - 特に人名の読みは間違いが許されない
   - Agent ツールを使って複数の名前を一度に検証すると効率的

### ステップ4：dic形式での出力

**フォーマット：** タブ区切り、6カラム

通常エントリ：
```
1	0	単語	読み		1
```

打線エントリ（正規表現有効）：
```
1	1	2（二）	2ばんせかんど		1
```

各カラムの意味：
- カラム1: 有効フラグ（常に `1`）
- カラム2: 正規表現フラグ（通常 `0`、打線エントリは `1`）
- カラム3: 単語（表層形 / 変換前）
- カラム4: 読み（ひらがな / 変換後）
- カラム5: 空（タブのみ）
- カラム6: フラグ（`1` で統一）

CRITICAL:
- エンコーディング: UTF-8、改行コード: LF
- 打線エントリ（半角数字＋全角括弧を含むもの）はカラム2を必ず `1` にすること。
  `0` のままだとYMM4が正規表現マッチを行わず、辞書エントリが適用されない。
- 全エントリのカラム6は `1` で統一する。

### ステップ5：ファイルの書き換え

1. ワークスペースフォルダの `ymm4_user.dic` を新しい内容で**完全に上書き**する
2. 前の台本のエントリは残さない
3. 書き換えたエントリ数をユーザーに報告する

### ステップ6：検証

1. 出力したdicファイルの行数を確認する
2. フォーマットが正しいか（タブ区切り、カラム数）をサンプルチェックする
3. 打線エントリのカラム2が `1` になっているか確認する
4. 特に注意した文脈依存の読みをユーザーに報告する
5. **漏れゼロの最終確認**：台本をもう一度読み返し、漢字を含む語で
   dicに未登録かつ誤読されうるものが残っていないか走査する。
   特に 王号・数詞＋助数詞・送り仮名複合語・割れる熟語（高望王/2回/落とし所/良馬 系）を重点チェック。
   1つでも見つかったら追加して上書きし直す

## 例

### 例1：伊達政宗の台本

ユーザーが言う：「この台本の読み仮名をdicにしてください。名前は伊達政宗と政宗の2つに分けてお願いします。」

アクション：
1. 台本テキストから人名・地名・歴史用語・打線表記を抽出
2. 「伊達政宗」と「政宗」を別エントリで登録
3. 最上義光→もがみよしあき、愛姫→めごひめ等の特殊読みをWeb検索で確認
4. ymm4_user.dic をこの台本の用語だけで書き換え

結果：ymm4_user.dic を112エントリで新規作成

### 例2：次の台本（織田信長）

ユーザーが言う：「信長の台本です、dic作って」

アクション：
1. 台本テキストから人名・地名・歴史用語を抽出
2. 前回の伊達政宗エントリは全て削除し、信長台本の用語だけで書き換え
3. ymm4_user.dic を新しい内容で上書き

結果：ymm4_user.dic を95エントリで書き換え

## トラブルシューティング

### 読みの正確性に確信が持てない
原因：珍しい人名や古語
解決策：必ずWeb検索で確認する。確認できなかった場合はユーザーに質問する。

### YMM4で打線エントリが適用されない
原因：カラム2が `0` になっている（正規表現が無効）
解決策：打線エントリ（半角数字＋全角括弧を含む表記）はカラム2を `1` にする。
