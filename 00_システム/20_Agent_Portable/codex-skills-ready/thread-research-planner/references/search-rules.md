# スレッド検索ルール

ネタ（テーマ）に合致するスレッドを効率的に見つけるためのルール。

---

## 1. 検索の手順

### ステップ1：WebSearchで広く探す

以下のクエリパターンで検索する（複数試す）。

```
{ネタのキーワード} 2ch まとめ
{ネタのキーワード} なんJ 歴史
{ネタのキーワード} 5ch まとめ
{ネタのキーワード} 歴史 スレ
```

### ステップ2：歴史系まとめサイトで深掘り

| サイト | 特徴 | URL |
|---|---|---|
| ガバガバ歴史速報 | 歴史関連スレのみ収集。検索精度が高い | gabareki.blog.jp |
| 歴史的速報 | 歴史関連スレのみ収集 | rekishiteki-hayawari.net |
| なんJ歴史部 | 幅広い歴史スレをまとめている | nanjhistory.2chblog.jp |
| かころぐβ | 全スレッド表示。元スレ特定に便利 | kakorogu.com |

### ステップ3：8〜10個をピックアップ

- 直接テーマに合うスレ：4〜5個
- 関連する別角度のスレ：3〜5個
- 全てのスレの中身をざっと確認し、タイトルと内容にズレがないか確認する

---

## 2. 元スレ重複チェック

### なぜ必要か
同じ元スレッドが複数のまとめサイトに転載されていることが非常に多い。
重複に気づかず採用すると、前半と後半で同じコメントが出てしまう。

### チェック方法
1. まとめサイトのページ下部にある「転載元」「引用元」「元スレ」のURLを確認する
2. URLに含まれるスレッド番号（例：`livejupiter/1484924570`）を比較する
3. 番号が同じなら同一スレッド → 片方を除外する

### Chromeでの効率的な確認手順
```
1. ページにアクセス
2. JavaScriptで転載元URLを抽出：
   document.body.innerHTML.match(/http.*(?:2ch|5ch|open2ch).*(?:livejupiter|livegalileo).*/)
3. スレッド番号を記録
4. 次のまとめサイトで同じ操作 → 番号を比較
```

## 3. ユーザーの指示に応じた検索先の切り替え

### 「元スレから」と指示された場合
- まとめサイトは検索の入口として使う
- コメント抽出は元サイト（5ch.net、open2ch.net、2ch.sc等）から行う
- 元サイトの方が全コメントが揃っている

### 「まとめサイトから」と指示された場合
- まとめサイトのコメントをそのまま使用する
- ただし同じ元スレのまとめは重複として除外する
- まとめサイトのコメントは不完全な場合があることを伝える

---

## 4. 検索で見つからない場合

1. キーワードの組み合わせを変える（人物名、時代名、事件名を入れ替え）
2. 関連する時代・人物名で範囲を広げる
3. 英語キーワードも試す（世界史ネタの場合）
4. それでも見つからない場合はユーザーに報告し、ネタの変更を相談する
