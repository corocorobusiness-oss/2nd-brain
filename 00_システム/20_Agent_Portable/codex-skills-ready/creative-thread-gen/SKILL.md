# 創作スレ生成スキル（creative-thread-gen）

本物の2chスレを学習して、本物そっくりの「創作スレ」をゼロから作る。
youtube-pipeline / youtube-script-parts と同じ文体で、台本にそのまま組み込める形で出力する。

## 入力
ユーザーから「テーマ」を受け取る（例：縄文人の生活、千利休、◯◯の人物像）。
前半/後半どちら用か、光（持ち上げ）/影（ツッコミ）どちらのトーンかも確認できれば聞く（曖昧なら光寄りで作り、確認する）。

## 手順

### STEP1: 学習素材を読む（必須）
1. `2nd-Brain/03_知識ベース/YouTube・コンテンツ制作/2chスレ文体パターン.md` を**必ず**読む
   - 「📐定番構造」「🗣口調・語彙」「✍️リズム」「🚫禁止事項（禁止ワード含む）」「🔥本物らしさルーブリック」を頭に入れる
2. `スレコーパス/` から、今回のテーマに**近い実スレを1〜2本**読む（リズム・言い回しの参考。コピペはしない）
3. **`スレタイバンク.md`（本物61本・型ラベルつき）を読む**＝スレタイは原則ここから本物スワップで作る（[[2chスレ文体パターン]]の「スレタイ生成プロトコル」）。テーマに近い型の実タイトルを最低3本、頭に入れておく
4. `台本執筆ルール.md` の禁止ワード・スレタイルール（スレタイはイッチ1レス目）を確認

### STEP2: 生成（本物らしさルーブリックを必ず満たす）
2chスレ文体パターン.mdの「🔥本物らしさルーブリック」6要素を**必ず全部入れる**：
1. 懐疑・煽り合い（は？盛りすぎ／ソースは？→知識ニキが言い返す摩擦）
2. 既出ツッコミ（1回だけ）
3. 死にレス（拾われない単発1つ）
4. 知識ニキへの茶々
5. gdオチ（綺麗に締めない・逆張りニキを残す）
6. 緩急（短い即レスと長文マジレスを交互、知識ニキ連発を割る）

構造：イッチ＝スレタイを1レス目に入れる → 懐疑 → 知識ニキ → 脱線 → 収束(gd)。
口調：7割なんJ語/3割標準語。カッコは引用「」で統一・入れ子でも「」（『』は使わない＝[[2chスレ文体パターン]]実測で1%）・感情表現（白目）等を適度に。
- **列挙系スレ（47都道府県・打線・ランキング・三大/大喜利など、OPが項目を反復投下する型）は、[[2chスレ文体パターン]]の「📋 列挙系スレのOP書式フォーマット規格」からジャンルに合う型を1つ選び、OPの全項目をその1型で貫く**（区切りの混在＝はちゃめちゃ＝最大のまとめ臭。県名と中身を「の」「は」「＝」で繋がない＝全角スペースか改行のみ）。(生没年)を付けるなら生成前にWebSearchで裏取り（諸説=最有力・不詳=省略）。**規格に無いジャンル／既存型に当てはまらない書式なら、同規格の「🔧 規格の育て方」手順（本家・コーパスを実測→型判定→規格に追記→更新履歴に記録）で足してから使う**（記憶で書式を決めない）
**生成尺（デフォルト・2026-06-16オーナー指定）：1スレあたり本文2,500字前後〜3,000字以上**を必達。短く綺麗にまとめず、知識ニキの長文解説・脱線・たられば議論を厚めに盛って尺を稼ぐ（コーパスの長文スレ18本＝今川義元18千字/家康江戸首都13千字 等が手本）。レス数の目安は本文量から逆算（おおむね50〜90レス）。明示で「短め」と言われた時だけ縮める。

⚡ **尺の一発化＝「尺設計図 → 長文ファースト生成」を必ず踏む（2026-06-17・最重要）**。上から時系列で書くと毎回8割で早期終了するのが弱点。[[2chスレ文体パターン]]の「📐 尺を出す機構」通りに：
1. **書き出す前に尺設計図**（脳内でなく明示）：長文10本×130〜150字≈1,400 ／ 中レス14本×50〜60字≈770 ／ 短レス26本×20〜25字≈600 ＝ **計≈2,720字/50本** に配分
2. **長文ファースト**：まず長文10本をトピック別（内政/軍事/外交/逸話/出自/IF/通説否定/晩年/同時代比較/小ネタ）に書き切る→1,400字の床を確定→その後に短・中レスを挟んで時系列に並べ替える。長文は**120〜150字死守**・**3人以上のIDに分散**・最初の長文は**8〜10レス目以降**
3. これで初稿一発2,500到達。**実測トップアップは保険**（FAIL時のみ・不足字数÷130＝追加長文本数）
本数：用途に合わせる（前半/後半の振り分けは台本全体で調整。ただし1スレの本文は上記の2,500〜3,000字以上を下限とする）。

⚠️ **順序厳守＝長文の床が先・淡々の質感は後（2026-06-21崇徳実走の教訓）**：「8〜9割は淡々・短レス」という本物質感のルール（フル本物化）は強力だが、これを**長文ファーストより先に効かせると短レスばかりで字数不足→初稿FAIL**する。必ず①長文・中尺で2,500字の床を確定→②そのあと短レス・淡々レスを挟んで質感を作る、の順で書く。淡々ルールは"床ができた後"の工程と心得る。

**スレタイは「スレタイ生成プロトコル」に従う（一発で書かない＝最大の弱点）**：文体パターン.mdの「🧵スレタイ・返信・締め」内のプロトコル通り。①**1案目は必ず`スレタイバンク.md`から本物スワップ**（近い型の実タイトルを選び人物/数字/通説だけ差し替え・記号と語尾は本物のまま固定）→②2案目以降は型を散らして計3〜5案（必ずセリフ寸劇を1案・とかいう最大1案）→③5点満点で自己採点→④**本物との直接対比で蛇足を3点削る**→⑤ベスト1を1レス目・本文を呼応→⑥qa_checkゲート。「実は〜だった/という真実」の結論書ききりと「、で二節」の説明過多はAI臭なので捨てる。

### STEP3: 歴史ファクトチェック（必須）
スレ内の「事実主張」（年代・人物・数字・出来事）をWebSearchで裏取りする。
- 誤り → 直す
- 諸説あり → 「諸説あるけど」「〜説もある」と濁してから言う
- 断定の誇張（世界唯一・ピラミッド級など）は避ける

### STEP4: QAスクリプトで機械判定（出荷ゲート・必須）
生成物をファイルに保存して**必ず** `qa_check.py` を通す。これが"毎回同じ品質"を保証する仕組み：
```
python3 ~/.claude/skills/creative-thread-gen/qa_check.py <生成スレ.md> [--era <時代タグ>]
```
判定項目＝**字数(2500〜3200)／禁止語・複合語(必死/必殺/脱糞等)／伏字／蔑称SLUR(一発FAIL)／属性侮辱疑い／非人間化ヘイト(集団語×害虫・駆除・根絶やし＝一発FAIL)／婉曲差別疑い(◯◯人の血・民族性・帰れ系・民度・優生＝WARN→LLM)／メタ締めテンプレ／安価率(18〜32%)＋整合／欠番率(20〜30%)／冷たい立ち上がり／緩急／摩擦の三分割／うろ覚え・吐き捨て・誤答訂正・全角ｗ／ID再登場／スレタイ(型ラベル＋とかいう乱用・結論書ききり・説明過多[、で二節]・二段重ね・ASCII"・42字)／時代タグ照合**。
- ⏳ **時代タグは自動で効く（2026-06-17 残課題④で配線）**：`--era`を付けなくても、スレタイ人物から時代を自動推定し、**本文に出る他の既知人物の時代混線を自動検出**（スレタイ人物と3段以上離れた人物が3回以上＝主役級に出たらWARN。比較・引き合いの1〜2回は許容）。**youtube-pipeline連携時など、ネタ管理スプシに時代分類タグがある回は `--era 戦国` のように渡す**＝分類事故（立花宗茂[戦国]に飛鳥タグ等・2段以上でFAIL）も同時に検出できる
- `❌ NEEDS-FIX` が出たら、**末尾の【リカバリ指示】通りに直して再実行**。✅ PASS（または許容できる⚠️のみ）になるまで繰り返す
- 尺不足は短レスでなく知識ニキ長文(100〜140字)で埋める（毎回初稿は70〜80%に着地する"法則"＝トップアップ前提）
- 詳細基準は `2chスレ文体パターン.md` の「🎭清書臭の解毒」「⚔️摩擦の作り方」「🧵スレタイ・返信・締め」を参照
- 規約が気になる回は youtube-script-checker にも通す（規約8層スキャン）

⚠️ **TTS（ゆっくり読み上げ）で使う台本は、欠番→連番に戻し・ID/ｗ/安価は読み飛ばす運用**（qa_checkの欠番/ID/ｗ項目はテキスト台本前提）。字幕・テキスト閲覧用ならそのまま最強。

### STEP5: 出力
- 創作スレ本文（「登場人物「セリフ」」形式 or レス番号形式、用途に合わせる）
- 末尾に「※creative-thread-gen / オリジナル生成」と明記
- 台本のネタ帳スプシのO列（スレ可用性）には「オリジナル生成」と記録（youtube-pipeline連携時）

## 学習ループ（全自動・2026-06-17 残課題③で完成）
コーパスが増えるほど本物らしさが上がる。収集→学習→承認→適用が回る。**ルールの自動書き換えはしない**（提案専用＝AIにWrite権限を与えない設計）。

1. **収集（週次・日曜21:00 `com.claude.corpus-collect`）**：`スレコーパス/_収集キュー.txt` に貼られたまとめスレURLを `run_corpus_collect.sh` が自動でスクレイプ→**蔑称マスク(`mask_slurs.py`で【NG】化)**→重複排除(`_収集済みURL.txt`照合)→`スレコーパス/YYYY-MM-DD_収集_*.md` に保存→#youtubeに結果報告。失敗URLはキューに残って次回再試行。"発見(URL投入)"は人＝品質ゲート、収集処理は全自動
2. **学習（週次・日曜22:00 `com.claude.thread-format-learning`）**：`run_thread_format_learning.sh` がコーパスを定点観測し、ルール未収録の書式・型・言い回しを探す。証拠1本は `_学習ウォッチリスト.md` に積んで週をまたいで2本目を待ち、**証拠2本で昇格→Discord #一般に「足す？」提案**。ジョブはWrite権限なし＝`<<<WATCHLIST_UPDATE…>>>`を出力するだけで、wrapperが決定的にウォッチリストを更新（マーカー行間だけ置換）
3. **承認→検証付き適用（イベント駆動・祐馬さんが「足す」と返信したとき）**：常駐リスナー(あおい)が下の安全ゲートで適用する。**ルールを書き換える唯一の経路**：
   - ① `apply_rule_safe.sh backup <文体パターン.md>` でバックアップ
   - ② 「🔧 規格の育て方」手順で実装（Edit）
   - ③ **別エージェント(Agent)に diff を審査させる**（実装者≠確認者）＝`apply_rule_safe.sh diff` の出力を渡し「承認提案と一致するか・既存ルールを消してないか(行数減少NG)・qa_check.pyがまだ動くか」を確認
   - ④ 審査NG or 不安なら `apply_rule_safe.sh revert`（即復元・vault git10分毎auto-commitも二重の網）
   - ⑤ OKなら `clean` してDiscordに差分報告「適用したよ、違ったら『戻して』で戻すよ」
   - 「却下」と来たら `_学習ウォッチリスト.md` の却下欄に記録（再提案しない）
- 手動スクレイプも従来通り可：`thread_scraper.py <URL>` → `mask_slurs.py` → `スレコーパス/` 保存

## 禁止・注意
- 禁止ワード（死/殺す/バカ/糞）は生成時から使わない（言い換え）
- 世界史・近代戦争（幕末まで）・残虐詳細・性風俗直接表現はNG
- 現代の特定個人・団体への言及はしない
- 実在しない史実の捏造はしない（諸説は明示）

## 🛡️ YouTube規約コンプライアンス（ヘイト・差別の絶対回避・最重要）
> 詳細は `2chスレ文体パターン.md` の「🛡️ YouTube規約コンプライアンス」セクション。規約原本は同フォルダの各ポリシー.md。
- **保護対象属性（民族・人種・国籍・宗教・障がい・性別・性的指向・年齢・従軍経験・被害者）に基づく侮辱/蔑称/憎悪は一切生成しない**。煽り・逆張りは"人物の能力・判断・実績・評価"だけに向ける（属性・出自・身体・病気・民族に向けない）
- **民族・国家蔑称は絶対NG**（チョン/シナ/ジャップ/土人 等）。コーパスに混じってても学習用に読むだけ・生成に出さない → qa_check.pyのSLUR検出で機械ブロック（検出=一発FAIL）
- **対外戦争ネタ（朝鮮出兵・元寇等）は史実を中立に**。他国/他民族をsageる・現代に結びつける方向へ発展させない
- **病気/障がい/出自/身分/女性を嘲笑・性的対象化しない**（吉継の病は友情文脈のみ、秀吉の出自は"成り上がりの夢"として）
- **二重ゲート必須**：qa_check.py（禁止語＋SLUR＋**非人間化ヘイト=FAIL／婉曲差別=WARN**）→ youtube-script-checker（規約8層＋LLM文脈チェック）。両方クリアで出荷。蔑称"語"が無くても、血統本質主義(◯◯人の血/民族性)・「国へ帰れ」系・民度煽り・優生思想・集団の非人間化(害虫/駆除/根絶やし)は機械側で拾う（2026-06-17 残課題②対応）
