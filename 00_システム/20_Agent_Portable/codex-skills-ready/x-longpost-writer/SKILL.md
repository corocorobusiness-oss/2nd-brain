# X長文ポストライター（抽出→生成→評価ループ）

ネタから「ワンメッセージ」のテーマを2本抽出し、それぞれ生成→品質チェック→ランク評価→
（必要なら）強化のループを回して、**Aランク以上のポストだけ**を納品する。

## 前提

- Vault: `/Users/kabushikikaishakorokoro/Library/CloudStorage/GoogleDrive-corocoro.business@gmail.com/マイドライブ/2nd-Brain`
- **必須参照コンテキスト（生成前に必ず読むこと）**:
  - `00_システム/00_UserProfile/03_執筆スタイル(Style_Guidelines).md` — 一人称・語尾・改行・NGワード（絶対遵守。勝手に「僕」「俺」と決めつけない）
  - `00_システム/00_UserProfile/01_価値観(Core_Values).md` — 思考のOS
  - `00_システム/00_UserProfile/02_最新コンテキスト(Active_Context).md` — 最新CTA定義
  - 過去ポストがあれば `04_アウトプット/01_Xポスト/` 内を型のfew-shotとして参照
- Roblox事業の発信に使う場合は `01_プロジェクト/Roblox事業/` の発信計画・X投稿テンプレも参照する
- 入力: 文字起こし／メモ／箇条書き、何でも可。なければ1つだけ聞く。

## STEP 1: テーマ抽出（The Single Message Rule）

長文ポストはワンメッセージでなければ死ぬ。以下の基準で情報源から「宝石」を見つける。

- **抽出基準**: ①具体性（具体的なエピソード・アクションを伴うか）②独自性（本人にしか語れない一次情報か）③解決策（読者の悩みを解決する明確なノウハウか）
- 情報源全体から「感情が動いた瞬間」「強い言葉」をマーキングし、テーマ候補を切り出す
- **独立した2本のポスト**として成立する Best 2 を選定（Parallel Production）
  - 絶対ルール1: 各テーマはワンメッセージ。「AとBについて」のような複合テーマは全除外
  - 絶対ルール2: 2テーマは内容が重複しない（例: 1本はノウハウ系、1本はマインドセット系）
- 各テーマに**ナラティブ型**を割り当てる:
  1. 問題解決型（悩み→真の原因→解決策） 2. 逆算型（衝撃的な結論から紐解く）
  3. 比較型（従来vs新視点） 4. ストーリー型（失敗談→教訓） 5. パラドックス型（一見矛盾する主張→真意）

テーマ2本（タイトル案・ターゲット・型・要約）をユーザーに提示。**異論がなければそのまま STEP 2 へ進んでよい**（軽量コンテンツなので確認は任意）。

## STEP 2: 生成（300〜400文字）

本人の人格を憑依させてドラフトを作成する。

- 構成: 【インパクトのある1行目タイトル】→ 読者の深層心理を突くリード → ボディ（選択した型で展開）→ ポジティブな結び → CTA
- **品質管理ロジック（生成時に自己添削）**:
  - 冗長表現の完全削除: 「実は」「〜だと思います」「〜ということ」「せいぜい」「いわば」→ 削除しても通じるなら全削除
  - 抽象→具体の強制変換: 「様々な価値」→「体験、関係性、物語」のように映像が浮かぶレベルへ
  - **ハッシュタグは一切出力しない**（ユーザーの特別指示がない限り）
- 改行・空白行・強調・絵文字の頻度は Style_Guidelines.md のルールを再現
- CTA: `Active_Context.md` の最新キャンペーン定義から文脈に合うものを1つ（ラベルなしで配置）。**定義がなければCTAは生成しない**

## STEP 3: 品質チェック（合格/要修正）

- **Show More突破力（最重要）**: 最初の3行（約70文字）で続きを読ませるか。1行目は数字・逆説・強い問いかけか。文字数350〜500文字程度か
- **視覚的リズム**: 文字の壁がないか。Style_Guidelines の構造・余白・装飾ルールに完全準拠か。禁止装飾（太字等）・許可されていない区切り線がないか
- **トーン&マナー**: 一人称・語尾・口調の統一、NGワード、Core_Values に反する内容がないか

NGがあれば具体的な修正を適用してから STEP 4 へ。

## STEP 4: ランク評価（100点満点）

- **Show More実効率（40点）**: 誰もが指を止めるフックか
- **インプレッション維持率（30点）**: 最後まで離脱しないリズムか
- **エンゲージメント誘発力（30点）**: 「保存して見返したい」鋭さがあるか
- ランク: **S（90+）**=即投稿すべき / **A（70-89）**=合格 / **B（69以下）**=投稿すべきではない

**Bランクの場合は STEP 5 で強化し、STEP 3〜4 を再実行する（最大2周。それでもBなら正直に報告して判断を仰ぐ）。**

## STEP 5: ディテール強化（Bランク時・または「強化して」と言われたとき）

1. 思考回路の整合性: Core_Values の価値観が滲み出ているか
2. ワンメッセージの徹底: メインテーマ以外は良い文章でも削除
3. 冗長表現の断捨離: 目安300文字程度までシャープに。本筋と関係ない例え話・自分語りはバッサリ
4. 抽象→具体: 「いろいろな出費」→「衣装代」「引越し費用」。手触りのある言葉へ
5. 比較対象の明確化: 「世間一般では」→「学校の先生はこう言いますよね」のように顔が浮かぶ対象に
6. 泥臭さの開示: CTA前に「200時間」「数十万円」など払ったコストを数字で示す（ソースにある実数のみ。捏造禁止）

**最終禁止事項**: 太字(`**`)強調 / 4行以上の壁テキスト / 定義外の一人称 / 文脈無視の自分語り / 本人が使わない絵文字・記号

## 出力

- 保存先: `04_アウトプット/01_Xポスト/YYYYMMDD_テーマ名.md`（2本を1ファイルにまとめてよい）
- 各ポストに: 本文（コピペ用コードブロック）＋ランク評価結果（点数内訳）を添える
- チャットには2本の本文＋ランク＋ファイルリンクを提示する
