# YMM4 読み仮名辞書ジェネレーター

歴史系YouTube台本のテキストから、TTS（音声合成）が正しく読めない可能性のある
漢字・用語の読み仮名を抽出し、YMM4の単語辞書(.dic)形式で出力するスキル。

---

## 基本方針

- **毎回書き換え方式**：台本が送られるたびに、その台本の用語だけでdicファイルを新規作成する
- 前の台本のエントリは残さない（上書き）
- 出力先：ユーザーのワークスペースフォルダ内の `ymm4_user.dic`

### このスキルの責務（v3で明確化）

このスキルは**最終読みの頭脳ではない**。担当は次の4つ。

1. 台本からの**候補抽出**（広く拾う）
2. 固有語中心の**dic生成**（登録は慎重に絞る）
3. validator v2 による**漏れ・誤読の機械検出**
4. 候補・除外理由を残す**証拠レポート**

最終読みの品質は `reading_fix.py`・行ごとのoverride・Hatsuon・YMM4実機検証
（VoiceLength全行非ゼロ）が担う。dicは固有名詞・歴史語・史料名などの補助輪。

### 成功条件 = fail-close（漏れたまま完了扱いにしない）※最重要

自然言語で「絶対に漏れない」は保証できない。代わりに次の2つを機械的に守る。

**① 候補は広く**：台本の全漢字語を候補化し、除外する場合は理由を残す。
「有名じゃないから」「一般語っぽいから」で黙って落とすのを禁止。

**② 未解決のまま完了しない**：validator v2 が FAIL を1つでも出したら完了報告禁止。
- 台本に出た必須語（known_readings / high_risk must）のdic未登録 → FAIL
- 登録済み語の読みが検証済み読みと不一致（例: 村岡素一郎=むらおかそいちろう）→ FAIL
- 既知の誤読（bad_readings）の再登録 → FAIL

**過剰登録は「害が小さい」前提を廃止（v3変更）**：dicはOpenJTalkユーザー辞書にも
効くため、文脈依存語をdicに固定すると**誤読固定事故**になる。
`葦（あし/よし）` のような多読み語は、台本の文脈で読みを確定できる場合だけ登録し、
迷う場合は per-line override へ送るかユーザーに確認する。

### 過去に漏れた実例とその教訓

| 漏れた語 | 正しい読み | 漏れた理由 | 教訓 |
|---|---|---|---|
| 高望王 | たかもちおう | 「王号」でフルネーム人名の型に当てはまらない | 皇族の称号付き名も人名として拾う（カテゴリA） |
| 良馬 | りょうば | 一般熟語に見えて名前候補から外れた | 文脈で読みが割れる熟語も拾う（カテゴリE） |
| 2回 | にかい | 数字＋助数詞でどのカテゴリにも無かった | 数詞＋助数詞を独立カテゴリで拾う（カテゴリF） |
| 落とし所 | おとしどころ | 送り仮名つき訓読み熟語で見落とし | 訓読み複合語・送り仮名語を拾う（カテゴリE） |
| 葦 | あし | 「一般語だから」で候補から外れた | 単漢字・自然物・生活語彙を拾う（カテゴリG・家康回2026-06-30） |
| 三河後風土記 / 隆慶一郎 / 民友社 | みかわごふどき / りゅうけいいちろう / みんゆうしゃ | 史料名・著者名・出版社名の拾いが弱い | 「本・出典・刊行・社」等の文脈キュー周辺を強制候補化（カテゴリH・家康回） |
| 村岡素一郎 | むらおかもといちろう | 登録済みだが読みが誤り（そいちろう） | **漏れより危険な誤読固定**。known_readings照合をvalidatorでFAILに（家康回） |

→ カテゴリ抽出だけに頼らず、ステップ2.5の全文スイープ＋ステップ5のvalidator v2で二重に止める。

## 手順

### ステップ1：台本の受け取りと確認

1. ユーザーから台本テキストを受け取る（CSVの場合は `セリフ` 列が正）
2. ユーザーから追加の指示があれば確認する（例：名前の分割指定、特殊な読みの指定）
3. `data/known_readings.tsv`（検証済み読み）と `data/dic_history/`（過去dic）があれば読み込み、
   台本に出る既知語を自動で候補化する
   - known_readings と一致する語 → 自動採用候補
   - 過去dicにあるだけの語 → `review`（自動採用しない）
   - 同じ表記で過去に複数読みがある語 → `conflict` として停止しユーザーに確認

### ステップ2：用語の抽出と分類

台本テキストから以下のカテゴリの用語を網羅的に抽出する：

**カテゴリA：人名（最優先）**
- フルネーム（例：伊達政宗→だてまさむね）
- 名前のみ（例：政宗→まさむね）
- フルネーム（苗字＋名前）しかない場合は、名前単体も必ず別エントリで登録する
  （例：上杉謙信 がある場合 → 謙信 も登録）
- ユーザーが分割指定した場合、フルネームと名前の両方を登録
- 氏族名（例：伊達→だて、伊達家→だてけ、伊達軍→だてぐん）
- 通称・あだ名・法名（例：南光坊天海→なんこうぼうてんかい、天海→てんかい）
- **皇族・王号・称号付きの名（漏れやすい！）**
  - 〇〇王（例：高望王→たかもちおう、護良親王→もりよししんのう）
  - 〇〇天皇・〇〇上皇・〇〇院（例：後醍醐天皇→ごだいごてんのう、白河院→しらかわいん）
  - 「王」「親王」「天皇」「院」で終わる固有名は、人名として必ず1エントリで登録する

CRITICAL: 名前単体エントリの自動追加時、以下の誤検知に注意すること。
「苗字＋名前」のパターンと「固有名詞＋一般語」を混同しないこと。

**名前単体として登録すべきもの（人名の下の名前）：**
- 上杉謙信 → 「謙信」を追加 ✓
- 毛利元就 → 「元就」を追加 ✓
- 明智光秀 → 「光秀」を追加 ✓

**名前単体として登録してはいけないもの（一般語・誤分割）：**
- 小田城 → 「城」は一般語であり名前ではない ✗
- 武田軍 → 「軍」は一般語であり名前ではない ✗
- 藤原家 → 「家」は一般語であり名前ではない ✗
- 足利将軍 → 「将軍」は役職名であり名前ではない ✗
- 伊達領 → 「領」は一般語であり名前ではない ✗
- 伊達姓 → 「姓」は一般語であり名前ではない ✗
- 小田原討伐 → 「小田原」を「小田＋原」に分割するのは誤り ✗
- 龍造寺四天王 → 「四天王」は称号であり名前ではない ✗
- 蜂須賀党 → 「党」は一般語であり名前ではない ✗
- 徳川宗家 → 「宗家」は一般語であり名前ではない ✗

**判定基準：**
分割後の「名前」部分が以下に該当する場合は登録しない：
1. 一文字の一般語（城、軍、家、領、姓、党、氏 など）
2. 一般的な役職名・称号（将軍、守、太守、四天王 など）
3. 一般的な名詞（宗家、重臣、討伐 など）
4. 分割自体が不自然（小田原→小田＋原 のように地名を壊す）

**カテゴリB：地名・城名・建造物**
- 城名（例：米沢城→よねざわじょう）
- 地名（例：仙台平野→せんだいへいや）
- 河川名（例：北上川→きたかみがわ）

**カテゴリC：歴史用語・専門用語**
- 役職名（例：藩主→はんしゅ、陪臣→ばいしん）
- 戦・事件名（例：関ヶ原→せきがはら、戊辰戦争→ぼしんせんそう）
- 制度・概念（例：石高→こくだか、慶長遣欧使節→けいちょうけんおうしせつ）

CRITICAL: 助詞「の」や複数語を含む表現でも、必ず1つのエントリとして登録すること。
TTSが個別の単語に分解して誤読するのを防ぐため、フレーズ全体を1エントリにする。

例：
- 三方ヶ原の戦い → みかたがはらのたたかい（1エントリ）
- 海道一の弓取り → かいどういちのゆみとり（1エントリ）
- 建武の新政 → けんむのしんせい（1エントリ）
- 錦の御旗 → にしきのみはた（1エントリ）
- 方広寺の鐘 → ほうこうじのかね（1エントリ）

バラバラに登録すると「三方ヶ原」「の」「戦い」が別々に読まれてしまい、
全体としての正しい読みが保証されない。

**カテゴリD：打線表記（台本に含まれる場合）**
打線が含まれている場合、以下の固定ルールで読みを付ける。
打線エントリは半角数字＋全角括弧を含むため、カラム2を `1`（正規表現有効）にすること。

| 表記 | 読み |
|---|---|
| 1（遊） | 1ばんしょーと |
| 2（二） | 2ばんせかんど |
| 3（三） | 3ばんさーど |
| 4（捕） | 4ばんきゃっちゃー |
| 5（中） | 5ばんせんたー |
| 6（左） | 6ばんれふと |
| 7（右） | 7ばんらいと |
| 8（一） | 8ばんふぁーすと |
| 9（投） | 9ばんぴっちゃー |
| （DH） | でぃーえいち |
| （指） | でぃーえいち |

※ 打順とポジションの組み合わせは台本ごとに異なる。上の表は一例。
台本に書かれている実際の組み合わせに従って読みを付けること。
例：台本に「3（投）」とあれば → `3ばんぴっちゃー`

**カテゴリE：その他TTSが誤読しやすい語**
- 難読漢字（例：瓢箪→ひょうたん、甲冑→かっちゅう）
- 文脈依存の読み（例：最上→もがみ（人名時）、大御所→おおごしょ）
- 作品名・書名（例：独眼竜政宗→どくがんりゅうまさむね）
- 2ch → にちゃんねる など
- **文脈で読みが割れる熟語（漏れやすい！）**
  - 良馬→りょうば、上手→じょうず/うわて、大勢→おおぜい/たいせい
  - 一般語に見えても、台本の文脈で誤読されうるなら登録する
- **訓読み複合語・送り仮名つきの語（漏れやすい！）**
  - 落とし所→おとしどころ、立ち位置→たちいち、後ろ盾→うしろだて
  - 駆け引き→かけひき、見せ場→みせば、捨て駒→すてごま、大火傷→おおやけど
  - 「所」「場」「位置」などを含む複合語はTTSが音読み（しょ・じょう 等）で誤読しやすい

**カテゴリF：数詞＋助数詞（漏れやすい！）**
TTSが数字＋カウンターを誤読・不自然読みするものを登録する。
- 2回→にかい、3回→さんかい、1回→いっかい
- 一騎→いっき、二手→ふたて、三度→さんど、数十年→すうじゅうねん、500部→ごひゃくぶ
- 半角／全角どちらの数字表記も台本に出た形で登録する
- ※打線表記（カテゴリD）とは別。打線以外の通常の数え語はカラム2=`0`でよい

**カテゴリG：単漢字・自然物・生活語彙（漏れやすい！家康回で新設）**
- 例：葦→あし、堀→ほり、鐘→かね、陣→じん、櫓→やぐら、柵→さく、
  沼→ぬま、湿地→しっち、渡し→わたし、船着場→ふなつきば
- **「一般語だから除外」を禁止**。単漢字は原則候補化し、除外する場合は理由を残す
- `葦（あし/よし）` のように読みが割れる語は台本の文脈で確定する。
  文脈で確定できない場合はdicに焼かず、per-line override送りにするかユーザーに確認する
- 単漢字をdicに入れる場合、部分一致の暴発（例: 「堀」が「堀田」にも効く）に注意。
  台本内でその字が別の登録語の内部にしか出ないなら単体登録は不要

**カテゴリH：史料名・書名・著者名・出版社名（漏れやすい！家康回で新設）**
- 例：三河後風土記→みかわごふどき、史疑徳川家康事跡→しぎとくがわいえやすじせき、
  駿府政事録→すんぷせいじろく、隆慶一郎→りゅうけいいちろう、民友社→みんゆうしゃ
- 「解説行・出典行に出る語」はセリフの主役でないため見落としやすい。**強制抽出**の対象
- **文脈キュー**：台本中の「本」「書物」「記録」「出典」「史料」「小説」「刊行」「社」
  「寺」「鐘」などの周辺に出る固有名は必ず候補化する

### ステップ2.5：全文スイープ（候補は広く）※必須・成功条件の中核

カテゴリ抽出（ステップ2）で拾った語は「型に当てはまったもの」だけ。
これだけでは必ず漏れる。ここで台本の**100%を走査**する。

**やり方：台本を頭から末尾まで、漢字を含む語を1つずつ見る。**
各語について「ゆっくり（TTS）のデフォルト読み」を頭の中で再生し、
それが**意図した読みと1文字でも違うなら候補にする**。
有名/無名・固有名詞/一般語で足切りしない。出てくる語は全部判定対象。

候補は3段階に分類し、`reject` にも必ず理由を残す（除外理由の記録が「漏れ」と
「意図した除外」を区別する唯一の証拠になる）：

- `must` … dic登録する（読みが安定する固有語・歴史語・史料名など）
- `review` … 文脈確認・Web裏取り・ユーザー確認のいずれかが必要
- `reject` … 登録しない。理由を1行残す（例：「TTSが正しく読む一般語」「別エントリに包含」）

走査時に必ずチェックする観点：
1. **漢字を含む語を全部見る** — 有名/無名で判断しない。良馬・高望王のような語を逃さない
2. **王・親王・天皇・院・公・卿で終わる語** → 皇族・称号付き人名（カテゴリA）
3. **数字＋助数詞（〇回・〇騎・〇手・〇度・〇万 など）** → カテゴリF
4. **送り仮名つき・「所/場/位置/盾/際/方/手」等を含む複合語** → カテゴリE（落とし所など）
5. **音読み/訓読みが割れる二字熟語** → 文脈の読みでカテゴリE
6. **既に抽出済みの人名の「下の名前」単体**が漏れていないか（カテゴリAのルール）
7. **熟字訓・当て字**（例：流石→さすが、土産→みやげ）
8. **元号・年号・干支・暦**（例：承平→じょうへい、天慶→てんぎょう）
9. **動詞・形容詞の難読活用形**（例：陥れる→おとしいれる、与する→くみする）
10. **単漢字・自然物・生活語彙**（例：葦・堀・鐘・陣・櫓）→ カテゴリG
11. **「出典・刊行・社・史料」等の文脈キュー周辺の固有名** → カテゴリH

### ステップ3：文脈に応じた読みの判定

CRITICAL: 以下の点に特に注意すること。歴史系台本では文脈依存の読みが非常に多い。

1. **人名の特殊読み**を必ずWeb検索で確認する
   - 最上義光 → もがみ**よしあき**（よしみつではない）
   - 愛姫 → **めご**ひめ（あいひめではない）
   - 村岡素一郎 → むらおか**もと**いちろう（そいちろうではない）
   - このような特殊読みは必ずWeb検索で裏取りすること

2. **同じ漢字でも文脈で読みが変わるもの**
   - 最上 → もがみ（人名・氏族名）vs さいじょう（一般語）
   - 石高 → こくだか（歴史用語）vs いしだか（一般語）
   - 葦 → あし vs よし（地域・文脈による）

3. **確信が持てない読みは必ずWeb検索で確認する**
   - 特に人名の読みは間違いが許されない
   - Agent ツールを使って複数の名前を一度に検証すると効率的
   - 「たぶんこう読む」で確定しない。少しでも揺れたら検索する
   - 検索でも確定できなかった語は、勝手に決めずユーザーに読みを質問する

4. **既存dic・過去dicは正解源ではなく候補源**
   - 過去に登録済みでも読みを鵜呑みにしない（村岡素一郎事故の教訓）
   - 正解源は `data/known_readings.tsv`（検証済み・典拠つき）だけ

### ステップ3.5：dicに入れる範囲の判定

**dicに入れる：**
- 固有名詞（人名・氏族名・通称・法名）
- 歴史語・役職・制度
- 史料名・書名・著者名・出版社名
- 地名・城名・事件名
- 読み揺れのある自然物・普通名詞（その台本の文脈で読みが確定するもの）

**dicに入れない：**
- `後`・`戦`・`上`・`方`・`行った`・`城`単体 など、文脈で読みが変わる超一般語
- 文脈で読みが確定できない多読み語（→ per-line override かユーザー確認へ）
- `/`・`_`・`<NUMK>` などHatsuon表記そのもの
- TTSが確実に正しく読む一般語（→ `reject` 理由を残して除外）

### ステップ4：dic形式での出力

**フォーマット：** タブ区切り、6カラム

通常エントリ：
```
1	0	単語	読み		1
```

打線エントリ（正規表現有効）：
```
1	1	2（二）	2ばんせかんど		1
```

各カラムの意味：
- カラム1: 有効フラグ（常に `1`）
- カラム2: 正規表現フラグ（通常 `0`、打線エントリは `1`）
- カラム3: 単語（表層形 / 変換前）
- カラム4: 読み（ひらがな / 変換後）
- カラム5: 空（タブのみ）
- カラム6: フラグ（`1` で統一）

CRITICAL:
- エンコーディング: UTF-8、改行コード: LF
- 打線エントリ（半角数字＋全角括弧を含むもの）はカラム2を必ず `1` にすること。
  `0` のままだとYMM4が正規表現マッチを行わず、辞書エントリが適用されない。
- 全エントリのカラム6は `1` で統一する。

### ステップ4.5：ファイルの書き換え

1. ワークスペースフォルダの `ymm4_user.dic` を新しい内容で**完全に上書き**する
2. 前の台本のエントリは残さない
3. 書き換えたエントリ数をユーザーに報告する

### ステップ5：validator v2 による機械検証 ※必須・FAILが残ったら完了禁止

```bash
python3 <このスキル>/scripts/validate_dic.py <出力したymm4_user.dic> \
  --script <台本.csv> --json candidate_report.json
```

validatorが検証すること：
- **FAIL（1件でも残ったら完了扱い禁止）**
  - フォーマット不正・重複・ASCIIダブルクォート・読みへの漢字混入・空読み
  - 全角括弧エントリのカラム2指定ミス
  - 台本に出た必須語（known_readings / high_risk must）のdic未登録
  - 登録読みが検証済み読み（known_readings）と不一致
  - 既知の誤読（bad_readings）の再登録
  - known_readings内の読み衝突
- **WARN（目視レビューの待ち行列。全件、登録するか除外理由を記録する）**
  - dicでカバーされない漢字連続・単漢字・数詞助数詞の候補
  - 高リスク語（high_risk review）の未登録
  - 台本に出現しない登録語（前回の残骸）
- **INFO**：エントリ数・候補数・カバレッジ

FAILが出たら該当語を修正して再実行。**FAIL 0 になるまでループする。**
WARNは1件ずつ判断し、登録しない場合は除外理由を完了報告に含める。

### ステップ6：正確性チェック（誤読ゼロ）※必須

出力前に、登録した**全エントリの読みが正しいか**を1件ずつ見直す。
網羅できていても読みが間違っていれば言い換えが発生する＝失格。

チェック項目：
1. **人名・地名・元号は読みを再確認する**
   - 特殊読みを記憶だけで書いていないか。少しでも不安なら**Web検索で裏取り**する
2. **同形異音語が文脈に合った読みになっているか**
3. **送り仮名・複合語の読みが自然か**（落とし所→おとしどころ 等）
4. **タイプミス・ひらがな変換ミスが無いか**（拗音・促音・長音の抜け落ちに注意）
5. **確定できない読みが1つでも残っていないか**
   - 残っていれば、推測で確定せずユーザーに質問する

### ステップ7：完了条件と報告

以下を**全て**満たしたときだけ完了報告する：

- [ ] validator v2 実行済み・**FAIL 0**
- [ ] `missing_must_candidates = 0` / `wrong_known_readings = 0` / `bad_readings再発 0` / `conflict 0`
- [ ] WARN全件に判断済み（登録 or 除外理由の記録）
- [ ] 全エントリの読みを見直し済み（ステップ6）

完了報告に含めること：
- エントリ数、validator結果（PASS＋WARN件数）
- 特に注意した文脈依存の読み
- `reject` した候補と理由（証拠レポート）

なお最終品質は後段が担保する：`reading_fix.py` → per-line override → Hatsuon →
YMM4実機で VoiceLength 全行非ゼロの確認。dicのPASSは「入口の合格」であって
制作完了ではない。

### ステップ8：読み資産の更新（制作後）

- 完成dicを `data/dic_history/YYYY-MM-DD_<ネタ名>.dic` として保存する（後から書き換えない）
- 今回Web裏取りまで済んだ安定読みは `data/known_readings.tsv` へ昇格候補にする
  - 昇格条件：全動画で読みが安定／文脈依存でない／典拠あり／実機で問題なし
  - 昇格してはいけない：今回だけ正しい読み・未確認の推測読み・同表記多読み語・
    短すぎて部分一致暴発しやすい語・`後`/`戦`/`上`/`方`/`城` など文脈依存の強い語
- 誤読事故が起きたら `data/bad_readings.tsv` に追記する（二度と通さない）
- 文脈依存で今回だけ正しかった読みは known_readings に入れず、必要なら
  `data/context_readings.tsv` に文脈つきで記録する

## 読み資産ファイル（data/）

| ファイル | 役割 | validatorでの扱い |
|---|---|---|
| `known_readings.tsv` | 検証済みの安定読み（正解源・典拠つき） | 台本に出たら登録必須＋読み照合。違反FAIL |
| `bad_readings.tsv` | 二度と通してはいけない誤読 | 一致したら即FAIL |
| `high_risk_terms.tsv` | 台本に出たら候補必須の高リスク語 | must=FAIL / review=WARN |
| `ignore_terms.tsv` | 候補抽出から外してよい一般語 | WARN抑制のみ（FAILには無関係） |
| `dic_history/` | 動画ごとの完成dic（候補源・書き換え禁止） | 参照のみ |

回帰テスト：`python3 tests/test_validate_dic.py`
（家康回2026-06-30の実漏れ6語＋村岡素一郎誤読をfixture化。これが通らないvalidatorは配布禁止）

## トラブルシューティング

### 読みの正確性に確信が持てない
原因：珍しい人名や古語
解決策：必ずWeb検索で確認する。確認できなかった場合はユーザーに質問する。

### YMM4で打線エントリが適用されない
原因：カラム2が `0` になっている（正規表現が無効）
解決策：打線エントリ（半角数字＋全角括弧を含む表記）はカラム2を `1` にする。

### validatorのWARNが多すぎる
原因：一般語がignore資産に足りていない
解決策：TTSが確実に正しく読む語だけを `data/ignore_terms.tsv` に追記する。
読みが1ミリでも割れる語（葦・堀・城 など）は絶対にignoreへ入れない。

### 同じ語なのに動画によって読みが違う
原因：文脈依存語（葦=あし/よし 等）
解決策：known_readingsへ固定せず、その台本での読みをdicに入れるか per-line override で対応。
迷ったらユーザーに確認する。
