# 設計書：creative-thread-pipeline（創作スレ台本 一気通貫スキル）

- 作成: 2026-06-20
- 起点: 平将門の祟り回の制作過程（創作スレ→YMM4台本）を再現可能なスキルに昇華する
- 状態: **設計フェーズ（実装前・要レビュー）**

---

## 1. 背景と目的
平将門回で「創作スレ（オリジナル2chスレ）→ 解説者パート → YMM4取り込み用CSV/xlsx」を作ったが、
**組み立て(アセンブル)とxlsx化・ビューアは使い捨てスクリプト(gen_csv.py / make_xlsx.py)で手組み**だった。
この工程を**毎回再現できるスキル**にする。生成系は既存スキルを呼び、本スキルは
**「組み立て規約 ＋ 検品ゲート ＋ 出荷物の生成」**を担う。

## 2. 既存スキルとの関係（重複回避が最重要）
| スキル | 役割 | 本スキルとの関係 |
|---|---|---|
| `youtube-pipeline` | **実スレscrape版**の6ステップ一気通貫（thread_scraper.pyで本物スレ取得） | 兄弟。creative版が本スキル。結合エンジンは共有化を狙う |
| `creative-thread-gen` | 創作スレ（前半/後半）生成＋qa_check（16+ゲート） | **STEP2で呼ぶ**（×2本）|
| `youtube-script-parts` | 冒頭・解説・前置き・締め生成 | **STEP3で呼ぶ**（解説は対象用語ぶん）|
| `youtube-script-checker` | 規約3段階チェック | **STEP5で呼ぶ** |
| `ymm4-dic-generator` | 読み仮名dic（毎回書き換え・漏れ/誤読ゼロ） | **STEP6で呼ぶ** |
| `ymm4-pronunciation-checker` | YMM4取り込み後の発音照合 | 任意・STEP7（dic適用後）|

→ **本スキルは生成を再実装しない。オーケストレーション＋アセンブル＋出荷に徹する。**
→ アセンブル規約は `scripts/assemble_csv.py` ＋ `refs/組み立て規約.md` に集約し、
   将来 youtube-pipeline もこの共有エンジンを採用して規約の二重化を防ぐ。

## 3. スキル定義
- **名前**: `creative-thread-pipeline`
- **トリガー**: 「創作スレ台本」「オリジナルスレで台本」「スレ台本一気通貫」「転載なしで台本」「creative pipeline」
- **立ち位置**: ネタ確定後〜YMM4取り込み直前までの創作スレ版フルパイプライン
- **各ステップ後にユーザー確認**（youtube-pipeline踏襲・暴走防止）

## 4. パイプライン全体
```
STEP1 ネタ確定（テーマ＋前半=タイトルネタ/後半の振り分け）
      ※ネタ出しは neta-research / thread-research-planner（任意・前段）
STEP2 創作スレ生成 ×2  →  /creative-thread-gen（前半・後半）
      ・qa_check 全項目PASSまで回す（字数/禁止語/SLUR/語尾/AI癖/SCAF/感情温度/壁化/欠番/時代タグ…）
      ・出力＝「番号: 本文」形式（>>Nアンカー保持）
STEP3 解説対象の自動選定 → 解説者パート生成  →  /youtube-script-parts
      ・スレ本文に出た難用語/固有名から解説対象をN個抽出しユーザー提案→確定
      ・冒頭/前置き/締め＋解説×N を生成
STEP4 アセンブル【本スキルの心臓部】→ /scripts/assemble_csv.py
      ・構成順に結合し、話者・大/下・返信先・解説挿入・句読点改行を機械付与
      ・出力＝CSV(utf-8-sig) ＋ xlsx ＋ ~/Desktop複製 ＋ 会話ビューア(show_widget)
STEP5 規約3段階チェック → /youtube-script-checker（8層＋LLM文脈＋ファクト）
STEP6 読み仮名dic → /ymm4-dic-generator（毎回書き換え・漏れ/誤読ゼロ）
STEP7 (任意) 発音チェック /ymm4-pronunciation-checker、タイトル/サムネ/概要欄 /youtube-script-parts、投稿予約
```

## 5. 入出力
**入力**：テーマ（人名/事件）、前半後半の振り分け方針、解説対象の希望（任意）、保存先フォルダ
**出力（保存先 = `01_プロジェクト/YouTube/創作スレ下書き/YYYY-MM-DD_テーマ`）**：
- `_台本.csv`（YMM4取り込み・utf-8-sig・句読点改行）
- `_台本.xlsx`（折返し整形）
- `.md`（人間可読・構成見出し付き）
- `~/Desktop` への複製（CSV/xlsx/dic）
- 会話ビューア（show_widget・スレタイ/話者/返信↳）
- `ymm4_user.dic`（STEP6）

## 6. CSV組み立て規約【正本＝refs/組み立て規約.md にも複製】
### 6.1 列構成
`登場人物, セリフ, 備考, 番号`（番号は「連番」または「連番>>返信先連番」）
### 6.2 構成順（備考列のセクション）
冒頭 → スレ① →〔解説:用語A〕→〔解説:用語B〕→ 前置き → スレ② →〔解説:用語C〕→〔解説:用語D〕→ 締め
※解説は**該当用語が初出したスレレスの直後**にスレの合間へ挿入
### 6.3 話者
- スレタイ（各スレ1レス目）＝**男性A**
- 以後 **男性A→女性B→男性C→女性D→男性E** をレス順にローテ
- 解説者パートは全部「解説者」
### 6.4 大（長文ラベル）
- `>>`/`↑`除去後の本文が **88字以上 → 大**
### 6.5 下・返信先（番号列 `N>>M`）
- 返信（本文が`>>`始まり or スレタイ反応）→ **下** ＋ 番号 `N>>M`（M=返信先のスレ内連番）
- **スレタイへの反応＝`>>1`**（各スレ冒頭の反応バースト）
- **下大→大（大優先）**：長文かつ返信でも「大」にし、下も>>も付けない
- **大の直後のレスは下も>>も付けない**（返信でも素の番号）
### 6.6 返信先は「会話の流れ」準拠（最重要・平将門①66の教訓）
- 返信先は**直近 or 同一話題のwall**へ。
- **話題が変わった後ろのレスへ戻るジャンプは禁止**（間に別話題が挟まる>>は不自然）。
- 自動導出時、後方ジャンプ（gap≥3かつ間に別話題）を検出したら**警告して直近へ寄せる**。
### 6.7 句読点改行（TTS用）
- セリフは `。、！？` の直後で改行（既存改行は一旦解除してから付与）
### 6.8 文字コード
- CSV=UTF-8(BOM付き/utf-8-sig)・LF。xlsxはセリフ列を折返し表示。

## 7. 自動化の肝（今回"手作業"だった所をコード化）
1. **返信マップの自動導出**：手マップを廃し、creative-thread-genの`>>N`アンカー＋スレタイ反応検知＋
   6.6の会話流れチェックから**自動生成**。スレ内部番号→CSV連番のマッピングを内部で持つ。
2. **番号振り直しの非破壊化**：欠番の不規則化（AI臭回避）とアンカー整合を**同時に保つ**。
   今回の「post-30一括振り直しで>>が壊れた」事故を再発させない＝振り直し時に本文`>>N`も連動更新。
3. **解説挿入アンカーの自動特定**：解説対象用語が初出するレス番号を本文走査で自動検出し、挿入位置に。
4. **大/下/>>の機械付与**：6.3〜6.6を決定的に適用（手判定ゼロ）。

## 8. 同梱物（skill配下）
- `scripts/assemble_csv.py` … threads(2)＋parts → CSV/MD（gen_csv.pyの汎用版）
- `scripts/to_xlsx.py` … CSV→xlsx＋Desktop複製（make_xlsx.pyの汎用版）
- `scripts/viewer_template.html` … 会話ビューア（スレタイ/話者色分け/返信↳）。show_widgetで投入
- `refs/組み立て規約.md` … 6章の正本（人間レビュー用）
- `refs/サンプル/` … 平将門の最終CSVを参照見本として同梱

## 9. エッジケース・失敗モード
- 解説対象がスレに出てこない → ユーザーに代替提案（用語追加 or 解説スキップ）
- qa_checkがPASSしない → creative-thread-genに差し戻し（本スキルは検品で止める）
- 大が連続しすぎ/壁化WARN → スレ側の問題なので生成スキルへ戻す（アセンブルでは直さない）
- 返信先の自動導出が会話流れ警告を出す → ユーザーに該当番号を提示して確認

## 10. 既存youtube-pipelineへの波及（任意・第2フェーズ）
- `assemble_csv.py` を共有エンジン化し、youtube-pipeline STEP2-4/STEP4 もこれを呼ぶよう差し替え。
- これで実スレ版／創作版の**両方が同一の組み立て規約**を使う＝規約の二重メンテ解消。
