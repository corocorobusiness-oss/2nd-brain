## 2. アラート条件（該当時だけ出力・出力なし＝正常）

1. 🕰️ **台帳の陳腐化**: built_at が8日超前 → 再生成を促す（**先頭に出す**＝これが古いと他の警告の信頼性も落ちるため）
2. ⏰ **公開予定の超過**: publish_date が過去なのに briefed/in_production のまま＝公開停止の兆候
3. 📅 **期限接近**: 7日以内に公開予定なのに台本未完成 → 前倒し指示
4. 📦 **在庫枯れ**: 出荷可能在庫（script_done）が2本未満

閾値は `--stale-days / --horizon-days / --min-stock` で調整可。`--today` でテスト可。
exit 0＝実行成功（警告有無問わず）／exit 2＝台帳が読めない（黙って死なない）。
