# Codex x Claude Code x Obsidian 運用ルール

最終更新: 2026-07-01

この文書は、祐馬さんのSecond Brainを中心に、Claude Code / Codex / Obsidianを1つのチームとして動かすための共通ルール。

## 結論

```text
Codex = 日常のメイン入口（移行フェーズ1）
Claude Code = 補助・退避先
Obsidian = 正本・共通記憶
06_エージェント運用 = AI管制塔
Claude Code Channels = 受付・通知・軽い指示
```

普段はCodexに話しかける。Claude Codeは、Codexで権限や接続が足りない作業、既存Claude前提の自動化、長文の壁打ち、緊急退避が必要な時に使う。

ただし、`agent-run` と既存のlaunchd/cron自動化は段階移行中。Codex切替完了とは扱わず、既存ジョブの停止・置換・自動実行ONは人間確認を挟む。

## 基本思想

Obsidian Vaultを「メモ置き場」ではなく、AIチームの中央リポジトリとして扱う。

- 決定事項、ルール、目標、学びはObsidianに戻す
- AIの現在地、未処理、確認待ち、実行キュー、ヘルスチェックは `06_エージェント運用/` に集約する
- Claude CodeとCodexに別々の記憶を育てない
- `CLAUDE.md` と `AGENTS.md` は「同じ思想の別入口」として揃える
- エージェント固有のキャッシュ、ログ、認証情報はSecond Brainに保存しない

## 役割分担

### Codex

日常のメイン入口。祐馬さんとの会話、方針整理、ファイル操作、自動化、検証、帳簿、整理をまず担当する。

向いていること:
- Obsidian内のファイル作成、追記、整理
- AI帳簿、CSV、スクリプト、検証
- YMM4辞書、台本チェック、制作進行管理
- ルール、テンプレ、仕様書の実装
- 大量ファイルの確認や安全な一括反映
- YouTubeネタの切り口、構成、台本の方向性
- 事業戦略、導線設計、コンセプト整理

### Claude Code

補助・退避先。Codexで権限や接続が足りない作業、既存Claude前提の自動化、長文の壁打ち、緊急退避が必要な時に使う。

向いていること:
- Codexで権限・接続・ツール都合により完了できない作業
- 既存Claude前提の自動化の保守
- 長文、台本、投稿文、教材文の壁打ち
- 複雑な判断の別視点レビュー
- 緊急時の退避先

### Obsidian

正本。AIの記憶ではなく、事業OSとして扱う。

保存するもの:
- 目標、ルール、ワークフロー
- YouTube台本ルール、企画ストック、投稿履歴
- 売上、帳簿、月次レポート
- 自動化の導入台帳、手順、失敗ログ
- 学び、判断、引き継ぎメモ

保存しないもの:
- トークン、Cookie、認証情報
- セッション履歴、キャッシュ、実行ログの生データ
- `.claude` / `.codex` のランタイムフォルダ丸ごと

### 06_エージェント運用

AI管制塔。24時間AIエージェント運用の現在地、未処理、確認待ち、実行キュー、ヘルスチェック、判断ログを管理する。

使い分け:
- `00_司令塔/` = 起動時に読む現在地・優先順位・権限
- `10_Inbox/` = 未処理・確認待ち
- `20_実行キュー/` = 今日やる、待機中、完了
- `30_ヘルスチェック/` = Watchtowerや自動化監査の要約
- `40_判断ログ/` = 決定事項と保留判断
- `50_レポート/` = 日次・週次のAI運用レポート

### Claude Code Channels

本線にはしない。スマホからの受付、軽い指示、通知、過去ログ参照に限定する。

重要な決定がChannelsやDiscordで出た場合は、Obsidianへ戻してから正本にする。

## 使い分けルール

```text
考える・書く・決める       -> Codex（必要ならClaude Codeで壁打ち）
作る・整える・検証する     -> Codex
残す・引き継ぐ・再利用する -> Obsidian
現在地・未処理・確認待ち   -> 06_エージェント運用
スマホから軽く投げる       -> Claude Code Channels
```

迷ったらCodexを入口にする。Codexが「これはClaude Code向き」または「この環境では権限不足」と判断したら、Claude Codeへ渡す依頼文を作る。

## 共通スキルの置き場

祐馬さんのVaultでは、記事でいう `_shared-ai/` 相当を以下に置く。

```text
00_システム/20_Agent_Portable/
```

役割:
- `specs/` = 共通ルール、設計書、移行仕様
- `codex-skills-ready/` = Codexへ移植済み・移植可能な事業スキル
- `archive/` = 旧Claude資産や移植元
- `backup_automation/` = 再現可能なバックアップ手順

## Codexへ渡す時の依頼テンプレ

```text
目的:

背景:

触ってよい場所:

参照してほしいファイル:

やってほしいこと:
1.
2.
3.

やらないでほしいこと:

完了条件:

Obsidianへ残す記録:
```

Codexは、Claude Codeへ渡す時や、別セッションへ安全に引き継ぐ時にこの形へ圧縮する。

## 終了時ルール

作業後は、必要に応じて以下へ戻す。

- ルール変更 -> `00_システム/10_Agent/rules.md`
- 手順変更 -> `00_システム/10_Agent/workflows.md`
- 目標変更 -> `00_システム/10_Agent/goals.md`
- 学び -> `00_システム/10_Agent/learning-log.md`
- AI運用の現在地・確認待ち・実行キュー -> `06_エージェント運用/`
- 事業の進捗 -> 該当プロジェクト or 今日の日誌

大事なのは、会話や各AIの中に記憶を閉じ込めないこと。最後はObsidianに戻す。
