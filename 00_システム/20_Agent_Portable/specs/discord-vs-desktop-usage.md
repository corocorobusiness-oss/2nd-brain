# Claudeチャンネルズ（Discord）と Claude Codeデスクトップの棲み分け

作成: 2026-06-16 ／ オーナー: 池田祐馬

## 基本コンセプト

軸は「**いる場所・デバイス**」と「**タスクの重さ**」の2つ。

|  | Claudeチャンネルズ（Discord） | Claude Codeデスクトップ |
|---|---|---|
| 正体 | 24時間常駐の「受付・秘書」 | 腰を据えて使う「作業場・工房」 |
| デバイス | スマホ／外出先・運転中・寝る前 | Mac mini前に座ってるとき |
| やり取り | 短い一往復（投げて受け取る） | 長い対話・反復・確認しながら進める |
| 正本 | ならない（通知・受付・ログ参照） | こっちで作って2nd-Brainに残す |

> CLAUDE.md の方針（「Discordはスマホ受付・通知・過去ログ参照用。正本にしない／本線はCodexメイン移行フェーズ1・Claude Code補助/退避先・Obsidian正本」）に準拠。

## 振り分け

### Discord（チャンネルズ）に任せること
反射的にやってほしい定型処理と、外にいる間の受付。
- 売上報告・レシート画像 → 自動記帳（#お金）
- メモの放り込み → 2nd-Brain保存（#メモ）
- タスク登録・「済」消し込み・期日督促の受け取り
- 「ネタ思いついた」「これ調べといて」の軽い投げ込み
- 自動化の定期実行レポート・エラー自己申告の受信（#レポート）
- Uber集計・週次プランなどの結果通知を受け取る場所

### Claude Codeデスクトップで腰を据えること
成果物を作る・仕組みをいじる・画面で確かめる。
- YouTube台本の制作パイプライン（リサーチ→構成→執筆→チェック→dic）
- 画面確認が要るもの（YMM4発音チェック、Uberスクショ確認、ブラウザ操作）
- Vault整理・知識ベースの再編・ナレッジ昇華のレビュー
- 自動化スキルそのものの設計・修正、CLAUDE.md/ルールの編集
- freee周りの設定変更・込み入った経理判断
- 複数ファイルまたいで考える／やり直しながら詰める系すべて

## 迷ったときの判断ライン

- **30秒で終わる定型 or スマホしかない → Discord**
- **作品・仕組み・確認が要る or PCの前 → デスクトップ**

一言で：**Discordは入口、デスクトップは作業台**。
Discordで受けた重いネタは「これはデスクトップで腰を据えてやろう」とそのまま移送する。
