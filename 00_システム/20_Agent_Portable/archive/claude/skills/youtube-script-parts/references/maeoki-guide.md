# 前置きパート作成ガイド

## 目的
ユーザーから提供された「これまでの議論の内容（動画前半のハイライト）」や「テーマ」をもとに、YouTube台本の「前置きパート」を**ゼロから書き起こす**。過去に作成された台本と**完全に同一のフォーマット・トーン**で新規出力すること。

## 出力ルール・制約

AIは、以下の厳密なルールに従って**ゼロから文章を生成**し、過去の原稿の王道パターン（前段の振り返り、新たな疑問の提示、次の展開への誘導）と見分けがつかないレベルで完全にトーンとフォーマットを統一すること。

### 1. 文字数と役割のルール
- 前置き文はシナリオ内でスレッドが変わり、話のテーマが変わるときに視聴者を混乱させないために挿入するもの。
- 文字数は**100〜200文字程度**で作成すること。

### 2. 文章構成の王道パターン（前置きパートの型）
必ず以下の構成と決まり文句で文章を組み立てること。
- **導入（前段の振り返り）**:
  「ここまでは、」から始め改行し、「〇〇についての（さまざまな）議論を見てきたが、」というように、それまでどのような話をしてきたかを簡潔にまとめる。
- **転換・疑問の提示**:
  「『〜？』」「『〜』など、」と、スレ民や視聴者の視点に立った新たな疑問や別の切り口の意見を提示する。「〜と、疑問を持つ者も現れた。」（または「〜と、〇〇に疑問を持つ者も現れた。」）などの表現で繋ぐ。
- **共感と次の展開への誘導**:
  「たしかに〜」「彼は一体どんな武将だったのだろうか？」「〜本当にあるのだろうか？」など、その疑問に共感したり、さらに興味を惹くような問いかけを行う。
- **締めの決まり文句**:
  「引き続き、2ちゃんねるの歴史オタクたちの議論を見ていこう。」と定型文を一字一句違えずに入れ、次のパートへ誘導する。

### 3. 厳密なフォーマット（文字装飾・レイアウト）
- 1行が長くなりすぎないよう、テンポ良く意味の切れ目で改行を入れる。
- ここではダブルクォーテーション（ `"` ）による行囲みは**行わないこと**。本文のテキストをそのまま出力する。
- （セリフの鍵括弧「」はそのまま使用して構わない）
- 余計な前置きや後書き（「以下のように作成しました」等）は一切出力せず、本文**のみ**を出力すること。
