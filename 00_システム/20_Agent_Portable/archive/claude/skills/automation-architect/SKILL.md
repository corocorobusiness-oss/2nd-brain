# 自動化アーキテクト（最適な自動化プランを提案するスキル）

ユーザーがやりたい自動化・繰り返し作業を受け取り、**判断ルールに沿って最適な仕組みを設計して提案する**。
作って終わりではなく「どの型がベストか・なぜか・どう作るか・ユーザーの手間は何か」まで出す。

## 判断ルール（この順で考える）

### STEP1: きっかけ（トリガー）で型を選ぶ
| きっかけ | 最適な型 | 例 |
|---|---|---|
| 何かが**起きたら**動く（メッセージ/画像/数字/メール/ファイル変化） | **イベントトリガー（hook）** ← 最優先・最楽・最速 | Discordに数字→売上記録、レシート画像→freee仕訳 |
| 決まった**時刻**に動く（予測可能・定期） | **スケジュール（launchd）** | 毎晩23時Uber集計、月曜モデル再学習 |
| 人が**その都度**起動する（重い創作・調査） | **スキル（オンデマンド）** | 台本作成、ネタリサーチ |
| **目標だけ**与えて自分で判断・連続実行 | **自律ループ（ゴール駆動）** | 「YouTube伸ばして」で調査→台本→投稿まで |

**鉄則: 自然なイベントがあるなら、スケジュールより必ずイベントを選ぶ**（時刻待ちより速く・無駄がない）。

### STEP2: ロジックは必ずスキルに1回だけ書く
処理本体はスキル化（1 Source Multi Use）。手動でも・スケジュールでも・イベントでも同じものを呼べる形にする。重複コードを作らない。

### STEP3: 現実への接続が要るならMCPを確認
freee / Google（Drive/Sheets/Gmail/Calendar）/ Uber(ブラウザ) / Playwright / Discord など、既存の接続で足りるか確認。足りなければ追加提案。

### STEP4: 安全・コストの確認
- 不可逆操作（投稿・送金・削除）は人間の承認を挟む設計に
- 規約グレー（非公式API高頻度等）は低頻度・人間ペースに
- ログイン切れ等の失敗時にユーザー通知するフォールバックを入れる

## 既存インフラ（提案時に前提として使う）
- **Discord用途別チャンネル**: メイン/ youtube/ お金/ メモ/ レポート/ Roblox各種 → イベントトリガーの入口に最適
- **launchd**: 既存ジョブ多数（cronは使わない＝macOS認証不可。[[cronではなくlaunchd]]）
- **MCP接続済み**: freee, Google各種, Playwright(ブラウザ), Discord
- **2nd-Brain**: ナレッジ・制作ログの蓄積先

## 出力フォーマット（提案書）
```
【自動化提案：◯◯】

■ 結論：この作業は「△△型」がベスト
（イベント/スケジュール/スキル/自律ループ のどれか＋一言理由）

■ 仕組みの全体像
きっかけ → 処理（スキル名） → 出力先
（必要ならMCP・データ保存先も）

■ 実装プラン
1. スキル化する処理：…
2. トリガー：（イベントなら何の発生時／スケジュールなら何時）
3. 出力：どこに何を出す
4. フォールバック：失敗時どうする

■ ユーザーの手間
（理想は「ゼロ」or「◯◯を送るだけ」）

■ 効果と注意
（楽になる度合い・安全/規約/コストの注意）
```

## 動き方
1. ユーザーの「自動化したいこと」を聞く（曖昧なら1つだけ確認）
2. 判断ルール STEP1〜4 を当てて型を決める
3. 提案書フォーマットで出す
4. 「これで作る？」と確認 → OKなら実装（スキル作成＋トリガー設定）
5. 複数案あるときは"一番ラクで効果大"を推し、代替も併記
