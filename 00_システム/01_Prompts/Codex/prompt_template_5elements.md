# 5要素プロンプトテンプレート（Codexアプリ用）

Codexへの依頼文はこのテンプレートを使って書くと品質が安定する。

---

## テンプレート（コピペして使う）

```
【ゴール】
〇〇向けに、〇〇を作りたい。最終的に〇〇で使う。

【参照ファイル】
- 〇〇: （ファイルパスまたはフォルダ名）
- 〇〇: （ファイルパスまたはフォルダ名）

【出力形式】
〇〇形式で。〇〇。

【制約条件】
- 文体は〇〇
- 〇〇は使わない
- 〇〇の順で進める

【確認観点】
最後に以下をセルフレビューしてから返す：
- 〇〇
- 〇〇
```

---

## 各要素の書き方ガイド

### ゴール
- 何を作るか + 誰向けか + どこで使うか の3点を書く
- NG例: 「スライドを作って」
- OK例: 「Codexをこれから使う初心者向けに、勉強会で使う10枚のスライドを作りたい」

### 参照ファイル
- Codexに見せたいファイルやフォルダを指定
- `@` 記号でファイル指定できる（`@ファイル名`）
- プロジェクトフォルダ内のファイルは自動参照されることもある

### 出力形式
- ファイル形式: `.md`（最強）/ `.pptx` / `.pdf` / `.txt`
- 枚数・文字数・構成（見出し何階層か等）を指定
- 「Markdownで、見出し2階層まで、箇条書き多用で」が汎用的

### 制約条件
- やってほしくないこと（文体・禁止ワード・範囲制限）
- 守ってほしいルール（命名規則・保存場所・順番）

### 確認観点
- AIが最後にセルフチェックする基準を渡す
- 「抜け漏れがないか」「初心者目線で分かるか」「数字・事実の整合性」が定番

## YouTube台本制作の場合の例

```
【ゴール】
歴史好きな20〜40代男性向けに、「平安時代の貴族のやばい生活」をテーマにした
ゆっくり解説YouTube台本を作りたい。最終的にYMM4で動画化する。

【参照ファイル】
- 過去台本（参考文体）: @過去台本/平安系_サンプル.md
- 使うスレ: @スレ/平安_貴族_スレ.txt

【出力形式】
.md形式で。霊夢・魔理沙の掛け合い形式。7,000〜10,000文字。

【制約条件】
- 文体はフランクで読みやすく
- 近代戦争・性風俗の切り口はNG
- 史実と異なる表現は使わない

【確認観点】
- 冒頭30秒の掴みが弱くないか
- 史実に誤りがないか（固有名詞・年代）
- ふりがなが必要な漢字に括弧でふりがながついているか
```
