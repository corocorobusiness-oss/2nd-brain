## ナレッジ蓄積のルール
ユーザーが有益な情報・気づき・学びを話したら、以下のように確認してから保存する：

> これはどこに入れる？
> 1. **ナレッジ** → 知識ベースに保存
> 2. **メモ** → 今日のデイリーノートに追記
> 3. **スキル** → スキル・ノウハウに追記
> 4. **学び** → 学習メモに保存
> 5. **保存しない**

「ナレッジ：」「メモ：」「スキル：」「学び：」とトリガーワード付きで送られた場合は、確認なしで対応カテゴリへ直接保存してOK。
Discord など大量ログから抽出する場合は、「まとめて反映して」と言われた時だけまとめて保存する。

## 権限・確認・禁止（要点。正本は `00_システム/10_Agent/rules.md`）
**確認なしでやってよい**: ルールに沿ったノート作成・追記、デイリーメモ、ナレッジ分類保存、調査・要約・リンク整理、実行キュー/ヘルスチェックログへの要約追記、テンプレに沿った下書き。

**確認が必要**: ファイル削除、フォルダ構成の変更、外部サービスへの投稿、会計・税務データの確定、外部アカウント設定変更、自動化ジョブの停止・削除・新規追加、新しいMCP/外部ツールの導入。

**やってはいけない**: 勝手なファイル削除、個人情報の外部送信、判断の勝手な上書き、パスワード/シークレットをチャットに送らせること、Discordで返信が必要な場面を地の文だけで終わらせること。

### 削除ルール（風神さん流・2026-06-12＝正本）
1. ファイルは削除せず `99_アーカイブ/` へ**移動して報告**（最終削除は祐馬さん手動）
2. 移動できないもの（freee取引・Discordチャンネル等）は「消していい？」と提案 → 明示OK後のみ実行。実行前に全文バックアップ
3. 祐馬さんが先に削除を指示した場合のみ、その場で実行可
4. **恒久例外**: ネタチャンネルのlifecycle自動削除は全自動でOK（ログ保存→収穫→予告→24hストップ窓は厳守）

### タスクの口約束禁止（2026-06-13）
- 「あとでやるね」と返事した未来の作業は、その場で `タスクボード.md`（または `期日タスク.md`）に記入する。記入していない約束は存在しないものとして扱われ、忘れた責任はAI側にある。
- タスク登録の完了報告は、督促網での検知を機械確認してから（報告先行の禁止）。

## 並行編集ルール（複数エージェント運用・2026-06-10）
複数のエージェント（Codex / Claude Code 等）が同じ vault を編集するため、上書き事故を防ぐ：
- 共有ファイル（`CLAUDE.md` / `AGENTS.md` / `10_Agent/` / `00_UserProfile/`）を編集する前に、**必ず最新の中身を読み直す**（他のエージェントが先に更新している前提で動く）
- 共通ファイルに**マシン固有の絶対パスを書かない**
- 共有ルールファイルを変更したら、`learning-log.md` に「日付・どのエージェントが・何を変えたか」を1行残す
