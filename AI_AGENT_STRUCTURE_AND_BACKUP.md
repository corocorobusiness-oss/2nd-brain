# AI エージェント構成とバックアップ方針

## 1. このドキュメントの目的と全体の憲法

このドキュメントは、祐馬の事業を支えるデータ（知識・作業・スキル・AI入口・動画メディア・税の証憑）が「どこに正本としてあり」「どこへコピーされ」「どう守られているか」の全体像と方針を定める。実行手順は書かない（方針・全体像だけ）。読み手は個人事業主の祐馬と、彼を支えるAIエージェント。

### 全体の憲法（1行ルール）

> 正本は常にローカル（テキスト/コード=Mac内蔵、巨大メディア=外付けSSD）。Drive と GitHub はコピー専用で直接書かない。AIは必ずローカルの正本を編集する。

### 例外は2つだけ

1. **税の証憑（経費精算）** は当面 Drive が正本。
2. **Googleネイティブ書類（.gsheet / .gdoc）** は当面 Drive（Google）が正本。

---

## 2. 3拠点の役割

### ローカルMac（内蔵256GB・空き約84GB）
軽量正本の唯一の置き場であり、AIの作業面。正本は4つ（知識 `~/2nd-Brain-master`、作業 `~/Projects/youtube`、スキル `~/agent-skills`、AI入口 `~/agent-adapters`）で、すべてローカルgit管理。巨大メディアはここに置かない。

### 外付けSSD（931GB・空き447GB）
大容量メディア正本（完成動画・編集素材・YMM4本体・Coin Quest）の置き場であり、週次バックアップの受け皿。役割別の箱に絞って運用し、曖昧な箱（`_old` / `_backups` 等）は新設しない。Windowsからは `map-ssd-N.bat` でSMB閲覧。

### Drive ＋ GitHub（オフサイト2経路・コピー専用）
GitHub private（org=corocorobusiness-oss）が4正本を日次pushで遠隔コピーする。Drive は知識の閲覧専用ミラー（15分・片方向）と、例外正本（税証憑・売上証憑・Googleネイティブ書類）の置き場。どちらにも直接書かない。

## 5. バックアップ設計と連鎖

### 連鎖（実装確認済）

```
ローカルgit（10分毎 auto-commit）
   → GitHub private（日次push）
      → Driveミラー（15分・片方向・閲覧専用）
         → 週次スナップショット（日曜4:30, tar → ~/claude-backups）
            → 外付けSSD（マウント時 ssd-backup）
```

週次スナップショットは Drive を `claude -p` 経由で読む（macOSのTCC制約のため）＝Claude常駐に依存。

### 各データの3-2-1達成状況
- **◎達成**: 知識/作業/スキル/AI入口（28M）、税証憑（16M・復元テスト2026-06-16 PASS・729件sha256）。
- **✗ 穴**: 動画メディア（395G・SSD単独）、売上証憑（出前館PDF・どこにも無い／唯一の本当の穴・即修正可）、クラウド表計算（542点・ローカルはポインタのみ）。
- **対象外でOK**: AIランタイム（キャッシュ）、Windows VM（再構築可）。

## 6. 運用ルール

- **命名規約**: フォルダ名は役割が一語でわかる名前にする（例: `02_完成` / `売上証憑` / `claude-backups`）。
- **Driveに書かない**: Drive配下は閲覧専用ミラーまたは例外正本。AIは直接書き込まない。
- **kill しない**: Driveの不調（EPERM）でも Google Drive の killall / 再起動はしない。原因はTCCキャッシュで、リスナー再起動でのみ回復する（実証済み）。
- **`_削除待ち` は月1で空にする**: 隔離した不要物は中身確認後に削除し、箱を空に戻す。
- **消す前にコピー確認**: 削除する前に必ず別媒体にコピー済みであることを確認する。
- **symlink入口は触らない**: `~/2nd-Brain` / `~/.claude/skills` / `~/.claude/CLAUDE.md` は実体への入口（symlink）。実体と間違えて編集・削除しない。
- **正直に報告**: エラーや中断は隠さず報告する。お金や証憑に関わる破壊的操作は実行前に祐馬へ確認する。

### 正本の見分けルール
> 実体がローカルにあり `.git` があるフォルダだけが正本。Drive配下と `.gsheet` ポインタは正本でない（税証憑・gsheet原生のみ例外）。

## 7. 既知の穴と方針

- **動画メディア（395G）**: オフサイトが無い。推奨＝2台目SSD（約1万円）へ週次ミラーで媒体2を即充足＋四半期に物理搬送でオフサイト1。【要判断 / 推奨・未実施】
- **売上証憑（出前館PDF）**: どのバックアップにも無い唯一の本当の穴。推奨＝週次スナップショットのコピー対象に `売上証憑/` を追加し、git/SSDに乗せる。【今週着手OK】
- **クラウド表計算（.gsheet/.gdoc 542点）**: ローカルは実体コピー無し。推奨＝業務クリティカルな数点だけ月次/四半期に Google Takeout で `.xlsx/.docx` 化し `~/2nd-Brain-master/_証憑取込/gsheet-export/` に落としてgit化。全件は過剰。対象選定は祐馬。【要判断 / 推奨】
- **税証憑の正本=Drive固定**: 推奨＝当面Drive正本のまま例外運用（規程改定・税務確認は急がない。既に実3コピーで復元PASS済み）。【要判断 / 推奨】
- **Codex非依存化（週次snapshotのClaude依存切り）**: 推奨＝後回し。当面Claude常駐前提。Codex移行を決めた時に対応。【要判断 / 推奨】

> 補足: `agent-run`（`~/agent-adapters/bin/agent-run`）はClaude↔Codex切替の唯一の継ぎ目として存在するが、どのスクリプトもまだ使っておらず全部 `claude` を直叩き、codex側も未配線＝今はCodex切替は実質不可。
