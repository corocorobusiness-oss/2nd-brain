## 運用（ハイブリッド体制・2026-06-10 v2）

### 週次 — 兵士（スクリプト・全自動／エージェント不要）
毎週月曜9:30に `scripts/kicho.py` がlaunchdから自動実行される：
freee APIから新規明細取得（認証はfreee-mcpと共有・自動リフレッシュ）→ 表記ゆれ正規化・重複排除
→ 仕訳生成 → 検証 → 収支管理.mdのマーカー区間を更新 → デイリーノートに「📒 週次記帳」レポート。
- 未知の明細パターンは**自動計上せず**「要確認」として停止 → エージェントに「要確認を解消して」と言う
- 検証NGは自動ロールバック。ログ: `logs/kicho.log`
- **ON**: `scripts/kicho_on.sh` ／ **OFF**: `scripts/kicho_off.sh`（切り替えたら台帳も更新すること）

### 月次 — 参謀（AIエージェント・月初）
エージェント（6月=Claude Code、7月〜=Codex）に「**帳簿監査して**」と言う：
1. 要確認・保留明細の解消状況チェック
2. 月次レポート生成（`02_経営/月次レポート/YYYY-MM.md`・目標と計画.mdとの対比つき）
3. 異常チェック（経費率の急変・二重計上の疑い・残高整合・仕訳ルールの見直し）
4. freee再同意の残日数確認（90日ごと・次回目安: 2026年9月上旬）

### 手動（いつでも）
- 「**記帳して**」→ エージェントが `scripts/kicho.py` を実行（週次と同じ処理・1ソース）
- 現金払いの経費はレシートの金額・日付・内容を伝える →「(借)○○費 (貸)事業主借」で計上
- freeeが使えない場合のフォールバック: GMOあおぞらの入出金CSVを `inbox/` に置いて「記帳して」
