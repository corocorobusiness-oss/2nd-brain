## 現行運用（2026-06-27整理）

### 正本
- 会計正本: freee（試算表PL/BS・取引一覧・申告画面）
- 証憑正本: Drive/証憑保管側
- 二重登録防止: `~/.claude/scripts/freee_registered_txns.json`
  - 金額集計の正本ではなく、freee登録済み判定用の補助台帳
  - 欠損・不整合・読み取り不能時は登録せず要確認で止める

### 週次・月次
- freee会計データへ書き込める自動化は原則 `weekly_accounting` の1系統だけ
- 月次は `monthly_accounting` と `freee_export/` スナップショットでfreee正本を点検する
- 経理系ジョブは二重計上防止のため、Watchtowerの自己修復対象に入れない

### kichoの扱い
- 現行 `scripts/kicho.py` は、freee明細を読むだけでなく、旧CSV・仕訳帳・収支管理・日誌を書き換える
- そのため、記帳係としての継続運用は停止済み
- 実機状態: 2026-06-27に `launchctl print` で未ロード、`print-disabled` でdisabledを確認済み。通常実行の `python3 scripts/kicho.py` は安全停止メッセージのみで終了する
- 後継を作る場合は、最初から読み取り専用の監査レポートとして小さく作る
  - 許可: freee明細、証憑、補助台帳、freee_exportの差分確認
  - 禁止: POST/PUT/PATCH/DELETE、取引作成、消込、紐付け変更、CSV/収支管理/日誌の自動更新
- 「記帳して」で `scripts/kicho.py` を呼ぶ導線は使わない

## 保存方針

- 旧CSV仕訳帳・総勘定元帳・レポートは参考アーカイブとして削除しない
- 申告数字の正本はfreeeに置く
- 証憑の正本はDrive/証憑保管側に置く。ただし電子帳簿保存法要件の充足は、検索性・訂正削除管理・保存状態を申告前に確認する
- freee内に残るレシート画像・証憑は、必要に応じてDrive側の証憑保管へ退避する

## freeeとの関係

- freeeが会計正本。2027年2月に1ヶ月だけfreee課金し、申告機能でe-Tax送信する方針
- freee APIは週次/月次の点検と登録に使う。ただしfreee会計データへの書き込みは1系統に絞る
- freee無料プランでは作成30日超の取引がAPIで不可視になることがあるため、`freee_registered_txns.json` を二重登録防止キー台帳として使う
- 旧CSV帳簿との差分が見つかった場合、CSV側を正として上書きしない。freee画面・試算表・証憑で確認してから扱いを決める
