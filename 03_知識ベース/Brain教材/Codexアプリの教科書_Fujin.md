## 第4章: Codexとは — ChatGPTの次の選択肢

### Codexの本質
- **ChatGPTは「会話するAI」、Codexは「作業するAI」**
- ChatGPTは答えを出すまで。Codexは「実行」（ファイル編集・生成・検証・差分作成）まで担う
- 役割分担: ChatGPT＝考える・整理する（方針/文章/構成/壁打ち）、Codex＝触る・直す・試す

### Codexには4種類ある
| 種類 | 概要 | 向いている人 |
|------|------|-------------|
| **アプリ版（推奨）** | PCにインストールするアプリ。AIエージェント機能をGUI操作で使える | 非エンジニア含む全員 |
| CLI | ターミナルで直接操作。細かく制御可能 | 開発者向け |
| IDE拡張 | VSCode等エディタ内で動作。CLIとアプリの中間 | IDEを使い慣れた人 |
| Web版 | クラウドバックグラウンドでタスク実行 | 軽い作業・コードレビュー時 |

### アプリ版が「一択」の理由
- **ChatGPTサブスク（月3,000円）内で追加費用ゼロで動く**（API課金なし）
- ターミナル不要、インストールはボタン1つ
- AGENTS.md / Agent Skills / Plugins / Automations / MCP / Browser Use / Computer Use まで全機能搭載
- GPT-5.5など最新モデルをそのまま利用可能
- Claude CodeやCodex CLIでできることのほぼ全てがアプリ版で可能

### AIエージェントとしての3要件
1. **環境を見られる** — フォルダ・ファイルの中身を直接確認できる
2. **作業を実行できる** — 編集・生成・検証をその場で進められる
3. **結果を直せる** — 確認しながら修正まで続けられる

### 作業を任せる4ステップ
1. ゴールを伝える（何を完成させたいか明確に）
2. 材料を渡す（フォルダ・資料・条件をセットで渡す）
3. 実行させる（AIに進めてもらう）
4. 確認して直す（結果を見て改善点を反映）

**重要**: 丸投げではなく途中確認しながら進める。完全自動化は段階的に信頼を積んでから。

### フォルダ参照の価値
- フォルダを指定すれば中のファイルを直接見て作業する → コピペ不要
- ナレッジを渡すほど精度が上がる（AIっぽい汎用出力を脱却できる）
- **「AI活用はプロンプトよりナレッジが先」**（著者の核心主張）

### 定期実行・拡張
- Automationsで時間指定の自動実行が可能
- **Agent Skills**はツール間で移植できる資産（Claude CodeのSkillをCodexアプリに持ち込める）

## 第5章: インストールと初期設定

### インストール手順
1. ChatGPTウェブ画面の左上メニュー「Codex」から、またはhttps://openai.com/ja-JP/codex/ から直接DL
2. Mac/Windows版インストーラを実行（普通のアプリインストールと同じ）
3. 起動後「ChatGPTで続行」でログイン（APIキー不要）
4. ブラウザでChatGPTアカウント認証 → Codex画面に到達

### モデル設定（重要）
- **必ずGPT-5.5（または最新モデル）を選ぶ**
- 旧モデル（GPT-5.4等）だとSkill読み込みやAGENTS.md活用精度が大幅低下

### Intelligenceレベル（知性レベル）
- **普段: 高**
- **複雑な作業: 非常に高い**（トークン消費増加、サブスク枠が早く切れる）

### 回答速度
- **基本: 標準**（「高速」は使用量約1.5倍増）

### 使用制限が0%になったとき
- 選択肢1: プランを上げる（推奨）
- 選択肢2: クレジット購入（従量課金）
- 「自動チャージ」はオンにすると課金感覚が掴めなくなるため、慣れるまでオフ推奨

---

### フォルダ設計（最重要の初期設定）

**やってはいけないこと**: PC全体をCodexに見せる
- 機密情報が見える
- 成果物の保存先が分からなくなる
- 勝手にファイルが作られたり消されるリスク

**推奨の始め方（5ステップ）**:
1. Codex専用フォルダを1つ作る（場所はローカルでもGoogle Driveでも可）
2. 自分のナレッジ素材を少しずつ入れる（ブログ記事・Xポスト・メモ・計画書等）
3. 音声・動画よりテキスト化（.txt/.md）したものを優先
4. **削除権限は基本オフ**（Codexによる削除は移動のみにとどめ、最終削除は手動で）
5. 重要ファイルはバックアップを取る

**Google Drive連携のメリット**:
- 複数デバイス（Mac↔Windows）で同じフォルダを共有
- 自動クラウド同期・バックアップ
- **注意**: クラウド同期中に削除すると双方から消える（Google Driveはゴミ箱から復元可）

## 第6章: 基本操作と入力効率を上げる方法

### 画面構成・主要操作

| 記号 | 機能 | 用途 |
|------|------|------|
| **@（アットマーク）** | プラグイン・ファイル指定 | 特定ファイルやサービスを狙い撃ちで参照 |
| **$（ドルマーク）** | Agent Skills呼び出し | インストール済みSkillをリスト表示して起動 |
| **/**（スラッシュ） | スラッシュコマンド | コードレビュー・ステータス・モデル選択等 |

- **プランモード**: 大規模・複雑な依頼で最初に計画を立ててから実行。普段は不要。
- **権限選択**: デフォルト権限（サンドボックス内）→ 自動レビュー → フルアクセスの順で段階的に上げる

### プロジェクト機能（必ず使う）
- 左サイドバーの「プロジェクト」欄にフォルダを登録
- 推奨: 「既存のフォルダを使用」で自分で場所を決めたフォルダを指定
- **データが蓄積→AIが関連ファイルを読み込む→出力が「あなた仕様」に寄っていく**
- プロジェクト未使用の単発依頼では毎回ゼロから前提説明が必要で非効率

### プラグイン（左メニュー「プラグインタブ」）
インストールはAPIキー不要。「Codexについて」ボタンを押すだけ。

**著者おすすめプラグイン構成**:
- コンピュータ操作系: コンピュータ操作・Chrome拡張
- スプレッドシート/プレゼン: Google Sheets・Google Slides
- 開発系: GitHub
- メール/カレンダー: Gmail・Google Calendar
- タスク/メモ: Notion・Slack（業務使用者向け）
- デザイン: Canva・Figma・HyperFrame

**原則**: 自分の業務動線にあるものだけを繋ぐ

### Skillsタブ（プラグインタブ内）
- 純正Skill（PDF作成・Playwright等）・自作Skill・配布Skillを一元管理
- 新規Skill作成: 「Skill Creator」プロンプトが表示 → 日本語で「○○のスキルを作りたい」と書くだけ

### オートメーション（左メニュー）
**作成手順**:
1. 「新規自動化」をクリック
2. タイトルを入力
3. プロンプトを書く（またはテンプレートから選ぶ）
4. 実行スケジュール（毎週/毎日/指定時刻）を設定
5. **プロジェクトを選択**（必須。未選択だと作成ボタンが押せない）
6. モデル・Intelligenceを設定
7. 作成

- 「今すぐ実行」で即時起動テスト可能
- 実行履歴は右側パネルで確認できる

---

### プロンプト（依頼文）の型

**5つの要素（この順番で書く）**:

| 要素 | 内容 | 例 |
|------|------|---|
| **ゴール** | 何を・誰向けに・どこで使うか | 「Codexをこれから使う人向けに10枚のスライドを作る」 |
| **参照ファイル** | 過去ブログ・Xポスト・メモ等のナレッジ | ファイル名指定 or プロジェクトフォルダ参照 |
| **出力形式** | ファイル形式・枚数・比率等 | .md / .pptx / .pdf、スライドなら枚数・16:9等 |
| **制約条件** | やってほしくないこと・守るルール | 範囲・文体・デザイン・禁止事項 |
| **確認観点** | 最後のセルフレビュー基準 | 抜け漏れ/初心者目線/数字整合性 |

**テンプレート**:
```
【ゴール】
〇〇向けに、〇〇を作りたい。最終的に〇〇で使う。

【参照ファイル】
- 過去ブログ：〇〇.md
- 構成メモ：〇〇.md

【出力形式】
.md形式で。見出し2階層まで。

【制約条件】
- 文体は〇〇
- デザインは〇〇
- 〇〇は使わない

【確認観点】
最後に以下をセルフレビューしてから返す：
- 抜け漏れ
- 初心者目線
- 数字の整合性
```

**出力形式のおすすめ**:
- Markdown（.md）が最強 → 見出し/太字/箇条書きが構造として残り、noteにも貼れる
- スライドは.pptx、配布物は.pdf

### 音声入力の設定（重要機能）

**概要**: Codexアプリをインストールするだけで、**PC全体（Slack・メール・Notion・ブラウザ等どこでも）**で使える音声入力が付いてくる。デフォルトはオフのため設定が必要。

**設定手順**:
1. 設定 → 一般タブ → 下にスクロール → 音声入力セクション
2. ホットキーを「トグル切り替え」に設定（推奨）
3. 設定ボタン → キーボード受信画面 → システム設定でCodexにキーボードアクセス権限付与
4. 使いたいキーを押して登録（著者は⌘+Q / Windows: Ctrl+Q）
5. マイク権限・キーボード権限の両方をCodexに許可

**ホットキーの種類**:
- 押している間だけ録音: 短文中心の人向け
- **トグル切り替え（推奨）**: 押すと開始、もう一回押すと終了。長文に最適

**認識辞書（神機能）**:
- 固有名詞・英語ツール名を登録すると誤変換を防止
- 著者登録例: Codex、Fujin、Claude Code、ChatGPT
- 登録場所: 同じ音声入力セクション内

**使いどころ**: AIへのプロンプト（文字入力比3倍速）、メール返信、Slackメッセージ、ブログ下書き、検索クエリ

## 第7章: AGENTS.md — Codexを自分好みのAIエージェントにする方法

### AGENTS.mdとは
**Codexへの取扱説明書**（業務マニュアル）。作業前にCodexが読み込み、前提・ルール・品質基準を頭に入れた状態で依頼を処理する。

例え: 新入社員への業務マニュアル、外注ライターへの執筆ガイドライン

**3つの役割**:
1. 前提を共有する（プロジェクトの目的・使う技術・関係者情報）
2. 作業方針を決める（進め方・確認の取り方・保存先）
3. 品質を安定させる（使う言葉・禁止事項・最終チェック観点）

**あり/なしの差**:
- なし: 毎回前提説明が必要、言い忘れが品質に影響、同じ依頼でも出力がブレる
- **あり**: 依頼が「今回の目的」だけで済む、品質バラつきが減る

---

### AGENTS.mdの2階層

| 種類 | 範囲 | 書く内容 |
|------|------|---------|
| **グローバル指定** | Codexアプリ全体（全プロジェクト共通） | 文体・言語・ファイル形式等の個人ルール |
| **プロジェクト指定** | そのフォルダ内のみ | プロジェクト固有のルール（テーマ・出力形式等） |

**Codexの読み込みフロー**:
プロジェクトフォルダを開く → AGENTS.md確認 → 方針を頭に入れる → 依頼文を読む → 作業

---

### AGENTS.mdに書く項目

- プロジェクトの概要（何を作っているか、誰向けか）
- 作業手順（依頼を受けてからの動き方）
- 確認方法（どのタイミングで誰の確認を取るか）
- 出力形式（保存先、ファイル名のルール）
- 禁止事項（やってほしくないこと）
- 品質基準（最終的に満たすべき条件）

**書き方のコツ**:
- 短いルールから書き始める
- 箇条書きで整理（長文は避ける、人間が一目で理解できる粒度）
- **最初から完璧を目指さない。使いながらアップデートする**

**特に効く場面**: 文書修正（文体・トーンが揃う）、資料作成（デザイン・構成が前回と揃う）、運用作業（保存先・命名規則が毎回同じになる）

---

### 用途別AGENTS.mdの考え方

作業ごとにAIに守らせたいルールが違うため、**用途ごとに専用AGENTS.mdを作る**のが効果的。

**著者が使っている4用途**:

#### 1. Brain記事制作用
- 読者の悩み（誰が何に困っているかを先に固定）
- 構成の型（導入 → 問題提起 → 解決策 → 販売導線）
- 文体ルール（煽りすぎず、実践感のある文章）

#### 2. 講座制作用
- 到達目標（受講後に何ができるようになるか）
- 章立て（基礎 → 実演 → 課題の順）
- スライド粒度（**1枚1メッセージで詰め込みを避ける**）

#### 3. リサーチ用
- 調査範囲（期間・地域・対象テーマを明示）
- 情報源（一次情報と信頼できる記事を優先。「直近6ヶ月以内」等の期間指定）
- 整理形式（**結論 → 根拠 → 活用案**に分けて出す）

#### 4. 商品設計壁打ち用
- 顧客像（誰のどんな状況を扱うか）
- 価値仮説（何を解決し、なぜ選ばれるか）
- 販売設計（**価格・導線・オファーまでセットで考える**）

**使い分け**: まずは2つから。「記事制作用」と「壁打ち用」あたりから始めると効果的。

## 第8章: Agent Skills・プラグイン・オートメーション完全攻略

### 三本柱の役割整理

| 柱 | 役割 | 一言 |
|----|------|------|
| **Skills** | どう進めるか（ルール） | 定型作業の作業レシピ |
| **Plugins** | どこで作るか（接続） | 作業範囲を外部ツールまで拡張 |
| **Automations** | いつ動かすか（タイミング） | 定期自動実行の仕組み |

**「どれが一番大事？」→「3つとも揃って初めて意味がある」**

---

### Agent Skills（スキル）

**一言**: 毎回同じ品質で進めるための作業レシピ

**Skillに固定する3要素**:
- 手順を固定（何を先に読むか、どの順で進めるか）
- 判断を固定（優先順位、禁止事項）
- 出力を固定（納品形式、確認観点）

**著者の使い方例**:
- 「音声データ → スライド資料」
- 「音声データ → ブログ記事」
- **73個のSkillを運用中**（ほぼ全部自作）

**向いている作業**: 繰り返すほど価値が出る作業
- 講座スライド化（構成から.pptxまで同じ型で）
- 記事から投稿文へ変換（文体とフォーマットを揃える）
- 提案書の下書き（必要項目と順番を固定）

**向いていない作業**: 一回限りの作業、雑談、毎回まったく違う相談

**$マーク呼び出し**: 入力欄でShift+4（$）→ インストール済みSkillリストが出る → 名指しで起動

---

### プラグイン（Plugins）

**一言**: Codexの作業範囲を外部ツールまで広げる

**できること**:
- 探す: 外部ファイルや情報を読む
- 作る: 外部サービス上に成果物を作る
- 更新する: 既存のデータや資料を直す

**プラグインなしの状態**: チャット内だけで完結。コピペ作業が発生。
**プラグインありの状態**: 実務の場所（Drive・GitHub・Notion等）に直接届く。コピペ作業が消える。

**インストール方法**: APIキー不要。プラグインタブ → 「Codexについて」ボタンを押すだけ。

---

### オートメーション（Automations）

**一言**: 人が毎回頼まなくても決まった仕事を動かす仕組み

**ChatGPTとの違い**: ChatGPTは人間が起点にならないと動かない。オートメーションは自動起動する。

**著者の活用例**: 「毎朝の市場ニュースまとめ」（ログインしなくても勝手に動いてまとめてくれる）

---

### 三本柱を組み合わせた業務フロー例

「毎週のコンテンツ制作を自動化する」:
1. Automation: 毎週月曜の朝に起動
2. Plugin: Driveのメモを読み込む
3. Skill: 投稿文の型に変換する
4. Plugin: 下書きや資料へ保存する

→ 月曜朝にDriveを開くだけで今週分の投稿下書きが揃っている状態

## 第10章: Google Drive連携とCodex活用事例

### Google Drive連携の必要性

**ローカルフォルダのみの課題**:
- そのPC内でしかファイルが触れない
- 複数デバイス（Mac↔Windows）間で同期できない
- バックアップが弱い

**Google Drive連携のメリット**:
- どのデバイスからも同じフォルダにアクセス
- Codexで作った成果物が自動で全デバイスに同期
- 削除があってもゴミ箱から復元可能

---

### Google Drive for desktop の設定手順

**やること**: Google Drive for desktop（パソコン版Google Drive）をインストールするだけ

1. https://support.google.com/a/users/answer/13022292?hl=ja からDL
2. Mac/Windows用インストーラを実行
3. 初回起動時にGoogleアカウントでログイン
4. アクセス許可を承認

**結果**:
- Mac: Finder → サイドバーにGoogle Drive追加
- Windows: エクスプローラーにGoogle Drive追加
- ローカルフォルダと同じ感覚でGoogle Drive内のファイルを操作可能

**CodexでGoogle Driveフォルダを使う**:
1. Codexアプリで「フォルダを選択」
2. ファイラーのGoogle Drive配下に移動
3. 使いたいフォルダ（例: Obsidian）を選択

**注意事項**:
- ローカルで削除したファイルはクラウドからも消える（Google Driveはゴミ箱に一定期間保持）
- 削除通知の設定推奨

**この神アプリはCodex以外のAIツールでも使える**:
- Claude Code
- Antigravity
- 各種ローカル系AIツール

## 著者からの核心メッセージ（重要Tips集）

### AI活用の本質論
- **「AI活用はプロンプトよりナレッジが先」** — 過去のアウトプットを渡すからこそ「あなたっぽい」出力になる
- ナレッジを渡さない = AIっぽい誰でも作れる出力しか出てこない
- Codex用フォルダを育てていくほど「あなた専用のAIエージェント」に育つ
- **「覚えておく場所」がフォルダの正体**（ChatGPTは覚える場所がないから毎回ゼロ）

### プロンプト設計論
- 良い依頼文 = AIへの作業設計書（「いい感じにやって」はNG）
- 5要素（ゴール・参照ファイル・出力形式・制約条件・確認観点）を埋める
- 確認観点を渡すとAIがセルフレビューしてから返す（GPT-5系はここの精度が高い）
- **「ナレッジは置きっぱなし、依頼文は型を埋めるだけ」が理想状態**

### 自動化の原則
- 完全自動化は実装難易度が高い。最初は確認ポイントを置きながら進め、信頼できる流れができてから自動化に寄せる
- 3つを組み合わせて初めて「業務フロー」になる（Skills単体・Plugins単体では不完全）

### ファイル安全運用
- Codexには削除はやらせない（整理なら「移動」のみ。最終削除は手動）
- 重要ファイルは事前にバックアップ
- 利便性と安全性はトレードオフ。ちょうどいいバランスを自分で設計する

### Skills資産化の重要性
- Agent Skillsは**ツール間で移植できる資産**（プラットフォームを乗り換えても腐らない）
- Claude Codeで作ったSkillをCodexアプリに持ってこれる
- 繰り返す作業だけSkill化する（一回限りの作業はSkill化不要）

## 付録: 設定・操作のクイックリファレンス

### キーボードショートカット
- **ポップアップ呼び出し**: 任意ホットキー（著者: ⌘+W）→ どの画面からでもCodex呼び出し
- **音声入力**: 任意ホットキー（著者: ⌘+Q / Ctrl+Q）→ PC全体で音声入力

### 入力欄の特殊記号
- `@` → プラグイン・ファイル指定
- `$`（Shift+4）→ Agent Skills呼び出し
- `/` → スラッシュコマンド（コードレビュー・ペット機能・モデル選択等）

### プロジェクト作成
- 左サイドバー「プロジェクト」の + → 「既存のフォルダを使用」（推奨）

### オートメーション作成
- 左メニュー「自動化」→「新規自動化」→ タイトル・プロンプト・スケジュール・プロジェクト選択（必須）→「作成」

### Skill作成
- Skillsタブ → Skill作成 → 「○○のスキルを作りたい」と日本語で書く → 案内に沿って入力

### 設定画面アクセス
- アプリ左下から設定画面へ
- 外観メニュー: テーマ切り替え・Pet機能設定
- 一般タブ: 音声入力・ホットキー・フォローアップ動作等
