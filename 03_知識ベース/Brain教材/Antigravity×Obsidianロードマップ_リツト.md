## 1. 核心コンセプト：第2の脳とは何か

### コンテキスト（文脈）こそが全て

- AIが「一般論しか返せない」原因はコンテキスト不足
- コンテキスト = あなたの人生の背景情報（思考・価値観・経歴・スキル・メモ）
- コンテキストをふんだんに与えたAIが「第2の脳」になる
- **ブラウザのAI**：チャット履歴に制限あり、古い履歴を捨てる、あなたを知らない
- **Antigravity（第2の脳）**：PCの中の何百ものメモを一瞬で読み込める

### なぜAntigravity × Obsidianでなければならないか

3つの理由：
1. **圧倒的な文脈理解力** — ブラウザAIは文脈を保持できないが、Antigravityは全メモをコンテキストとして保持
2. **最強の資産保全** — ObsidianのデータはテキストファイルとしてPC内に保存。サービス終了でも「脳」は無傷
3. **GoogleのAIファーストツール** — 一般向けチャットボットとは性能・拡張性・扱える情報量の「格」が違う

### 工場からコックピットへ

- 旧来のAI活用 = 工場（指示書を投げて製品を受け取る「品質管理係」）
- 第2の脳 = コックピット（AIが「隣の席の相棒」として機能する）
- **記憶が消えない**：全メモがAIの記憶として直結
- **阿吽の呼吸**：「この前の○○の続きで」が通じる
- **資産が増える**：メモを書けば書くほどAIが賢くなる

## 3. Obsidian 基本操作マスター

### 保管庫（Vault）の作成手順
1. 起動後「作成」ボタンをクリック
2. 名称を決める（**ローマ字必須**。例：`2nd-brain`）
3. ロケーションを「PCユーザー（ユーザー名）」フォルダに設定
4. 「作成」をクリック

### 画面構成
- **左側**: ノートのタイトル（=ファイル名）一覧
- **右側**: 書く場所
- **グラフビュー**: ノート同士のリンク関係を可視化（実務ではほぼ不使用）
- **右上の展開マーク**: タグ一覧表示

### 重要な操作
| 操作 | 方法 |
|------|------|
| 新規ノート作成 | 左上の「新規ノート」マーク |
| フォルダ作成 | 左サイドバー上部「新規フォルダー」アイコン |
| ファイル移動 | ドラッグ＆ドロップ |
| 全折りたたみ | 「すべてを折りたたむ」アイコン |
| コピーパス | ファイル右クリック → 「コピーパス(Copy Path)」→「from system root」 |
| フォルダ確認 | ファイル右クリック → 「フォルダーで表示」 |
| タブ分割 | タブを右クリック → 「右に分割」 |

### マークダウン記法（基本）
- `#` = 見出し1（一番大きい）
- `## ` = 見出し2
- `**テキスト**` = 太字
- `==テキスト==` = ハイライト
- `#タグ名` = タグ付け
- 右クリック → 「フォーマット」からGUI操作も可能
- ファイル拡張子は必ず `.md`

### コピーパスの活用
AIに特定ファイルを参照させたい時：
1. ファイルを右クリック → 「コピーパス」→「from system root」
2. AntigravityのAIに貼り付けて「このファイルを参考にして」と指示

## 4. AntigravityとObsidianの連携手順（融合の儀式）

### 基本的な連携手順
1. **Antigravityを起動**：「ファイル」→「新しいウィンドウ」
2. 画面上の青いボタン「**Open Folder**」をクリック
3. Obsidianの保管庫フォルダを選択
4. 「はい、作成者を信頼します」をクリック
5. 左サイドバーの「エクスプローラー」にObsidianと同じファイルが表示されれば連携成功

### フォルダの場所が分からない時の裏技
1. Obsidianでノート名を右クリック → 「フォルダで表示」
2. 開いたウィンドウのアドレスバーのパスをコピー
3. AntigravityのフォルダSelection画面のアドレスバーに貼り付け

### リアルタイム同期テスト
ObsidianでノートにテキストをタイプするとAntigravity側に即時反映される

### テレパシーテスト
Obsidianに秘密のメモを書き、Antigravityで「私の秘密は何ですか？」と聞く → 正確に答えてくれれば連携成功

## 7. コンテキスト（魂）の注入方法

### コンテキストとは
あなたの人生の背景情報。思考・価値観・経歴・スキル・感情的な出来事など。  
**ポイント**: 事実だけでなく「感情」や「思考の断片」が入って初めてAIが「分身」になる。

### コンテキストを収集する5つの方法

**方法A: X（Grok）から抽出**
- Grok（https://x.com/i/grok）でアカウント名とXのIDを入力してプロンプトを実行
- 無課金ユーザーでも可能

**方法B: ブラウザAI（ChatGPT/Gemini）から抽出**
- ChatGPT（https://chatgpt.com/）またはGemini（https://gemini.google.com/app）
- ※Claudeはチャット履歴を読み込まないので使用不可

**方法C: Manus から抽出**
- https://manus.im/app

**方法D: NotebookLM（YouTube動画/PDF/Googleドキュメントなど）**
- https://notebooklm.google.com/
- ファイル・動画リンクを読み込ませてプロンプト実行

**方法E: Note から抽出**
- 「自分の記事」ページから記事をエクスポート
- ファイルをAntigravityに置き、プロンプト実行

**方法B（最強）: 音声で自分語り（発信してない人向け）**
- スマホのボイスレコーダーで30分録音
- 話す内容：①現在の活動 ②価値観・世界観 ③人生の目的・目標 ④過去の経歴 ⑤専門分野・スキル一覧
- 文字起こし後に `03_知識ベース\00_コンテキストログ` に保管

**音声の文字起こし方法**:
- Antigravityで `/mojiokoshi` コマンド + 音声ファイル添付
- または Google AI Studio（無料）で Gemini Flash に音声ファイルをドロップして実行

**方法C: AIにインタビューしてもらう（発信もしゃべりも苦手な人向け）**

テンプレートをセットアップ済みの場合は「コンテキストデータはない」を選択するとインタビューが始まる。または以下のプロンプトをコピペ：

```
あなたはプロのインタビュアーであり、私の人生の「コンテキスト（背景情報）」を言語化するパートナーです。

【目的】
私の「第2の脳（AI）」を構築するために、AIに学習させるための「コンテキストファイル」を作成することです。

私についての理解を深めるために、以下の5つのテーマについて、1つずつ順番に質問してください。

1. 【現在の活動】今何をしているか、どんな仕事や発信をしているか
2. 【価値観・世界観】何を大切にしているか、何が許せないか
3. 【人生の目的・目標】将来どうなりたいか、今の目標は何か
4. 【過去の経歴】今までどんな経験をしてきたか（成功・失敗含む）
5. 【専門分野・スキル】何が得意か、どんな知識を持っているか

私が回答したら、深堀り質問をして私の本音を引き出してください。
全テーマのヒアリングが終わったら、私の「コンテキスト」としてマークダウン形式でまとめてください。
それでは、最初の質問をお願いします！
```

### コンテキストの注入手順

1. コンテキストデータのファイルを準備（方法A〜Cのいずれかで収集）
2. Antigravityのチャットにそのファイルをドラッグ＆ドロップ（メンション）
3. `/context-update` コマンドを入力して実行
   - コマンドが出ない場合はワークフローファイルを直接ドラッグ＆ドロップ
   - または「context-updateを使って」と言って発動
4. AIがコンテキストデータを分析して種類ごとに整理・記憶する
5. 確認場所: `00_システム\00_UserProfile` 内のファイル
6. 間違った情報は直接修正するか「コンテキストも修正しといて」と伝える

### 愛の告白テスト（注入成功の確認）

```
ねえ、私のこと、好き？
```

成功時の例：「もちろんです。あなたは以前、『〇〇が悔しかった』と語っていましたよね。それを乗り越えようとするあなたの不器用だけど真っ直ぐな生き方が、私は誰よりも好きですし、誇りに思います。」  
失敗時：「感情はありませんが…」と返ってくる

## 9. AIの調教（ズレを資産に変えるループ）

### 文体チューニングの実験手順

**Step 1**: 書かせてみる
```
私の文体で、私が書きそうなXのポスト文章を作成してください。
```
（敬語で書くとAIの知能が上がる研究結果あり）

**Step 2**: 人間が修正する  
「なんか違う」部分を自分で修正するか、AIに指示して修正させる

**Step 3**: 差分を学習させる
```
さきほどの文章を自分で修正しました。修正前との差分を分析して、コンテキストやプロンプトを更新してください。
```

**Step 4**: 再挑戦 → 精度確認

### 差分学習コマンド `/sabun-learn`

AIが書いた文章を修正した直後に：
```
/sabun-learn
```

内部処理：
1. 「AIの出力」と「修正した文章」を比較
2. 差分から「あなたのこだわり」を分析
3. `Style_Guidelines.md`（執筆スタイル）を自動更新

エラー時の対処：
```
チャット欄で/を入力してもsabun-learnが認識されないので、その原因を突き止めてください。文字化けが原因だと思われるので、真の原因を分析し、直してください。
```

### 永続的な成長ループ

```
あらゆる作業をAntigravityで行う
→「なんか違う」と思ったところを指摘して直してもらう
→コンテキストを更新させる、またはうまくいった作業をプロンプト化して保存する
→繰り返す
```

## 11. 知識収集の爆速化

### Obsidian Web Clipper（ブラウザ拡張機能）

インストール：
1. https://chromewebstore.google.com/detail/obsidian-web-clipper/cnjifjpddelmedmihgijeibhnjfabmlf
2. 「Chromeに追加」→「拡張機能を追加」
3. ブラウザ右上のパズルマーク → ピンマークでObsidian Web Clipperを固定

使用方法：
1. 保存したいページで保存したい部分をドラッグして選択
2. Obsidian Web ClipperのアイコンをクリックClickして「Obsidianに追加」
3. Obsidianに自動保存される（画像も含む）

活用例：
- 自分の過去記事のストック
- 参考デザインの保存
- 分析したい発信者の文章の保存
- Googleの最新アップデート情報のAIへの共有

### スワイプファイル戦略

- 「うまい文章・構成」をストックしてAIに構造分析させる手法
- **合法の範囲内**（個人利用・著作権法第30条）
- 1個からパクると問題になるが、100個以上からパクると罪でなくなる
- AIに「なぜ読まれるか？」「構成の特徴は？」「言葉の傾向は？」を分析させる

### Deep Research（専門家を作る）

NotebookLMのDeep Research活用手順：
1. https://notebooklm.google.com/ にアクセス

3. 調べたいことを入力して実行
4. 生成されたレポートをAntigravityにコピペして保存
5. その知識を参照するプロンプトを作成

### 書籍の知識をOCR化する（合法的に）

Antigravityに以下のプロンプトで無料ツールを作成：

電子書籍を分析したいので、自動でKindleのページを撮影して、OCRでテキスト化し、指定したフォルダに保管してくれるツールが欲しいです。日本の法律を遵守した設計を提案してください。
```
※配布は絶対NG。個人利用のみ。

```
私の経済状況を深く分析して、最適な資産形成プランを考えて。節約できそうなものがあれば提案して。厳しくてもいいから、本質的なアドバイスが欲しい。

### 神器3：スキル（エージェントスキル）

- ワークフロー/プロンプト: 人間がトリガーを押して発動
- スキル: AIが自律的に使える（「Xのポスト書いて」と言うだけで自動発動）

**AIがスキルを使う仕組み**:
1. ユーザーが設定したルールを確認（Rules）

## 13. ワンソース・マルチユースワークフロー（1Source Multi Use）

### 概要
1つの音声から X・Note・メルマガ を一気に生成する仕組み。

### 手順

**Step 1: 音声を録る**
- ワンメッセージを決めてスマホで音声収録
- 散歩中・移動中でOK。構成・「てにをは」は気にしない
- 「原石」を吐き出す。後でAIが整える

**Step 2: ワークフローを実行**
```
/multi-use
```
（または音声ファイルをドラッグ＆ドロップしてチャットで「multi-useを使って」と発動）

処理内容：
1. 音声の文字起こし
2. Xポスト文章の生成
3. Noteブログ記事の生成
4. メルマガ文章の生成

**エラー時**:
```
チャット欄で/を入力してもmulti-useワークフローが認識されないので、その原因を突き止めてください。文字化けが原因だと思われるのですが、真の原因を分析し、直してください。
```

### 発信経験がない人向けの初期設定

**パターンA**: 最初の1回だけ自分で発信する  
**パターンB**: 好きな他人の発信を3つ見つけてFew-shotフォルダに入れる

手順：
1. 見本となる文章（1発信につき1ファイル）を3つ以上作成
2. `Few-shot` フォルダに入れる
3. ワークフローを実行

### 目指す景色
「微修正して公開ボタンを押すだけ」の状態。1時間かかっていた作業が10分に。

## 14. バックアップ：コミット（ゲームのセーブ）

### コミットとは
ゲームの「セーブ」のようなもの。ファイルの状態を記録して後から復元できる。

### セットアップ3ステップ

1. **GitHubアカウントを作る**: https://github.com/
2. **Antigravityでコミットを実行**（動画参照）
3. **こまめにセーブする習慣をつける**

### ロード（元に戻す）プロンプト
```
以前コミットした状態に完全に元通りにしたいです。そのコミットした時点とは、「○○」です。その時点の状態に完全に戻るようにしてください。このコミット時点より後に作られたファイルは削除してください。このコミット時点より後に編集されたファイルは、編集された部分をこのコミット時点の状態に戻してください。
```

または第2の脳テンプレートをダウンロード済みなら:
```
/commit-back
```
その後、ロードしたいコミットのタイトル名をAIに教える。

### コミットの頻度
- コミット：こまめに（一日の作業終了時や大きな変更前）
- プッシュ（GitHubへのアップロード）：一区切りついたら

## 17. 注意事項・よくある落とし穴

- 保管庫名・フォルダ名は**必ずローマ字**（日本語・絵文字・特殊文字はNG）
- `.md` 拡張子をファイル名末尾に必ずつける
- `2nd-brain` フォルダは「ダウンロード」に置いたままにしない（誤削除リスク）
- `.obsidian` フォルダは触らない（システムファイル）
- 既存保管庫にテンプレートを統合しようとするとワークフローが正しく動かなくなる
- 最初は「なんか違う」が当然。育てていくものなので一発目で100点を目指さない
- 音声入力を日常的に使うことでAI活用がより身近になる（キーボードより速い）
- 投資のアドバイスはAIの言うことを鵜呑みにしない
- 他人の著作物をスワイプファイルとして保存する際は個人利用の範囲を超えない
