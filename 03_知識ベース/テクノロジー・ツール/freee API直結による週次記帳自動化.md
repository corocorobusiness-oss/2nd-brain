# freee API直結による週次記帳自動化

> ⚠️ **2026-06-27追記**: このノートは2026-06-10時点の学び。現行方針では会計正本はfreee、旧AI帳簿CSV・収支管理は参考凍結。weekly-kichoは旧CSV等を書き換えるため記帳係から外し、後継は読み取り専用監査レポートとして作る。

(2026-06-10 日誌より)

## 学び

週次記帳（weekly-kicho）の全自動化テストに成功した。ブラウザ操作を介さず **freee APIに直結** して明細を取得・検証できることが確認できた。

- 新規明細0件のケースも正常にハンドリング（明細最終日まで取得できていることを確認）
- スケジュール型の自動化（週1実行）として安定運用できる見込み

## ポイント

- 会計まわりの自動取得は「ブラウザ自動操作」より「API直結」の方が壊れにくい
- 自動化の型は [[24時間AIエージェント運用設計]] の分類どおり「AI帳簿＝スケジュール型」で正解だった
- ジョブ本体の台帳記録は `01_プロジェクト/AI自動化/導入済み.md` を参照

## API仕様の注意点（trial_plの当月分算出）

(2026-06-12 / 2026-06-13 日誌より)

- freee の `trial_pl`（試算表P/L）は **年初来累計（YTD）** を返す。当月分だけが欲しいときはそのまま使えない。
- **当月分 = 現時点の累計 − 前月末時点の累計** で差分計算する。経営ダッシュボードの「今月売上／経費／利益」はこの方式で算出している。
- 口座残高は freee 帳簿の値（GMOあおぞら）を使用。
