## 結論

祐馬さんが作るべき「24時間動き続けるAIエージェント」は、万能AIを常時走らせるものではない。

正解は、Second Brainを正本にして、決まった手順は自動化し、判断が必要なところだけAIに任せる「壊れにくいAI会社」。

## 基本思想

AIエージェントは増やすほど壊れやすい。

40体のAI社員を並べるより、以下の3層に分ける。

| 層 | 役割 | 祐馬さん環境での置き場 |
|---|---|---|
| Workflow | 決まった手順 | `00_システム/10_Agent/workflows.md`、各プロジェクト手順 |
| Agent | 判断・文章・分析 | Claude Code / Codex |
| Tools | 実行・保存・取得 | freee MCP、AI帳簿、Obsidian、Google Drive、ブラウザ、launchd |
| Hooks | 暴走防止・強制ルール | 危険操作ブロック、Read強制、リトライ検知、終了時保存 |

## 自動化の選び方

| やりたいこと | 型 | 理由 |
|---|---|---|
| 売上記録 | イベント型 | 「Uber 5000」などを送った瞬間に処理できる |
| AI帳簿 | スケジュール型 | 週1で明細取得と仕訳検証をすればよい |
| YouTube制作 | オンデマンド型 | 調査・構成・台本は判断が重い |
| ナレッジ抽出 | スケジュール型 | 日誌から週1または毎晩抽出できる |
| 事業全体の改善提案 | 週次レビュー型 | 完全自律より、人間の視野を入れた方が壊れにくい |

## いきなり作らないもの

- 何でも勝手に判断する万能エージェント
- 外部投稿まで自動実行するエージェント
- 会計データを勝手に確定変更するエージェント
- 削除や設定変更まで自動で行うエージェント
- 台帳なしの常駐ジョブ

## 安全柵

### 1. Second Brainを正本にする

会話や各AIの記憶に閉じ込めない。

重要な判断・ルール・学びは必ずSecond Brainへ戻す。

### 2. 自動化台帳に必ず記録する

新しい自動化は `01_プロジェクト/AI自動化/導入済み.md` に記録する。

必須項目:

- ジョブ名
- 型
- 実行マシン
- 作成日
- 止め方
- 壊れたときの検知方法

### 3. AIの権限を分ける

AIに全部触らせない。

- 読み取りだけ
- 下書きまで
- 追記まで
- 実行前に確認
- 禁止

のように段階を分ける。

### 4. 失敗時は停止して報告する

同じ処理を短時間で何度も失敗したら、自動で粘らない。

停止して、原因と次の選択肢を報告する。

### 5. 外部送信は確認制

SNS投稿、Discord投稿、会計確定、ファイル削除、外部アカウント設定変更は人間確認を挟む。

### 6. Hooksで物理的に守る

テキストルールは、長時間動くほど守られなくなる。

だから「ルールとして書く」だけでなく、Hookで実行前後にチェックする。

祐馬さん環境で優先するHook:

| Hook | タイミング | 目的 |
|---|---|---|
| Dangerous Action Guard | 実行前 | 削除、外部投稿、会計確定、設定変更を止める |
| Read Before Write Guard | 書き込み前 | 共有ファイルを編集する前に最新内容を読ませる |
| Retry Loop Guard | ツール実行後 | 同じ失敗を短時間で繰り返したら止める |
| Session End Recorder | 会話終了時 | 重要な判断・学びをSecond Brainへ戻す候補を出す |
| Context Refresh Hook | 長時間作業中 | コンテキストが膨らんだら要約・引き継ぎメモを作る |

Hooksは万能AIを強くするものではなく、AIの権限を狭くして壊れにくくするための安全装置。

## 最初に作るべきMVP

名前: Yuma OS Watchtower

目的:

祐馬さんの事業OSがちゃんと動いているかを毎日または週次で見張る。

やること:

1. 今日のデイリーノートがあるか確認
2. AI帳簿の週次記帳ログを確認
3. 自動化台帳と実際のジョブのズレを確認
4. YouTube制作の未完了タスクを確認
5. 必要なら日誌に「今日見るべきこと」を追記
6. ヘルスチェック要約を `06_エージェント運用/30_ヘルスチェック/ヘルスチェックログ.md` に残す

Hooksとの関係:

Watchtowerは「毎朝の健康診断」。

Hooksは「作業中の事故防止」。

両方入れることで、24時間AIエージェントの土台になる。

やらないこと:

- ファイル削除
- 外部投稿
- 会計確定
- SNS投稿
- 勝手な新規ジョブ作成

## 判断基準

迷ったら、以下の順で判断する。

1. 祐馬さんの自由時間が増えるか
2. 繰り返し作業が減るか
3. 売上に近いか
4. 壊れたときに止められるか
5. Second Brainに記録が残るか
