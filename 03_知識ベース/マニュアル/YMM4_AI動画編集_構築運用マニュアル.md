# YMM4動画編集AI社員 構築・運用マニュアル

## 1. この文書の目的

この文書は、あなたが作成したYMM4プロジェクトを見本にして、AI社員が次の仕事をできるようにするための実践マニュアルです。

- 既存の完成済み動画を再現する
- 新しい台本から同じ編集スタイルの動画を作る
- 台本CSV、読み仮名辞書、素材を扱う
- 足りない素材を探し、候補を整理する
- YMM4プロジェクトファイルを作成する
- プレビューを確認し、問題を修正する
- 確認用動画を書き出す
- 人間の修正を次回のルールへ反映する

ここでは、YMM4プロジェクトファイルの拡張子を「`.ymmp`」と表記します。

---

## 2. 最初に覚えておく考え方

完成した`.ymmp`をAIに一度コピーさせるだけでは、新しい台本の動画は作れるようになりません。

必要なのは、完成動画から「あなたの編集レシピ」を取り出すことです。

| 項目 | たとえ |
|---|---|
| 完成した`.ymmp` | 完成した料理 |
| 台本CSV | 注文内容 |
| 読み仮名辞書 | 読み方のルール |
| 画像・動画・BGM・SE | 食材 |
| 編集ルール | レシピ |
| Hermes Agent | 編集担当社員 |
| YMM4操作ツール | 社員が使う作業道具 |

AI社員を育てる基本の順番は次のとおりです。

1. 同じ台本、辞書、素材から、同じ動画を再現する
2. 完成済みプロジェクトから編集ルールを取り出す
3. 一部分だけ変えた動画を作る
4. 似たテーマの新しい台本を作る
5. 完全に新しい台本を作る
6. 人間の修正をルールとして保存する
7. 次の動画へ自動適用する

---

## 3. おすすめのシステム構成

最も安定する構成は、AIにYMM4の画面を最初から最後までクリックさせる方法ではありません。

次の3つに分けます。

### 3.1 Hermes Agent

Hermesは動画編集担当者の「頭脳」です。

主な担当は次のとおりです。

- 台本の内容を理解する
- 台本をシーンへ分ける
- 必要な素材を考える
- 素材候補を選ぶ
- 編集設計図を作る
- プレビューを確認する
- 問題の原因を考える
- 修正方針を決める
- 作業ログを残す
- 人間へ報告する

### 3.2 YMM4用自動操作ツール

自動操作ツールは、大量の反復作業を担当します。

- セリフを追加する
- 画像を追加する
- 動画素材を追加する
- BGMと効果音を追加する
- 開始位置を設定する
- 長さを設定する
- レイヤーを設定する
- プロジェクトを保存する
- プレビュー画像を取得する
- 確認用動画を書き出す

実装方法は、YMM4用プラグイン、MCP、専用スクリプトなどです。

### 3.3 Computer Use

Computer Useは、APIやMCPで対応できない操作だけに使います。

- YMM4を起動する
- プロジェクトを開く
- エラーダイアログへ対応する
- 設定画面を操作する
- 動画出力画面を操作する
- YMM4を安全に終了する

大量のセリフ追加や素材配置を、Computer Useのマウス操作だけで行わせるのは避けます。

## 5. 案件フォルダの基本構成

案件ごとに、すべての入力と成果物を1つのフォルダへまとめます。

```text
JOB-2026-001/
├─ input/
│  ├─ script.csv
│  ├─ pronunciation.dic
│  ├─ instructions.md
│  └─ reference.mp4
│
├─ assets/
│  ├─ images/
│  ├─ videos/
│  ├─ bgm/
│  ├─ se/
│  └─ characters/
│
├─ template/
│  └─ master_template.ymmp
│
├─ work/
│  ├─ edit_plan.json
│  ├─ draft.ymmp
│  └─ draft_preview.mp4
│
├─ output/
│  ├─ final.ymmp
│  ├─ final.mp4
│  └─ thumbnail.png
│
└─ reports/
   ├─ input_validation.json
   ├─ asset_report.json
   ├─ qa_report.json
   ├─ edit_log.json
   └─ project.lock.json
```

### 重要なルール

- 元の`.ymmp`を直接上書きしない
- 素材は案件フォルダ内へコピーする
- 編集中はWindowsのローカルストレージを使う
- 共有フォルダ上のファイルをYMM4で直接編集しない
- 完成後にMac miniや共有ストレージへ戻す
- 案件ごとにフォルダを分ける
- 同じファイル名を使い回さず、案件番号を付ける

## 6. 最初に用意する「ゴールデン案件」

最初の完成済み動画を、AI社員の正解見本にします。

必要なものは次のとおりです。

- 完成した`.ymmp`
- 完成動画
- 元の台本CSV
- 読み仮名辞書
- 使用したすべての素材
- 使用したBGM
- 使用した効果音
- 使用した立ち絵
- 使用したフォント
- 使用した音声エンジン
- YMM4のバージョン
- 使用プラグイン
- 人間が行った修正内容
- 修正理由

フォルダ名の例は「`golden_case_001`」です。

### 最初の目標

AI社員へ次の仕事をさせます。

- 元の`.ymmp`をコピーしない
- 同じ台本、辞書、素材を使う
- 新しい`.ymmp`をゼロから作る
- 元のタイムラインと比較する
- 違いをレポートする
- 差が許容範囲になるまで修正する

## 8. 編集内容を4種類に分ける

完成済み動画の編集を、次の4種類へ分類します。

### 8.1 毎回固定するもの

- 動画サイズ
- フレームレート
- 字幕フォント
- 字幕位置
- キャラクターの基本位置
- 背景
- レイヤー構成
- BGMの基本音量
- オープニング
- エンディング

固定部分は「`master_template.ymmp`」へ入れます。

### 8.2 決まったルールで処理するもの

- セリフ終了後に一定の間隔を空ける
- 長い字幕は2行へ分ける
- セリフが終わるまで画像を表示する
- 話題が変わったら素材を切り替える
- 疑問文では疑問・驚き表情を使う
- 数字説明ではグラフを優先する
- 同じ素材を連続使用しない
- 画像は最低2秒表示する

これらは「`style_rules.yaml`」へ保存します。

### 8.3 AIが意味を考えるもの

- どの素材がセリフに合うか
- どこでシーンを分けるか
- どの言葉を強調するか
- どの表情が合うか
- 写真と図解のどちらを使うか
- 素材が不足しているか
- どんな検索語で素材を探すか

これはHermesが判断します。

### 8.4 特別な例外

- 特定キャラクターだけ字幕位置が違う
- オチだけ専用SEを使う
- 商品名が出たらロゴを表示する
- 注意喚起だけ背景色を変える
- 特定のコーナーだけBGMを変える

これらは「`exceptions.yaml`」へ保存します。

## 9. 新しい台本から動画を作る流れ

新しい台本を受け取ったら、いきなりYMM4を操作しません。

先に編集設計図を作ります。

1. 入力ファイルを検査する
2. 台本を読み込む
3. シーンへ分割する
4. セリフごとの役割を理解する
5. 必要な素材を決める
6. 素材候補を探す
7. `edit_plan.json`を作る
8. 設計図を検査する
9. YMM4へ反映する
10. プレビューを確認する
11. 問題を修正する
12. 確認用動画を書き出す
13. 人間へ提出する

### 編集設計図に入れる情報

- セリフ番号
- シーン番号
- 話者
- セリフ本文
- 読み方
- 感情
- 必要な素材の種類
- 素材検索キーワード
- 使用素材ID
- 背景
- 表情
- 字幕の強調語
- BGM
- 効果音
- トランジション
- 人間による確認が必要か

### `edit_plan.json`の例

```json
{
  "mode": "house_style",
  "scenes": [
    {
      "scene_id": "S001",
      "purpose": "動画のテーマを紹介する",

      "background": "default_room",
      "visuals": [
        {
          "line_id": "L001",
          "type": "image",

        {
          "line_id": "L002",
```

## 11. 読み仮名辞書の管理

案件ごとの`.dic`だけでなく、マスター辞書を作ります。

### マスター辞書の項目

- 表記
- 読み方
- 辞書の種類
- 使用音声エンジン
- 優先度
- 承認済みか
- 備考

### 例

```csv
surface,reading,type,voice_engine,priority,approved,notes
YMM4,ゆっくりムービーメーカーフォー,word,all,100,true,製品名
MCP,エムシーピー,word,all,90,true,技術用語
OpenAI,オープンエーアイ,word,all,90,true,企業名
```

### 自動検査する項目

- 重複登録
- 全角と半角の混在
- 正規表現エラー
- 短い単語が長い単語より優先されている
- 音声エンジンと辞書形式が合っていない
- 未承認の読み方
- 一文字だけを置換する危険な登録
- 固有名詞の読み不足

## 12. 素材ライブラリを作る

素材をフォルダへ入れるだけでは、AIが正しく選びにくくなります。

「`asset_catalog.csv`」を作ります。

### 登録する情報

- 素材ID
- 素材の種類
- 内容タグ
- 保存場所
- サイズ
- 動画の長さ
- 使用承認済みか
- ライセンス
- 作者
- クレジット
- 取得元
- 過去の使用回数
- ファイルのハッシュ値

### 例

```csv
asset_id,type,tags,path,approved,license,credit
IMG001,image,"AI,パソコン,技術",assets/images/ai_pc.png,true,internal,
IMG002,image,"会議,会社,社員",assets/images/office_meeting.png,true,licensed,作者名
VID001,video,"データ,グラフ,上昇",assets/videos/chart_up.mp4,true,licensed,作者名
SE001,audio,"驚き,疑問",assets/se/question.wav,true,internal,
BGM001,audio,"明るい,解説",assets/bgm/explanation.wav,true,licensed,作者名
```

### 素材を探す優先順位

1. 案件に付属した素材
2. 承認済み素材ライブラリ
3. 過去案件で再利用できる素材
4. 契約済み素材サイト
5. 不足素材として人間へ報告

### 外部素材のルール

AIがWebから見つけた素材を、そのまま本番で使用しないようにします。

最初は候補だけを集めます。

記録する内容は次のとおりです。

- 素材の取得元
- 作者
- ライセンス
- 取得日
- 元ページ
- クレジット表記
- 商用利用可能か
- 編集可能か
- 使用予定の場面

人間が承認した素材だけ、承認済み素材フォルダへ移します。

### 歴史人物イラストの固定ルール（2026-07-14）

- 人物イラストの第一候補は、じたんだの歴史人物素材とする
- 使用するのは、青系で**胸より上まで入った透過PNG**。顔だけの画像とSVGは使わない
- じたんだのサイズ欄は「横512px・縦0px」とする。縦0は高さ0の画像ではなく、比率維持で縦を自動計算する指定
- 同一人物は、同じ体・同じ衣装・同じ色味の5表情セットから選ぶ
- 5表情は「真顔・怒り・困り・微笑み・冷や汗」。台本上は「悲しみ→困り」「焦り→冷や汗」と対応させる
- 信長・家康など有名人物は、視聴者が一般に想像する定番像を優先する。若い姿や別デザインへ自動で切り替えない
- 織田信長は `j485_3_5.png`〜`j485_3_9.png` の512×641px胸上透過PNGを承認セットとし、顔中心の `j485_3_0`〜`j485_3_4` と旧320×401px版は不採用とする
- 台帳には、取得ページ、利用規約URL、SHA-256、表情、人物の正準キー、シリーズID、実使用案件を記録する
- ダウンロード済み・承認済みの枚数と、規約上の「使用数」を混同しない。使用数は案件への採用記録で集計する
- 素材ファイル自体を素材集として再配布しない。公開前に案件単位の権利確認を通す

参照: [じたんだ「織田信長」](https://jitanda.com/2018/02/10/j485_3/) / [じたんだ利用規約](https://jitanda.com/terms/)

## 13. タイムライン作成の基本ルール

### 音声の配置

文字数から音声時間を推測するだけでは、読み方によってズレます。

おすすめは次の方法です。

1. 音声を生成する
2. 実際の音声長を取得する
3. 音声終了位置を確認する
4. 指定した間隔を空ける
5. 次のセリフを配置する

### シーンの長さ

シーンの終了位置は、次のうち最も長いものへ合わせます。

- 音声の終了位置
- 画像の最低表示時間
- アニメーションの終了時間
- トランジションの終了時間

### レイヤーの役割

レイヤー番号を場当たり的に使わず、役割を固定します。

例：

- 背景
- 補助背景
- メイン画像・動画
- 図形・グラフ
- キャラクター
- 字幕
- 強調テキスト
- 効果音
- BGM

この対応表は「`layer_map.yaml`」へ保存します。

### 再実行できる設計

同じ案件をやり直したとき、アイテムが二重に追加されないようにします。

各アイテムを、案件番号、セリフ番号、素材番号で管理します。

既に存在する場合は更新し、存在しない場合だけ追加します。

## 14. AI社員の練習段階

### レベル1：完全再現

- 台本は同じ
- 素材は同じ
- 演出は同じ
- 自由な判断は禁止

合格条件：

- 素材欠落がない
- セリフ欠落がない
- 音声重なりがない
- 字幕切れがない
- タイムラインが正解表とほぼ一致する
- YMM4で開ける
- 動画を書き出せる

### レベル2：一部分だけ変更

- 画像を1枚だけ変更
- セリフを1文だけ変更
- BGMだけ変更
- 効果音を1つ追加
- キャラクター順を変更

変更した場所以外が壊れていないか確認します。

レベル2は、同じエージェントが変更と合格判定を兼ねません。

- 5課題は毎回、未変更のゴールデン案件から別案件として開始する
- 画像・セリフ・BGM・効果音・キャラクター順の各変更は、それぞれ制作担当が1種類だけ実施する
- 制作に参加していない別エージェントが、読み取り専用で変更範囲外の不変性を検査する
- 検査担当へ制作担当の自己評価を渡さず、固定見本・変更指示・機械差分だけで合否を決める
- 人間は最後に見た目と聴感を確認する

### レベル3：似たテーマの新しい台本

例：

- AI画像生成
- AI音声生成
- AI社員
- 業務自動化
- 動画制作効率化

既存の承認済み素材だけを使います。

### レベル4：完全に新しい台本

- 新しいテーマ
- 新しいシーン構成
- 素材不足の可能性あり

素材不足時は勝手に本番採用せず、候補一覧を提出します。

### レベル5：人間は最終確認だけ

- 台本を受け取る
- 編集設計図を作る
- 素材を選ぶ
- YMM4プロジェクトを作る
- プレビューを検査する
- 問題を修正する
- 確認用動画を出力する
- 人間が承認する

## 15. 人間の修正をAIのルールへ変える

Hermesが作った動画を人間が修正したら、必ず記録します。

### 記録する内容

- 修正前
- 修正後
- 修正理由
- 今後も使うルールか
- 適用条件
- 例外条件
- 優先度

### 例

Hermesが売上説明に人物写真を使ったが、人間がグラフへ変更した場合：

「売上、増加、減少、割合、パーセントなどの数値説明では、人物写真よりグラフや図を優先する」

このルールを「`style_rules.yaml`」へ追加します。

### 保存先

- `style_rules.yaml`
- `exceptions.yaml`
- `dictionary_master.csv`
- `asset_catalog.csv`
- `qa_rules.yaml`
- Hermes用の`SKILL.md`
- 修正履歴ファイル

AIの会話記憶だけに頼らず、必ずファイルへ保存します。

## 16. 品質チェック

### 必ず確認する項目

- すべての素材が存在する
- すべてのセリフが入っている
- 音声が正しく生成されている
- 音声が重なっていない
- 字幕が画面外へ出ていない
- 字幕が長すぎない
- 読み仮名が正しい
- 長時間の黒画面がない
- 画像が音声より先に消えていない
- BGMが大きすぎない
- 効果音が不自然でない
- 立ち絵と感情が合っている
- 同じ素材を使いすぎていない
- 素材の権利情報が記録されている
- YMM4でプロジェクトを開ける
- 動画を書き出せる
- 完成動画が正常に再生できる

### 自動修正の回数

最大3回までにします。

1. 1回目は素材欠落、セリフ欠落、構造問題を修正
2. 2回目は音声、字幕、タイミングを修正
3. 3回目は見た目、素材選び、演出を調整
4. まだ問題が残る場合は人間へ報告

無限にやり直させないようにします。

- PLANNED：編集設計図作成済み
- WAITING_FOR_ASSETS：素材待ち

## 26. 最重要ポイント

3. 似た台本を作る
4. 人間が修正する
5. 修正理由をルール化する

8. 人間の確認作業を少しずつ減らす

AI社員を育てる中心は、AIモデルの追加学習ではありません。

次の資産を整えることが中心です。

- 正解見本
- 編集設計図
- 編集スタイルルール
- 例外ルール
- 読み仮名辞書
- 素材カタログ
- 修正履歴
- 品質確認ルール
- 安全なYMM4操作ツール

最初の一歩は、あなたが作った完成済み`.ymmp`に含まれる、すべてのアイテム、レイヤー、開始時間、長さ、素材、設定を一覧化することです。

その後で、新しい台本から編集設計図を作る段階へ進むのが、最も安定する方法です。

## 27. このチャンネルで確定した人物素材・容量・Level 1運用

### 人物素材

- 基本人物は、じたんだの青系・胸より上・透過PNGを使う
- サイトのサイズ入力は横512px、縦0px（縦は比率維持）
- 承認済みファイルの実寸は512×641pxとして検査する
- 真顔、怒り、悲しみ・困り、笑い・微笑み、焦り・冷や汗の同一デザイン5表情から選ぶ
- 顔だけ、SVG、旧320×401px、背景非透過、若年版、別シリーズは自動採用しない
- 信長・家康などの有名人物は、視聴者が一般に想像する定番の肖像・姿を優先する

### 容量と移行

- 編集中は現行案件1件だけWindowsローカルへ置く
- 移行用handoffと完了案件はSSDへ保管する
- SSDへコピーしただけで本体を削除せず、transport manifestの全ファイル・サイズ・SHA-256一致を確認してから削除する
- SSDのhandoffをWindowsノートへコピーした後も、ノート側の検証がPASSするまでSSD正本を削除しない
- 共有フォルダ上の`.ymmp`や素材をYMM4で直接編集しない

### Level 1の必須順序

1. YMM4を起動し、伏字辞書の有効件数が0であることを確認する
2. 単語辞書を全消去する
3. その案件の`pronunciation.dic`だけをインポートする
4. YMM4を終了して設定を確定し、PRE_VOICE検査で単語311件・伏字有効0件を確認する
5. ここまでPASSしてからCSVまたは候補`.ymmp`を開き、音声を生成・更新する
6. 表示文字は漢字・数字のまま、読みだけを発音欄へ適用する
7. 元ファイルを上書きせず別名保存する
8. 正解表を2回抽出して一致を確認し、新規MP4を書き出す
9. MP4を全デコードし、golden動画と尺・解像度・フレーム数・音声・見た目を比較する

Level 1が合格する前にLevel 2へ進めない。Computer Useの呼び出し可能な操作ツールがない場合は、GUI工程を実行済みと報告しない。
