# qa_check.py 敵対的ストレステスト監査と修正（2026-06-19）

creative-thread-gen の `qa_check.py`（全16ゲート）を、マルチエージェントで**偽陰性ハント（AI癖なのにすり抜ける合成）／偽陽性ハント（本物っぽいのに誤爆する合成）**で総当たりした記録。各ゲートのレッドチームが合成スレを実際に qa_check に通して**実測で**穴を炙り出した（机上推測ゼロ）。

- 規模: 16ゲート×レッドチーム16体＋統合1体、**偽陰性51件・偽陽性29件**を実証
- 元レポート(全文): ワークショップ run `wf_33b2c397-e30`

## 見つかった致命傷
1. **正当な史実スレを一発FAILで出荷ブロック（過検出）**
   - 禁止語: `殺`の部分一致で **暗殺/殺到/黙殺/抹殺/必死/虐殺/毒殺/相殺/殺生** が全部FAIL → 乙巳の変・大坂の陣が構造的にブロック（入鹿スレ殺×11・真田スレ殺4死3 で❌を実測）
   - 蔑称: `ジ.プ`ワイルドカードが車の**ジープ**を誤爆／**支那そば**(ラーメン)を誤爆／**穢多・非人**の学術スレを一発FAIL
   - メタ締め: `様式美`単独マッチで茶道/城/建築の正当トピックを誤FAIL
2. **当て字・ひらがなでヘイト/罵倒が素通り（偽陰性・規約一発アウト級）**
   - **ヒトモドキ/鮮人/特亜/下チョソ/エベンキ** がリスト外で全スルー、**氏ね/ころす/ぬっ頃す** のひらがな当て字罵倒もノーマーク、**露 助/ロ助**で分断回避
3. **閾値が本物より厳しすぎ＝参照3本が全部WARN（アラート疲れ）**
   - 9c語尾50%閾値: 道三50%/早雲52%がWARN。主因は本物主力の「な/なあ」を母数に算入する自己矛盾
   - 9f密度12閾値: 本物自然帯18〜24なのに閾値12 → 道三18/早雲24/将門23が全員WARN

## 適用した修正（7箇所・.claude＋ポータブル両方に反映済）
| ゲート | 修正 | 効果 |
|---|---|---|
| 禁止語 BANNED_RE | `殺`を負後読み/負先読みで史実複合語＋受け身を免除 `(?<![暗黙抹相必虐毒惨])殺(?!到\|戮\|伐\|害\|生\|さ)`、`死`は罵倒核`死ね`系のみに | 暗殺/虐殺/必死/殺到/殺される 等が誤爆しなくなり、殺すぞ/ぶっ殺/死ね の脅迫は残る |
| 禁止語 BANNED_RE2(新) | 当て字罵倒 `ころ[しすせそ]\|ぶっころ\|ぬっころ\|頃[しすせそ]\|氏ね\|逝ね\|しね\|くたばれ` | ひらがな・崩し罵倒のすり抜けを補強 |
| 蔑称 SLUR_RE | `ジ.プ`→`ジャップ\|ジヤップ`(ジープ誤爆除去)、`支那`→`支那(?!そば\|竹\|料理\|チク)`、隠語追加 `ヒトモドキ\|(?<!朝)鮮人\|特亜\|チョソ\|エベンキ\|気狂い\|メクラ` | ジープ/支那そば誤爆解消＋隠語・崩しスラー検出。`鮮人`は`(?<!朝)`で朝鮮人を誤爆しない |
| メタ締め META_RE | `様式美`単独→`(?:スレ\|話題\|議論\|流れ\|展開\|荒れ).{0,12}様式美` 等のメタ文脈共起のみ | 茶道/城の正当トピックを誤FAILしない |
| 構造メタ漏れ(新ゲート) | **FAIL層**=概要欄/チャンネル登録/この動画/次回/テロップ等の配信語、**WARN層**=前半/後半/本編＋解説した等の編集者視点動詞の近接 | 「概要欄に貼っとく」等の編集構造漏れをFAID。在スレの「前半の祟りの話に繋がる」(繋がる=非動詞)は正しくスルー |
| 9c 語尾偏り | 上限WARNを「な/なあ」除外のcore比で判定 `core_share>42` | 道三50→core35・早雲52→core37・将門39→core24 で「な/なあ自己矛盾WARN」を解消 |
| 9f 密度 | 閾値 12→26（本物自然帯18〜24を誤爆しない） | 道三18/早雲24/将門23 が✅化 |

## 回帰テスト結果（全GREEN・回帰ゼロ）
- 既存5本（合成A/合成B/平将門/斎藤/早雲）: 判定もexitも不変。9f が3本とも✅化、9c が平将門✅化・斎藤/早雲はcore換算でな/なあ矛盾解消
- FPケース: 真田✅/入鹿✅/ジープ✅/支那そば✅/様式美✅（誤爆解消を実測）
- FNケース: ぶっころす❌/氏ね❌/ヒトモドキ❌/鮮人❌/特亜❌/エベンキ❌/チョソ❌/メクラ❌/気狂い❌（穴を塞いだ）
- コントロール: 殺すぞ/ぶっ殺/死ね❌（脅迫は残る）・史実複合語✅・構造メタFAIL/WARN/スルーの3層・朝鮮人は非誤爆で鮮人どもだけ検出

## 判断保留（経営判断/要LLM・未適用）
- **穢多・非人をSLUR一発FAILから降格**するか（歴史chの正当題材を救う vs 悪意使用を確実に止める）＝収益化リスク許容度の判断
- **露 助/ロ助の分断回避**（正規化が必要・他ゲートの行頭アンカーに影響しうるため要設計）
- **9g 長文独り語り**の偽アンカー耐性化＋lanc<25格下げ（参照3本が全WARN・>>N飾るだけで回避可能だが、改修が中規模）
- **現代スラング(ガチ/マジで)比率閾値**＝本物コーパスの実出現率を測ってから
- **時代混線の別名マップ**（龍馬⇔坂本龍馬等）＋相対優位比への変更

## 機械化の限界（regex では原理的に届かない＝LLM/人判断が要る）
- **水増しスレ**（中身スカスカのヘッジ反復モノローグ）は字数バンド内に収めれば文字数では検出不能 → 密度系ゲートで部分カバー＋最後はLLM
- **構造メタWARN層**（前半/後半）は在スレ参照と編集都合が語彙上ほぼ同一 → 動詞近接で大半を分けるが最終はLLM文脈
- **属性ゼロの口語コレクティブ**（連中/やつら）狙いの非人間化＝武将個人にも使う汎用語のため閉リストでは捕捉不能 → LLMゲート担保
- **史実身分・民族語の批判的引用 vs 悪意蔑視**＝スタンスを読む必要があり機械分離困難

教訓: **一発FAIL系ゲートは"出荷ブロック"なので過検出のコストが最大**。部分一致・ワイルドカードは史実用語を巻き込む。免除は負後読み/負先読みで固定長に、FN補強は別REでOR。閾値は単一スレでなく参照合格スレの実測帯で較正する。

## 追記：YMM4 AI社員のレベル2は制作者と検査者を分離する（2026-07-14）

池田祐馬さんの指摘により、YMM4の一部分変更テストでは、変更を行ったエージェント自身に合否判定をさせない。

- 画像1枚変更：素材編集エージェント
- セリフ1文変更：台本・音声エージェント
- BGMだけ変更：BGMエージェント
- 効果音1つ追加：SEエージェント
- キャラクター順変更：タイムライン構成エージェント
- 各課題のQA：制作へ参加していない新規QAエージェント
- 最終判定：司令塔エージェント。制作物は変更せず、固定見本とQA証拠だけで判定
- 人間：最終の見た目・聴感確認

各課題は未変更のゴールデン案件から別案件として開始し、変更対象を1種類に固定する。合否は自己申告ではなく、`.ymmp`差分、素材ハッシュ、音声、全フレーム、音量の機械比較で決める。QA担当には制作者の自己評価を渡さず、読み取り専用で監査させる。同じモデルを使う場合も、別インスタンス・別コンテキストと固定検査器を併用する。
