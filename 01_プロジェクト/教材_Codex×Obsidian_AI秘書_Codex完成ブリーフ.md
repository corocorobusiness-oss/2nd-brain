# Codex完成ブリーフ集 — 教材出荷をCodexで完成させる

作成日: 2026-07-06
正本: このファイル（コピーが `~/Projects/kyozai-launch/_ブリーフ集.md` にある）

## 使い方（祐馬さんの手順）

1. Codexデスクトップアプリで `~/Projects/kyozai-launch` をプロジェクトとして開く
2. ブリーフ1→3→4を順にコピペして実行（各ブリーフの完了報告を見てから次へ）
3. ブリーフ2だけは `kyozai-launch/demo/2nd-brain` を別プロジェクトとして開いて実行（読者と同じ環境を再現するため）
4. ブリーフ3のレポートに修正点が出たら、レポートをあおいに渡す（原稿の修正はあおい＝正本側で行う）
5. 最後に入稿キットを使ってBrainに貼る。**公開ボタンは必ず人間が押す**

※スクショ撮影リスト.md のブリーフA/Bはこのブリーフ集に統合済み（こちらが最新）。

## ブリーフ1: デモ環境準備（kyozai-launchで実行）

```text
【ゴール】
教材スクショ撮影用のデモ環境を作る。

【手順】
1. ./demo/2nd-brain/ を新規作成し、原稿/00_事前準備.md と 原稿/Day1.md の記述どおりの最小構成を作る
   （AGENTS.md、00_system/にprofile.md・goals.md・rules.md・workflows.md、01_projects/、02_daily/、03_memos/、04_outputs/、90_archive/）
2. 各ファイルの中身は 特典テンプレ/ の同名ファイルをコピーする（冒頭の「置き場所」行は削除する）
3. 02_daily/ に今日の日付（YYYY-MM-DD.md）のデイリーノートを作り、原稿/Day4.md のBefore例のような架空の雑メモを6行入れる
   （例: 教材のDay4を書きたい / 販売ページに図を入れる / Xで制作過程を投稿したい など。実在の情報は使わない）
4. 03_memos/brain-ideas.md を作り、架空のアイデアメモを4行入れる
5. 完成した構成をツリー表示で報告する

【制約】
- ./demo/ の外に書き込まない
- 何も削除しない
- 実在の個人情報・実売上・実URLを書かない
```

## ブリーフ2: 実演＋スクショ準備（demo/2nd-brainをプロジェクトとして開いて実行）

```text
【ゴール】
教材スクショ用の「画面の状態」を作る。私が撮影するので、各ステップで止まってほしい。

【手順】
1. 「今開いているフォルダの名前と、中にあるファイルやフォルダの一覧を教えてください」に答える
   → 答え終わったら「S8撮影OK」と言って止まる
2. 私が「次」と言ったら、00_system/goals.md と今日のデイリーノートを読んで、メモを
   「今日やること（3つまで）・タスク・アイデア・学び・明日の一手」に整理する。元メモにない内容は足さない
   → 整理し終わったら「S11・S12撮影OK」と言って止まる
3. 私が「次」と言ったら、AGENTS.mdの「役割」を一言で要約する
   → 答えたら「AGENTS.md読み込み確認OK」と言って終了

【制約】
- このフォルダの外を触らない
- ファイルを削除しない
- 私が「次」と言うまで先に進まない
```

## ブリーフ3: 通し再現テスト（kyozai-launchで実行）

```text
【ゴール】
教材を買った初心者になったつもりで、原稿だけを頼りに進めたとき詰まる箇所を洗い出す。

【手順】
1. 原稿/00_事前準備.md → Day0.md → … → Day7.md の順に読む
2. ファイル操作で再現できる指示（フォルダ作成・ファイル作成・テンプレ記入・プロンプト実行）は、
   ./test-run/2nd-brain/ に実際に再現してみる
3. 次の観点で問題を探す:
   - 手順どおりにやると詰まる・順番が前後している
   - 説明が曖昧で初心者が迷う
   - ファイル名・フォルダ名・見出しの表記ゆれや矛盾
   - プロンプトを実際に実行すると教材の説明と食い違う
   - 誇張・約束しすぎの表現
4. インストールやサインインなどUI操作は再現できないので、「文章として指示が明確か」だけ判定する
5. 結果を ./レポート_通しテスト.md に書く。形式:
   | 対象ファイル | 該当見出し | 問題 | 修正案 | 重要度(高/中/低) |
   問題ゼロの観点も「確認済み・問題なし」と明記する

【制約】
- 原稿/ は絶対に編集しない（レポートに書くだけ。修正は別の担当が行う）
- 何も削除しない
```

## ブリーフ4: 入稿キット生成（kyozai-launchで実行）

```text
【ゴール】
Brainの入稿画面に「貼るだけ」の状態を作る。

【手順】
1. ./入稿キット/ を新規作成する
2. 入稿キット/01_販売ページ_無料部分.md を作る:
   販売ページ.md の内容をそのままコピーし、[画像挿入: ...] の行を
   「▼▼ ここに 画像/○○.png を挿入 ▼▼」という目立つ1行に置き換える（文章は一切変えない）
3. 入稿キット/02_有料部分_本文.md を作る:
   原稿/00_事前準備.md → Day0.md → … → Day7.md をこの順で結合する。
   各パートの間に「---」の区切りを入れる。文章は一切変えない
4. 入稿キット/03_特典_まとめ.md を作る:
   特典テンプレ/ の8ファイルを「## ファイル名」の見出しつきで1ファイルに結合する（配布用）
5. 入稿キット/04_画像/ を作り、画像/ の8枚を貼る順（表紙→01→…→07）にコピーする。
   demo/2nd-brain/04_outputs/screenshots/ にスクショがあればそれも末尾にコピーする
6. 入稿キット/05_入稿チェックリスト.md を作る:
   - 貼る順番（無料部分→価格設定→有料部分→特典→画像挿入位置）
   - 価格: 先着10名980円 / 11〜30名1,580円 / 31名以降2,980円
   - 公開前チェック: スマホ表示 / 画像の文字サイズ / 個人情報の写り込み / 公開ボタンは人間が押す
7. 完成したら入稿キットのファイル一覧を報告する

【制約】
- 原稿・販売ページ・特典テンプレの文言を変えない（結合と画像指示の整形のみ）
- 何も削除しない
- Brainへのアップロードや外部送信はしない（人間がやる）
```
