# 経営ダッシュボード Phase 1

Second BrainとYouTube制作フォルダを読み取り、昨日までの経営状況をMac内だけで表示する。
既存データ、会計、制作物、launchd、認証情報には書き込まない。

## 起動

```bash
cd ~/2nd-Brain/01_プロジェクト/AI自動化/経営ダッシュボード
python3 dashboard.py
```

ブラウザで <http://127.0.0.1:8765> を開く。終了は `Control-C`。

## 表示する正本

- 配達予算・YouTube収益目標: `02_経営/目標と計画.md`
- YouTube投稿予定: `03_知識ベース/YouTube・コンテンツ制作/YYYY-MM_ネタslate.md` の「公開カレンダー」（正本。ダッシュボードが直接読み込む）
- 日別売上: `05_日誌/YYYY-MM-DD.md`
- 経費（確認済み）: freee APIの読み取り専用確認。通常30分間は同じ結果を使い、失敗時は5分後に再確認する。認証情報は書き換えない
- 経費（速報）: Google Drive `経費精算/YYYY/MM/` のレシート。画面が1分ごとに再読込する
- freeeが読めない時: `02_経営/帳簿/freee_export/` の最新ローカルスナップショット
- AIジョブ: `01_プロジェクト/AI自動化/導入済み.md`
- YouTube制作工程: `~/Projects/youtube/創作スレ下書き/`
- 最近できたYouTubeの仕組み: `~/agent-skills/`、`06_エージェント運用/00_司令塔/NOW.md`、`~/Projects/youtube/eval/runs/`、YMM4完成記録

集計基準日は常に昨日。欠損を0円と見なさず、`確定 / 暫定 / 未取得` を表示する。
YouTube収益はAPI反映が通常2〜3日遅れる。経費は `freee確認済み + Driveにだけある未反映レシート` で表示し、同じ日・同じ金額・同じ取引先のものは二重に足さない。曖昧な一致や古いfreee明細は「要確認」として赤く表示する。

売上は昨日まで、経費は今日追加されたレシートまでを速報表示する。そのため「利益の目安」は経費側だけ今日分を含む保守的な数字になる。

YouTube画面の「最近できた仕組み」は、同日の更新時刻も含めて新しい順に表示する。ClaudeとCodexの両方から同じスキル正本を読め、最新変更後の確認も通っている場合、または完成記録がある場合だけ青い「使える」にする。未完了・昇格見送り・未確認の変更・情報源欠損は赤い「待機中 / 改善中 / 確認が必要」として表示する。仕組みの情報だけ読めない場合も、売上・経費の画面はそのまま表示する。

## Phase 1の境界

- チェックボックスは制作物から判定した表示専用。
- 外部公開、スマホからの外出先アクセス、自動起動は未設定。
- 本番タスク更新と外部アクセスは、別の危険操作承認後に実装する。
