# YMM4 dic漏れゼロ改善案 天才会議 2026-06-30

## テーマ

`ymm4-dic-generator` が家康台本で多数の漏れ・誤読を出したため、次回以降「漏れたまま完了扱い」にならない改善案を作る。

## 前提

厳密な意味で「絶対に漏れない」はAI/自然言語では保証できない。よって今回の設計目標は次の2つ。

1. 漏れを限界まで減らす
2. 漏れが残った場合でも、完了扱いにせず機械的に止める

## 評価基準

1. 漏れ検出力: 固有名詞・史料名・書名・出版社名・地名・単漢字・自然物・普通名詞を拾えるか
2. 読み正確性: `村岡素一郎` のような登録済み誤読を防げるか
3. 誤爆安全性: 文脈依存語をdicで固定しすぎないか
4. 機械検証: validatorが漏れ・誤読をFAILにできるか
5. 運用負荷: 毎回重すぎず、重要回だけ強化できるか
6. 既存STEP1との整合: `reading_fix.py` / Hatsuon / YMM4実機検証の責務を壊さないか

## 5人会議の結論

結論はほぼ一致。

`dicをもっと増やす` だけでは不十分。今回の本当の問題は、`validate_dic.py` が「dicに入っている語が台本にあるか」は見ていたが、「台本にある危険語がdicから漏れているか」を見ていなかったこと。

したがって最案はこれ。

> 候補抽出は広く、dic登録は慎重に、未解決のままHatsuonへ焼かない。

厳密な「絶対に漏れない」は保証しない。代わりに、**漏れたまま完了扱いにしない fail-close 設計**にする。

## 役割別の合意

### 1. 抽出設計

- CSVの `セリフ` 列だけを正として、行番号つき本文を作る。
- LLM目視の前に、機械で広く候補を出す。
- 候補生成は4系統に分ける。
  - 全漢字スイープ: 漢字連続、漢字+かな、`AのB`、数字+助数詞
  - 危険語スイープ: 単漢字、自然物、生活語、建物、史料名、書名、出版社名
  - ローカル辞書照合: 共通dic、過去dic、固定読み表にある語が台本に出たら候補化
  - 文脈キュー抽出: `本`、`書物`、`記録`、`出典`、`史料`、`小説`、`刊行`、`社`、`寺`、`鐘` 周辺を強制候補化
- 候補は `must / review / reject` に分ける。
- `reject` した語も理由を残す。これで「漏れた」のか「意図して除外した」のかを区別する。

### 2. 読み正確性

- 既存dicは正解源ではなく候補源として扱う。
- `村岡素一郎=むらおかそいちろう` のような登録済み誤読は、漏れより危険な「誤読固定」としてFAIL。
- `known_readings.tsv/json` を持つ。
  - `surface`
  - `reading`
  - `category`
  - `context`
  - `source_type`
  - `source_url`
  - `confidence`
  - `verified_at`
  - `note`
- 永続表に入れるのは、全動画で安定して同じ読みになる語だけ。
- `葦` のような多読み普通語は、永続dicに即固定せず、文脈つき候補または行別overrideへ送る。

### 3. validator v2

`validate_dic.py` は「形式チェック」から「台本×dicの漏れ検出器」に昇格する。

追加する中核は3つ。

- `required_if_present`: 台本に出たらdic登録必須の語リスト
- `known_readings`: 登録済み語の正しい読み照合リスト
- `candidate_extractor`: 台本から高リスク候補を抽出する層

判定は3段階にする。

**FAIL**

- フォーマット不正、重複、ASCII `"`
- 台本に出た必須語がdic未登録
- 既知読みと登録読みが不一致
- 読みに漢字が混じる、空読み、カラム指定ミス
- 未解決flagsを残したまま `.ymmp` へ進もうとした
- YMM4保存後に VoiceLength 0 が残る

**WARN**

- 広めの候補抽出で拾った高リスク語が未登録
- 単漢字、自然物、生活語、普通名詞の候補漏れ
- 文脈で読みが割れる語
- 低確信語が多い

**INFO**

- 総行数、候補数、登録済みカバー率
- カテゴリ別候補数
- 除外語と除外理由
- dicでカバー済み候補の一覧

## 採用しない案

- `.dic` に全部背負わせる
- 台本全文を `.dic` スキルで直接Hatsuon化する
- 文脈依存語を共通dicへ固定する
- 候補に出た語を全部自動登録する
- validatorのWARNを全部FAILにする
- 重要回用の5人監査を通常回の必須工程にする

理由: 漏れは減っても、誤読固定・過剰停止・運用疲れが起きる。

## 採用する設計

層を4つに固定する。

1. `per-video dic`: その動画で安定して読む固有名詞・歴史語
2. `reading_overrides`: 全動画で安定して同じ読みになる既知修正
3. `per-line context overrides`: 文脈で読みが変わる語
4. `Hatsuon`: 最終的に `.ymmp` に焼く行単位の読み

`ymm4-dic-generator` は、最終読みの頭脳ではなく、**候補抽出・dic生成・漏れ検出・証拠レポート**を担当する。

最終読みの中核は引き続き `reading_fix.py` と Hatsuon。最後はYMM4実機で VoiceLength を回収して閉じる。

## 実装順

1. `known_readings.tsv/json` を追加する
2. `high_risk_terms.tsv/json` を追加する
3. `candidate_report.tsv/json` を出す
4. `validate_dic.py` をv2化する
5. 家康回fixtureを追加する
6. `SKILL.md` の完了条件を書き換える
7. STEP1の入口で `validator v2 PASS + unresolved flags 0` を必須にする
8. 最終出口で `VoiceLength全行非ゼロ + 重なり0 + 欠損0` を必須にする

## 通常回と重要回の違い

**通常回**

- validator v2
- candidate_report確認
- unresolved flags 0
- YMM4保存
- VoiceLength / tighten検算

**重要回**

- 通常回に加えて、5視点の独立監査
- 人名・史料名・書名・寺社名・地名の出典確認
- 耳チェック候補を最大15〜20件まで増やす

## 最終結論

`ymm4-dic-generator` の強化は必要。ただし、方向は「全部dicに入れる」ではない。

最案は、**候補抽出を広げる、登録は絞る、validatorで漏れと誤読を止める、HatsuonとYMM4実機証拠で最終確認する**。

これなら、自然言語としての「絶対に漏れない」は言わずに、制作システムとして「漏れたまま完了しない」はかなり現実的に作れる。
