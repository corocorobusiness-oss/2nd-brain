# 外部レビュー指摘台帳

外部AI（Fable 5 / 独立レビューAI 等）からのレビュー指摘を、ID付きで採用/保留/却下と反映状態まで追跡するための台帳。
根拠: `AIエージェント編成ルール_天才会議_2026-07-01.md`「複数AIの出力を統合する時は、採用/却下/却下理由/残リスクを記録する」の記録先を本ファイルに固定する。

## 運用ルール

- 判定は `採用 / 保留 / 却下 / 未判定` の4値。採否の判定はCodex側・祐馬さんの権限で行う（外部レビュアーは関与しない）。
- **append-only。既存行の書き換え・削除は禁止**。判定や状態の変更は**新行**として追記する（元IDに `-r2` `-r3` … の枝番を付ける。同一IDの最新は最下行）。typo訂正も新行で行う。
- 状態は `未反映 / 反映済み(YYYY-MM-DD) / 期限切れ / 対象外` のいずれか。`採用` なのに `未反映` のまま残っている行は月次点検で拾う。
- **次回の外部レビュー依頼文には、本台帳の却下・保留一覧（判定=却下・保留のIDと要約）を添付**し、同じ指摘の蒸し返しを防ぐ。
- 月次点検への接続は現時点では未配線。後続案件で期日タスク等へ接続するまでは、`採用` かつ `未反映` の行と `未判定` の行を人手で確認する。
- `期限切れ` は、個別期限または `next_review` を過ぎても `未反映` / `未判定` のまま残っている状態に使う。
- 記載禁止: 秘密情報、トークン、Cookie、生ログ、未マスクの個人情報、認証済みURL。指摘は要約のみ記載する。
- 指摘の原文が必要な場合は、出所のレビュー文書（履歴ファイル）を参照する。本台帳は索引と採否の正本であり、レビュー本文の複製先ではない。

## ID規約

`REV-YYYYMMDD-<出所>-NN`

- `YYYYMMDD`: レビュー実施日
- `<出所>`: 小文字英数（例: `fable5`, `codex`, `gpt`）
- `NN`: 同日・同出所内の連番2桁
- 改訂行（判定変更・訂正）: 元IDに `-r2` 以降を付与。ID重複は禁止（枝番で一意化する）。
