## 5. 実装仕様

### P1: 台帳＋正準キー解決
- 新設 `~/Projects/youtube/asset-pipeline/`。台帳は `assets_ledger.jsonl`（追記型・git管理）
- 台帳フィールド: `id`(sha256) / `canonical_key`(Wikipediaページ名) / `kind`(固有|汎用) / `tags[]` / `caption` / `file_path` / `source`(commons|generated|manual) / `source_url` / `license` / `license_ok` / `credit_text` / `gen_prompt` / `gen_model` / `used_in[]` / `rejected[]`
- **正準キーは自由記述禁止**。台本の固有名詞→Wikipedia日本語版ページ名（リダイレクト解決込み）に正規化してから台帳検索。解決不能語は `未解決リスト` に出す（推測登録しない）
- 二重登録防止: sha256一致、または（canonical_key＋source_url）一致なら追加せず既存を返す
- テスト: 教師CSV3本（新選組・家康影武者・将門）で抽出→解決リストと未解決リストを出力

### P2: Commons取得＋ライセンス判定（fail-close）
- 固有物は検索でなく**本人のWikipediaページ直**→ページ内画像→Commons `imageinfo` の `extmetadata` でライセンス機械判定
- **許可リスト方式: PD / CC0 / CC-BY / CC-BY-SA のみ**。判定不明・NC/ND・fair use は取得しない（候補にも出さない）
- CC-BY系は `credit_text` の自動生成を必須ゲートにする
- **固有物のAI生成フォールバック禁止**（史実素材の捏造防止）。取れなければ欠落として未解決リストへ
- DL先: `/Volumes/SSD/YouTube/素材ライブラリ/固有/<正準キー>/`
- API利用は常識的なレート（リクエスト間隔を空ける・並列乱射しない）。User-Agent明示

### P4: Pass 2（YMM4保存済み .ymmp への ImageItem 追加）
- 入力: YMM4保存済み .ymmp（テストは `_ymm4自動化テスト/` 内で新規に用意。教師データ4本は読み取り専用の参照のみ）
- **既存アイテム（VoiceItem/Hatsuon/VoiceLength/Characters含む）は一切変更しない。追加専用**
- 無変更検証は**オブジェクト単位の意味比較**で行う（JSON再シリアライズの改行・数値表記ノイズをバイトdiffで誤検知しないこと）。検証FAILなら出力破棄
- 画像タイミングは保存済みVoiceItemの Frame/Length（確定値）から算出。CSV行⇔VoiceItemの1:1対応を座標系に使う
- 配置プリセット: 教師データ4本から用途別に抽出（例: コメント画像=Y230/Zoom80/Outline+Jump）。位置・タイミングに小さな乱数揺らぎを付けてテンプレ感を回避
- 出力: 新規連番 `auto_*.ymmp`。FilePath は UNC正準形で書く
- 実機確認: 祐馬さんがWindows側YMM4で開く（Codexは「実機確認済み」を自己申告しない）

### コンタクトシート（P4の出荷物に含める）
- 採用画像の一覧（画像＋使用行＋出典URL＋ライセンス＋クレジット＋確信度）をHTMLか画像1枚で出力

## 6. 禁止事項（実装中も恒久も）

- 発音（Hatsuon）の生成・修正（YMM4の自動生成が正。読み辞書は別系統）
- 既存音声アイテムへの変更
- いらすとや・AC系サイトへの自動アクセス・自動DL
- NC/ND・fair use・ライセンス不明素材の使用
- launchd/cron登録・常駐化
- 認証情報・トークンへのアクセス
- テスト目的でも教師データ4本・既存案件 .ymmp の上書き
