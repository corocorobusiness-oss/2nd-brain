# オリジナルスレ先生 Fable Five依頼文 天才会議 2026-07-03

## 目的

Anthropic最上位モデル Fable Five に、2ch歴史チャンネル用「オリジナルスレ先生」スキル/パイプラインの改善案を渡すための依頼文を作る。

## 固定した採点基準

1. スレ本文を主役にする。
2. 解説パートをスレ本文の復唱にしない。
3. 通常制作で回る軽さを保つ。
4. 既存の creative-thread-pipeline / creative-thread-gen / youtube-script-parts / script-third-party-review に反映できる。
5. Fable Five から抽象論ではなく、Codexが改修に落とせる成果物を返させる。
6. 工程増殖、講義化、全自動化の暴走を防ぐ。

## 5視点の結論

| 視点 | 要点 |
|---|---|
| 編集長 | Fable Fiveは実装者ではなく設計レビュー担当。返答形式を固定し、既存STEPへの最小差分を求める。 |
| スレ職人 | 情報を削るのではなく、知識を揉めごとの燃料として出す。知識ニキは講師ではなく集合知。 |
| パイプライン設計 | 既存STEP差分マップ、仮予約ライフサイクル、モード表、早期品質ゲート、スキル別改修仕様を必須成果物にする。 |
| 反対派 | 「再設計」より「既存パイプラインを壊さないMVP差分レビュー」と呼ぶ。新STEP・新DB・新常駐ジョブは禁止寄り。 |
| QA | 感想ではなく、段階別検査表・差し戻し先・重複防止データ設計・テストケースを納品させる。 |

## 統合方針

Fable Fiveへの依頼は「新パイプラインを作って」ではなく、既存パイプラインを壊さず、次の思想を最小差分で組み込むレビューにする。

> 仮予約 -> スレで振る -> 生成後に確定 -> 解説で短く回収

## 最終貼り付け文

```text
あなたは実装者ではなく、2ch歴史チャンネル用「オリジナルスレ先生」スキル/パイプラインの設計レビュー担当です。

目的は、新パイプラインを一から作ることではありません。
既存の creative-thread-pipeline / creative-thread-gen / youtube-script-parts / script-third-party-review を尊重し、
通常制作で毎回回る軽さを保ったまま、次の設計思想を既存STEPへ最小差分で組み込むことです。

設計思想:
「仮予約 -> スレで振る -> 生成後に確定 -> 解説で短く回収」

最重要方針:
- メインは2ちゃん風スレの掛け合い
- 解説パートは補足であり、スレ本文の要約・復唱・言い直しではない
- スレ本文を薄くしない
- 知識ニキを講師化しない
- STEPを増やしすぎない
- STEP7第三者レビューで初めて問題が見つかる運用を避け、STEP2〜4.5で前倒し検知する

現状の問題:
- スレ本文で説明しすぎて、解説パートが同じ話になる
- 解説対象を「難読固有名トップN」で選ぶと、視聴者に必要な補足とズレる
- 前半と後半で同じ驚き・論点を繰り返す
- 話者配置や掛け合いの流れが崩れる
- 解説後に会話へ自然に戻れていない
- STEP7で構造的な問題が見つかると差し戻しが遅い

スレ本文の作法:
- スレ本文の主役は情報ではなく、住民同士の反応・誤解・反論・茶々・脱線である
- 史実や知識カードは、講義ではなく「揉めごとの燃料」として出す
- 知識ニキは1人の全知ナレーターにせず、複数の半可通知識人の集合知として設計する
- 本文内で結論・背景・意味づけを全部閉じない
- スレでは「振る」「揉める」「半分だけ見える」までに留め、生成後に残った補足価値だけを解説で回収する

禁止:
- 新しい大工程を増やすこと
- 新規スキル、新規常駐ジョブ、新規DB、新規巨大台帳を提案すること
- 全自動化ロードマップに広げること
- 「先生」要素を講義パート化・教材化・百科事典化すること
- 看板回だけの重い品質追求を通常運用に混ぜること
- 生成前に解説内容を細かく確定し、それに本文を従属させること

MVPの前提:
1. 仮予約
   生成前に「スレ中で自然に触れたい論点・用語」を3〜5個だけ仮置きする。
   この時点では解説対象として確定しない。

2. スレで振る
   creative-thread-gen の生成指示に、
   「仮予約した論点を説明ではなく、住民の疑問・ツッコミ・脱線として自然に出す」
   という方針を入れる。

3. 生成後に確定
   生成されたスレ本文を読んで、実際に会話として立ったものだけ解説対象にする。
   出なかったものは破棄する。無理に解説で回収しない。

4. 解説で短く回収
   解説は各スレ最大2本、1本150〜220字程度を目安にする。
   目的は視聴者が迷子にならない最低限の補助であり、教科書化しない。

出力してほしいもの:
```
