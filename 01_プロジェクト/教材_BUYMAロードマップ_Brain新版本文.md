### 0-1 この教材で進む4段階

この教材は、BUYMAの作業を次の4段階に分けています。

1. **登録・出品**: ルール、利益計算、商品ページの作り方を理解する
2. **仕入れ**: 国内、海外、直営店連携、VIPから、自分が実践する方式を選ぶ
3. **高値売り**: 安さだけではなく、商品・仕入れ・配送・見せ方の違いを作る
4. **外注化・AI活用**: 自分で確認できる作業だけを切り出し、下書きと整理を効率化する

章頭画像は、この4段階を「リサーチ、仕入れ、価格設計、高値売り、外注化、AI活用」の6作業へ細分化したものです。最初から全作業を同時に始める必要はありません。

### 0-2 正しい読み順

| 段階 | 読む章 | この段階で決めること |
|---|---|---|
| 基礎 | 第0〜1章 | 公式ルール、総コスト、初出品の確認方法 |
| 仕入れ | 第2〜6章 | 各方式を理解し、最初に実践する方式を1つ選ぶ |
| 高値売り | 第7章 | 候補商品の差別化理由と価格根拠 |
| 採用・教育 | 第8章 | 委託する業務、募集サービス、契約、教育 |
| 実践資料 | 第9章 | 作業に使うSOP、管理表、チェック表、文例 |
| 運営設計 | 第10章 | 承認、停止条件、証憑、権限、引継ぎ |
| AI活用 | 第11〜12章 | AIへ渡す資料、人が確認する項目、5つのプロンプト |

第2〜5章は、すべての仕入れ方法を一度理解したうえで、今回実践する方式を1つ選びます。理解していない方式を飛ばしてよいという意味ではありません。

### 0-3 日数ではなく完了条件で進む

本教材では「30日で外注化」「何日目までに何品」といった固定日程を設定しません。前の段階の確認が終わってから次へ進みます。

| 次へ進む前の確認 | 完了条件 |
|---|---|
| 第1章から仕入れ編へ | 根拠資料と総コストを確認した商品ページの下書きが1件ある |
| 仕入れ編から第7章へ | 選んだ仕入れ方式で、商品・買付先・配送・関税・利益を説明できる |
| 第7章から第8章へ | 自分で一度行い、正常時と異常時の判断を説明できる作業がある |
| 第10章からAI編へ | AIへ渡してよい情報と、人が承認する情報を分けられる |

この表は学習順を整理するための商品設計です。売上や利益の達成時期を保証するものではありません。

### 0-4 最初に守る5つのルール

#### 1. 買付先を確認する

BUYMAでは、国内EC、CtoCサービス、品質や真贋に不安のある買付先など、利用できない買付先があります。禁止・要注意一覧は随時更新されるため、教材内のショップ名だけで判断しません。

[BUYMA公式: 出品を禁止されている買付先](https://qa.buyma.com/shopper/prohibited-item/4007.html)

#### 2. 画像の権利を確認する

ブランドやECサイトなど、第三者が著作権を持つ画像は、加工しても無断掲載できません。自分が権利を持つ画像、または利用許諾を得た画像を使い、必要に応じて許諾の根拠を提出できる状態にします。

[BUYMA公式: 出品画像による権利侵害の禁止](https://qa.buyma.com/shopper/prohibited-item/15733.html)

#### 3. 出品者責任で検品する

商品は、注文内容、色、サイズ、数量、付属品、汚れ、傷、破損などを確認してから購入者へ発送します。出品者責任での検品を経ない買付先からの直送は禁止されています。

[BUYMA公式: 検品・梱包・発送で気をつける点](https://qa.buyma.com/shopper/order-to-delivery/4201.html)

#### 4. 事実だけを表示する

商品と関係のないキーワード、誤ったカテゴリ・ブランド・テーマ、虚偽や誤解を招く説明・画像、許可のない著名人画像は使いません。「限定」「日本未入荷」「即発」なども、その条件を確認できる場合だけ使います。

[BUYMA公式: 出品時の禁止事項](https://qa.buyma.com/shopper/prohibited-item/11394.html)

#### 5. アカウントの責任は登録者にある

作業の一部を外部へ委託することや、パートナーがログインすることは一律に禁止されていません。ただし、パートナーの操作を含むアカウント上の全行為は登録者の責任です。利用制限歴のある人への委託も避けます。

[BUYMA公式: 出品者登録に関するよくある質問](https://qa.buyma.com/shopper/reg/13732.html)

### 0-5 数値・リスト・ツールの読み方

教材には、元教材が作られた当時の価格、利益率、報酬、ショップ、配送日数、サービス名が含まれていました。新版では次のように扱います。

| 表示 | 意味 | 実践時の行動 |
|---|---|---|
| 現行値 | 公式情報で確認した数値 | 記載された確認日と公式URLを確認する |
| 当時例 | 元教材で実際に使われていた例 | 現在の標準・相場とは扱わず、自分の条件で再計算する |
| 未確認 | 営業・正規性・規約適合を確認できていない情報 | 確認が終わるまで使わない |
| 保留 | 原本や適法性を確認できていない手順 | 推測で実行しない |

自動出品など、BUYMAが許諾していない外部プログラムやサーバーへ負荷をかけるツールは使いません。

[BUYMA公式: 自動出品ツール等への注意](https://qa.buyma.com/shopper/tips/30118.html)

### この章のゴール

この章の完了状態は、出品数や売上ではありません。

> **買付先、画像権利、商品情報、総コスト、変更できない項目を確認した商品ページの下書きが1件あること。**

実際に公開する前に、仕入先資料と入力内容をもう一度照合します。

### 1-1 BUYMAと無在庫販売の基本

BUYMAでは、商品を出品し、注文が入った後に買付を行う運営が可能です。先に大量の在庫を持つ必要はありませんが、在庫切れ、価格変動、配送遅延、返品・再送などのリスクがなくなるわけではありません。

基本の流れ:

1. 商品と買付予定先を確認する
2. 商品ページを作る
3. 注文後に在庫と条件を再確認する
4. 商品を買い付ける
5. 注文内容と品質を検品する
6. 追跡できる方法等、商品ページで約束した条件で発送する
7. 発送通知と取引対応を行う

初心者が最初に整えるのは、大量出品ではなく、1件の取引を正しく追跡できる状態です。

### 1-2 出品者登録の条件

2026年7月13日にBUYMA公式ガイドで確認できた一般出品者の必須条件は次の3つです。

- 満20歳以上
- 日本語で購入者と円滑にやり取りできる
- 日本へ商品を発送できる

一般出品者は個人登録です。法人で活動する場合は、一般出品者とは別にショップ登録の案内を確認します。居住国によって事前の登録や届出が必要な場合もあります。

[BUYMA公式: 出品者になるための条件](https://qa.buyma.com/shopper/reg/2303.html)

### 1-3 正規の登録手順

元教材の「BUYMA登録とパッチテストマニュアル」からは、本人が自分のアカウントを正規登録する部分だけを使います。外注候補者に新規アカウントを作成・出品・退会させるパッチテストは行いません。

現在の公式手順:

1. BUYMAの会員登録を完了する
2. ログインしてマイページを開く
3. 会員情報から「パーソナルショッパー登録」を選ぶ
4. 基本情報、プロフィール、PR情報を入力する
5. 入力内容を確認して登録を完了する
6. 成約代金の振込先口座を登録する

同じ入力画面を長時間開いたままにすると内容が消える場合があるため、プロフィールとPR文は事前に事実だけで下書きしておきます。

[BUYMA公式: 出品者登録方法](https://qa.buyma.com/shopper/reg/2301.html)

### 1-4 プロフィールとPR情報

元教材には、アパレル経験、海外経験、コネクションなどを含むプロフィール例がありました。自分にない経歴や関係性は使いません。

事実確認用の下書き:

```text
【活動について】
[実際の居住地・対応地域]から活動しています。

【取り扱い】
[実際に調査・買付できるブランドやカテゴリ]を中心にご案内します。

【ご注文前】
在庫と買付条件は変動するため、ご注文前に確認いたします。

【検品・発送】
注文内容と商品の状態を確認し、商品ページでご案内した方法で発送します。

【お問い合わせ】
確認が必要な内容は、買付先等へ確認したうえで回答します。
```

書いてよいのは、現在の自分が説明・証明できる内容だけです。「日本未入荷」「入手困難」「正規品保証」「関税込み」なども、商品や取引条件を確認してから使います。

### 1-5 開始前に準備するもの

- BUYMA専用のメールアドレス
- 成約代金を受け取る本人名義等の対応口座
- 自分の資金計画と買付先に合う決済手段
- 利益計算と出品管理の表
- 注文明細、レシート、検品画像、発送伝票の保管場所
- 自分が権利を持つ画像、または利用許諾を確認できる画像
- 問い合わせと発送期限を確認する連絡環境

特定のカードや銀行を、全員に最適なものとして推奨しません。年会費、海外利用、為替手数料、締日、支払日、利用可能枠、繰上返済の条件を各社公式情報で比較します。生活費や納税資金を買付に流用しない資金計画も必要です。

### 1-6 手数料・上限・利益計算

#### 2026年7月13日の現行値

| 項目 | 一般出品者向け公式案内 |
|---|---|
| 出品料 | 無料 |
| 成約手数料 | 商品価格の5.5〜7.7%税込。一般出品者は原則7.7% |
| 振込手数料 | 海外口座1,000円、国内口座385円、楽天銀行220円 |
| 販売価格 | 500〜9,000,000円 |
| 出品数上限 | 5,000品 |
| 100品以上の出品 | 特定商取引法上の販売事業者に該当 |

成約手数料は、ポイントやクーポンによる割引前の商品価格に料率を掛けて計算されます。自分のアカウントに表示される料率も確認してください。

- [BUYMA公式: 出品者側の手数料](https://qa.buyma.com/bm/usage-fee/4503.html)
- [BUYMA公式: 出品価格・出品件数の上限](https://qa.buyma.com/shopper/sell/4102.html)

#### 1件ごとの総コスト

```text
仕入商品代
+ 現地税・買付先送料
+ 国際送料
+ 関税・輸入消費税・立替手数料（出品者負担の場合）
+ 決済・為替コスト
+ 国内送料
+ 梱包費・外注費
+ 振込手数料の配賦額
+ 返品・再送等に備える金額
= 成約手数料を除く総コスト
```

一般出品者の成約手数料を`r`、成約手数料を除く総コストを`C`、確保したい利益を`P`とすると、確認用の販売価格は次の式で逆算できます。

```text
確認用販売価格 = (C + P) ÷ (1 - r)
```

端数を調整したら、実際の販売価格でもう一度、成約手数料と利益額を計算します。元教材の「利益率20%」を標準値として固定しません。

### 1-7 初出品までの5ステップ

#### STEP 1 アカウントとプロフィールを準備する

本人の正しい情報で登録し、プロフィールとPR情報を事実だけで整えます。

#### STEP 2 商品と買付先を確認する

次を証拠と一緒に保存します。

- 買付先名、運営主体、商品URL
- ブランド、商品名、型番
- 色、サイズ、素材、付属品
- 商品価格、税、送料、返品条件
- 在庫確認日
- 禁止・要注意買付先との照合結果

マーケットプレイスの場合は、サイト名だけでなく実際の販売元まで確認します。

#### STEP 3 利用できる画像を準備する

商品画像は最大20枚まで掲載できます。ただし、枚数よりも権利と正確さを優先します。

- 自分で撮影し、自分が著作権を持つ画像
- 第三者からBUYMA掲載の利用許諾を得た画像
- BUYMAの正規機能で利用条件を満たす画像

ブランドやECサイトの画像は、保存・加工しただけでは使えません。許諾を得た場合は、その根拠も保管します。

#### STEP 4 商品ページを入力する

画面の並びは変わることがあります。項目名が違う場合は、BUYMAの現在の入力画面と公式ガイドを優先します。

| 項目 | 入力時の確認 | 出品後 |
|---|---|---|
| 商品画像 | 権利、商品一致、色味、付属品 | 追加・削除・並び替え可能 |
| 商品名 | ブランド、商品名、型番等の事実 | 変更不可。誤りは再出品 |
| 商品コメント | 仕様、付属品、配送、注意事項 | 変更可能 |
| カテゴリ | 実際の商品カテゴリ | 変更不可。誤りは再出品 |
| ブランド | 正式なブランド | 変更不可。誤りは再出品 |
| シーズン・テーマ・タグ | 条件を満たすものだけ選ぶ | 変更可能 |
| 色・サイズ・在庫 | 仕入先情報と照合 | 変更可能 |
| 発送方法 | 追跡、送料、日数等を実態どおり設定 | 変更可能 |
| 購入期限 | 在庫・買付条件に合わせる | 変更可能 |
| 買付地 | 実際の買付予定地 | 変更不可。誤りは再出品 |
| 発送地 | 実際の発送地 | 変更不可。誤りは再出品 |
| 商品価格 | 全コスト、手数料、利益を反映 | 変更可能 |
| 参考価格 | 公式な根拠がある場合だけ入力 | 現行条件を再確認 |
| 関税 | 発送地と負担者を正確に設定 | 取引条件と一致させる |
| 型番・管理メモ | 商品特定と内部管理に使う | 購入者表示との違いを確認 |

[BUYMA公式: 出品後に変更できる項目](https://qa.buyma.com/shopper/sell/4107.html)

#### STEP 5 公開前確認と出品審査へ対応する

公開前に、商品ページと仕入先資料を上から順に照合します。確認者が別にいる場合も、最終責任は登録者にあります。

BUYMAの条件に一致した商品は自動的に「出品審査中」となり、審査には最大4営業日かかります。追加確認では、登録情報、出品商品・方法、買付先情報などの提示を求められる場合があります。

レシートや注文明細等は、BUYMA公式案内に従い取引完了から1年程度保管します。未買付の場合は、買付予定先を説明できる資料を残します。

[BUYMA公式: 出品審査](https://qa.buyma.com/shopper/sell/12609.html)

### 1-8 商品コメントを事実と提案に分ける

商品コメントでは、確認できる事実と、着こなし等の提案を混同しません。

```text
【商品情報】
ブランド:
商品名・型番:
色:
サイズ:
素材・仕様:
付属品:
情報確認元:

【ご案内】
在庫確認:
買付予定先:
発送地・発送方法:
お届け目安:
関税負担:

【スタイリング等の提案】
確認できる商品仕様を前提に、断定せず記載:
```

次の表現は根拠なしに使いません。

- 限定、日本未入荷、新作、完売、残り1点
- 大人気、入手困難、希少
- 著名人愛用、雑誌掲載
- 正規品保証、関税なし、返品不可

商品と関係のないキーワードを並べる行為や、誤ったカテゴリ・ブランド・テーマへの出品も禁止されています。

### 1-9 初心者のリサーチ

元教材では、他の出品者を参考に商品を探す方法が扱われていました。新版では、文章・画像・商品構成をそのまま真似るのではなく、観察した事実と自分の仮説を分けます。

| 区分 | 記録する内容 |
|---|---|
| 観察できる事実 | ブランド、カテゴリ、販売価格、取引表示、問い合わせ、配送条件等 |
| 仮説 | なぜ選ばれた可能性があるか |
| 自分で確認する事実 | 買付先、在庫、総コスト、画像権利、配送、規約 |
| 判断 | 出品候補／保留／見送りと、その理由 |

他の出品者の販売価格だけを見て、その人の利益額を断定することはできません。仕入値、割引、税、送料、返品、外注費などが見えないためです。

- 商品URLと確認日
- 買付先と販売元
- 商品識別情報
- 在庫と仕入総額
- 最低販売価格
- 需要理由の仮説
- 画像の利用根拠
- 未確認事項

### この章のゴール

この章では、国内の実店舗で買い付ける1件について、次を説明できる状態を作ります。

> **商品、店舗、在庫確認、総コスト、検品、発送、証憑、精算の流れを1つの管理記録へまとめ、実行可否を判断できること。**

実取引がまだない場合は、候補1件を使って判断項目を埋めます。レシート取得や発送通知など、注文後にしか行えない作業を章の学習完了条件にはしません。

### 2-1 国内買付の対象

本章で扱う国内買付は、**日本国内の実店舗で商品を買い付ける方法**です。

BUYMAでは、主に日本人向けで、運営拠点と発送地が日本国内にある国内ECサイトからの買付が禁止されています。楽天市場、Amazon.co.jp、Yahoo!ショッピング、Qoo10などが公式ガイドの例に含まれています。メルカリ、ラクマ、ヤフオクなどのCtoCサービスも買付先にできません。

[BUYMA公式: 出品を禁止されている買付先](https://qa.buyma.com/shopper/prohibited-item/4007.html)

国内ブランドの公式オンラインサイトは、型番・仕様・店舗在庫等の確認に使う場合がありますが、本章の買付は実店舗で行います。オンラインで注文する場合や、店舗から商品を発送してもらう場合は、第4章の検品経路とBUYMAの現行ルールも確認します。

### 2-2 国内実店舗買付の強みと注意点

#### 強みになり得ること

- 商品を直接確認してから買い付けられる
- 国内発送までの工程を把握しやすい
- 店舗限定、地域限定、国内完売等の商品を扱える場合がある
- 店舗の取り置き、返品、交換、付属品の条件を確認しやすい
- 海外配送や為替変動がない取引では、経費項目を整理しやすい

#### 必ず確認すること

- 同じ商品への出品が多い場合は価格競争になりやすい
- 交通費、買付時間、配送料、梱包費、報酬を入れると利益が残らない場合がある
- 取り置き、購入制限、電話注文、転売、返品・交換の条件は店舗ごとに異なる
- 国内定価より高い価格でも売れることはあるが、必ず売れるわけではない
- 国内発送でも、商品ページの発送地・配送方法・到着目安を実態どおりに設定する

元教材には初心者向け価格帯として2万〜6万円の例がありましたが、現在の推奨価格や成功条件として固定しません。自分の資金、総コスト、需要、返品リスクを商品ごとに確認します。

### 2-3 国内買付の3STEP

#### STEP 1 リサーチして候補を作る

1. BUYMA内で、国内買付で動いているブランド、カテゴリ、商品、価格、配送条件を観察する
2. 商品名、型番、画像等から商品を特定し、ブランド公式情報と照合する
3. 自分または契約済みパートナーが訪問できる国内実店舗を確認する
4. 店舗での取扱、在庫確認方法、取り置き、購入制限、返品・交換条件を調べる
5. 商品代、交通費、配送、梱包、報酬、BUYMA手数料を含めて利益を計算する

商品名や型番で検索しても特定できない場合、画像検索は商品識別の補助に使えます。ただし、検索で見つけた第三者画像を出品画像として保存・転載してはいけません。

![国内ショップを選ぶ判断基準](BUYMA本文挿絵/D02_第2章_国内ショップを選ぶ判断基準.png)

#### STEP 2 問い合わせ・受注後に在庫を確認する

在庫確認では、商品を取り違えないための識別情報と、買付条件を分けて確認します。

```text
お世話になっております。

[ブランド名]の下記商品について、店頭在庫を確認したくご連絡しました。

商品名:
型番:
カラー:
サイズ:

在庫の有無、取り置きの可否と期限、付属品、購入制限、返品・交換条件を教えていただけますでしょうか。
```

店舗への連絡時に、BUYMA出品に関係のない虚偽の身分・購入目的を伝えません。店舗が転売や代理購入を禁止している場合は、その条件に従います。

確認結果は次のように記録します。

| 項目 | 記録内容 |
|---|---|
| 店舗 | 正式名称、所在地、公式連絡先 |
| 確認日時 | 電話・訪問・公式在庫情報を確認した日時 |
| 商品 | ブランド、商品名、型番、色、サイズ |
| 在庫 | 在庫あり／なし、数量、再確認が必要な時点 |
| 取り置き | 可否、期限、名義、受取条件 |
| 購入条件 | 購入制限、支払方法、本人確認等 |
| 商品状態 | 新品、展示品、訳あり等 |
| 付属品 | 箱、保存袋、タグ、保証書等 |
| 返品・交換 | 可否、期限、条件 |

#### STEP 3 買付・検品・発送する

1. 注文内容、在庫、買付上限を再確認する
2. 実店舗で商品を買い付け、レシート等の購入証明を受け取る
3. 型番、色、サイズ、数量、付属品を注文内容と照合する
4. 汚れ、傷、ほつれ、破損、機能不良を確認する
5. 日付や取引IDと商品状態が分かる検品画像を残す
6. 商品が動いたり傷ついたりしないよう梱包する
7. 商品ページで約束した配送方法で購入者へ発送する
8. 購入者宛てに発送した後、追跡番号等を確認して発送通知する

発送していないのに通知する行為や、店舗から出品者・パートナーへの配送開始時点で購入者向け発送通知を行う「仮発送」は禁止されています。

### 2-4 店舗・商品・総コストの判断

候補商品は、次の5項目をすべて確認してから出品判断を行います。

| 判断項目 | 確認すること | 見送り・保留の例 |
|---|---|---|
| 店舗情報 | 実在、公式連絡先、購入条件、証憑 | 出所や販売主体を確認できない |
| 商品確認 | 型番、色、サイズ、付属品、状態 | 同一商品と確認できない |
| 価格確認 | 商品代、交通費、送料、梱包、報酬、手数料 | 現在価格では必要利益が残らない |
| 在庫確認 | 在庫、取り置き期限、再確認時点 | 受注後に確保できる見込みがない |
| 発送確認 | 検品者、発送地、配送方法、期限 | 未検品直送になる、期限を守れない |

国内買付だからという理由だけで高い価格を設定しません。配送速度、商品状態、入手経路、付属品、問い合わせ対応など、購入者が確認できる違いと総コストの両方から判断します。

### 2-5 国内買付パートナーへ依頼する12工程

BUYMAでは、共に買付を行うパートナーが商品を検品することは認められています。ただし、パートナーの行為を含め、購入者へ問題のない商品を届ける責任は出品者にあります。買付先店舗のスタッフは買付パートナーとして認められません。

[BUYMA公式: 買付先から購入者への直送](https://qa.buyma.com/shopper/order-to-delivery/4205.html)

#### 1. 在庫確認を依頼する

管理番号、商品URL、ブランド、商品名、型番、色、サイズ、店舗、確認期限を伝えます。

#### 2. 在庫と買付条件を確認する

在庫、取り置き、商品状態、付属品、購入制限、返品・交換条件を記録してもらいます。

#### 3. 買付条件と前払を承認する

商品代、交通費上限、支払方法、精算方法、買付期限を書面または記録が残る方法で合意します。パートナーへ自己負担や自身のカード利用を当然の条件として求めません。

#### 4. 実店舗で買い付ける

依頼内容と現物を照合し、差異や異常があれば購入前に作業を止めて連絡してもらいます。

#### 5. 買付完了と証憑を報告する

レシート、購入日時、店舗、支払額、商品情報を報告してもらいます。不要なカード番号等は共有しません。

#### 6. パートナーが検品する

商品、付属品、状態の3系統を確認し、必要な検品画像を撮影してもらいます。

#### 7. 出品者が画像と注文内容を照合する

出品者が検品画像、注文情報、商品ページを確認し、問題がある場合は発送を止めます。

#### 8. 商品に合う方法で梱包する

薄葉紙、緩衝材、箱、テープ等は商品の素材・形・配送方法に応じて選びます。元教材の資材一覧を全商品共通の必須セットにはしません。

#### 9. 購入者へ発送する

出品時に設定した発送地と配送条件に従い、購入者の住所・氏名を確認して発送します。

#### 10. 追跡番号と発送証憑を報告する

発送日、配送会社、追跡番号、送料、発送控えを管理表へ記録します。

#### 11. 経費と報酬を精算する

前払金、商品代、交通費、送料、梱包資材、その他承認済み経費、契約した報酬を照合します。固定3,000円等を標準報酬として使いません。

#### 12. BUYMAで発送通知する

購入者宛ての発送が完了し、追跡情報等を確認した後に発送通知を行います。

### 2-6 パートナー募集は利用サービスごとに確認する

外部委託自体がBUYMAで一律禁止されているわけではありません。しかし、募集に使うサービス側が、代理購入やEC運営代行を禁止している場合があります。

クラウドワークスは、無在庫販売、ECの出品・商品登録・在庫管理代行、アカウント共有、注文・購入・受取・買付等の代理を禁止対象として明記しています。そのため、**クラウドワークスを国内買付パートナーの募集先として案内しません。** 規制を別表現で「すり抜ける」募集も行いません。

[クラウドワークス公式: 仕事依頼ガイドライン](https://crowdworks.jp/pages/guidelines/job_offer.htm)

募集方法、取引条件、契約、教育、個人情報、アクセス停止の詳細は第8章で扱います。本章では、すでに適切な手続きで契約済みのパートナーへ実務を依頼する流れだけを説明します。

### 2-7 検品・梱包・発送・精算

#### 検品

| 区分 | 確認内容 |
|---|---|
| 商品一致 | ブランド、商品名、型番、色、サイズ、数量 |
| 付属品 | 箱、保存袋、タグ、保証書、替え部品等 |
| 外観 | 汚れ、傷、穴、ほつれ、変色、型崩れ |
| 機能 | ファスナー、留め具、ボタン、電池、可動部等 |
| 表示 | 商品ページの画像・説明と異なる点がないか |

異常がある場合は発送せず、買付先と購入者への連絡、交換、返品、発送期限延長等を判断します。検品画像は、日付や取引IDが分かる状態で保管します。

#### 梱包

- 商品を清潔な状態で整える
- 素材や付属品が擦れないよう保護する
- 箱の中で商品が動かないよう隙間を埋める
- 水濡れ、潰れ、破損のリスクに合わせて資材を選ぶ
- 値札や商品タグは、BUYMA公式案内と商品の状態に合わせて扱う

#### 発送

- 注文住所・氏名と送り状を照合する
- 商品ページで設定した配送条件を守る
- 発送控えと追跡番号を保存する
- 発送期限は注文確定日時から18日後の同じ時刻であることを受注リストで確認する
- 期限内に発送できない場合は、期限前に購入者へ連絡し、同意が必要な延長申請を行う
- 期限経過後は延長申請や発送通知ができないため、日付だけでなく時刻まで管理する

#### 精算

```text
承認済み経費
= 商品代
+ 交通費
+ 配送料
+ 梱包資材
+ その他事前承認した費用

最終精算額
= 承認済み経費
- 前払金
+ 契約した報酬
```

金額だけでなく、レシート、発送控え、支払日も同じ管理番号へ紐付けます。

## 第3章 海外買付完全解説マニュアル

![第3章 海外買付完全解説マニュアル](BUYMA章頭挿絵/第3章_海外買付完全解説マニュアル.png)

![海外買付の基本フロー](BUYMA本文挿絵/A03_第3章_海外買付の基本フロー.png)

<!-- SOURCE: ROOT-03 / ROOT-12 / CHILD-05 / SHEET-OVS-01〜03 / 公式更新=本章末リンク -->

海外買付は、海外にいるパートナーへ丸投げする仕組みではありません。

**出品者が商品・価格・期限・買付先を決め、パートナーが決められた範囲で買付または受取、現物検品、梱包、発送を行い、出品者が証拠を確認して取引全体に責任を持つ仕組み**です。

本章のゴールは、候補商品1件について次の5点を説明できる状態になることです。

1. 店舗買付、オンライン買付、転送のどれを使うか
2. 誰が買付、受取、現物検品、発送を行うか
3. 為替や国際送料を含む仕入れ総額はいくらか
4. 購入者情報を海外へ渡す必要があるか
5. 関税負担と発送期限をどう案内・管理するか

### 3-1 店舗買付・オンライン買付・転送の違い

海外買付には、主に次の3方式があります。

| 方式 | 買付する人 | 受取する人 | 現物検品する人 | 向いている場面 | 主な注意点 |
|---|---|---|---|---|---|
| 店舗買付 | 海外パートナー | 海外パートナー | 海外パートナー | 実店舗の在庫を確認して買付したい | 移動費、店舗条件、買付上限、購入証明 |
| オンライン買付 | 海外パートナー | 海外パートナー | 海外パートナー | 現地オンラインストアから買付したい | 実際の販売者、注文名義、返品条件、配送遅延 |
| 転送 | 出品者 | 海外パートナー | 海外パートナー | 出品者が注文し、現地で受取・検品してほしい | 買付先規約、決済名義、配送先名義、受取拒否 |

ここでいう海外パートナーは、出品者と業務内容を合意し、現物を確認できる人です。店舗スタッフやオンラインショップの発送担当者は、出品者の買付パートナーには当たりません。

また、パートナーから届いた写真を出品者が確認することは大切ですが、**写真確認だけで現物検品を代替することはできません**。現物を手に取ったパートナーが検品し、出品者はその検品結果と証拠を確認します。

#### 方式を選ぶ5つの質問

1. その買付先はBUYMAで利用できるか
2. 買付先の規約上、購入名義・決済名義・配送先に問題がないか
3. 誰が現物を受け取り、検品するか
4. 返品・交換が必要になった時、誰がいつまでに動けるか
5. 為替、報酬、現地費用、国際送料、関税まで含めて利益が残るか

ブランドごとに販売経路や購入条件は異なり、同じブランドでも国、店舗、時期、商品によって変わります。「このブランドは必ず店舗買付だけ」と固定せず、注文時点の公式販売経路と店舗条件を確認してください。

### 3-2 海外買付のメリットとリスク

#### メリット

- 日本から購入しにくい現地在庫へ買付範囲を広げられる
- オンライン在庫だけでは見つからない商品を実店舗で探せる
- 複数国の在庫、価格、配送条件を比較できる
- 現地で商品を受け取り、購入者へ送る前に検品できる
- 継続取引により、店舗の入荷時期や購入条件を正確に確認しやすくなる場合がある

#### リスク

- 為替変動、送金手数料、国際送料、報酬で仕入れ総額が増える
- 現地税、関税、輸入消費税の扱いが取引経路で変わる
- 返品期限内に検品や連絡が間に合わないことがある
- 高額商品、配送事故、紛失、破損への備えが必要になる
- パートナーへ資金や購入者情報を渡すため、契約・権限・記録管理が必要になる
- 現地法、配送会社、決済会社、買付先の規約も確認する必要がある

海外買付の強みは「海外だから安い」ことではありません。**その商品を、許可された買付先から、検品可能な経路で、期限内に届けても利益が残ること**が判断基準です。

### 3-3 仕入れ総額を先に計算する

![仕入れ総額の考え方](BUYMA本文挿絵/D03_第3章_仕入れ総額の考え方.png)

商品価格だけを見て買付を決めると、為替差や国際送料で利益が消えます。依頼前に次を試算します。

```text
海外買付の仕入れ総額
= 商品代の円換算額
+ 現地税の円換算額
+ 現地送料または店舗までの交通費の円換算額
+ パートナー報酬の円換算額
+ 梱包資材費の円換算額
+ 国際送料・必要な補償
+ 決済・送金手数料
+ 出品者負担となる関税・輸入消費税等
+ 返品・再配送に備える予備費

見込利益
= 販売価格
- BUYMA成約手数料等
- 海外買付の仕入れ総額
- その他の販売関連費用
```

各費用は同じ通貨単位、ここでは円へ換算して合計します。カードや送金サービスの為替スプレッドは、参考レートへ別に足すのではなく、明細で確定した実際の円換算額または手数料として反映します。

試算時と確定時は分けて記録します。

| 段階 | 為替の扱い | 記録するもの |
|---|---|---|
| 依頼前 | 確認時点の参考レート | レート提供元、確認日時、通貨、想定手数料 |
| 注文時 | 決済時のレートまたは請求予定額 | 決済額、決済通貨、利用サービス、注文日時 |
| 精算時 | 実際に確定した円換算額 | 明細の確定額、送金手数料、着金額、差額 |

固定の為替率や関税率を管理表へ埋め込まず、取引ごとに根拠と確認日を残します。報酬も一律相場として決めず、業務範囲、移動、保管、高額商品の責任、通貨、税務上の扱いを確認して契約します。

### 3-4 依頼前に共通指示書を作る

3方式のどれを使う場合も、チャットだけで依頼を完結させません。1件につき1つの管理番号を付け、次を指示書と管理表へ記録します。

| 項目 | 記録内容 |
|---|---|
| 商品 | ブランド、商品名、URL、品番、色、サイズ、数量、状態 |
| 買付先 | 店舗・サイト名、所在地またはURL、実際の販売者 |
| 価格 | 表示価格、通貨、購入上限、許容しない追加費用 |
| 付属品 | 箱、保存袋、保証書等、商品ページ上のセット内容 |
| 期限 | 在庫回答、買付、受取、検品、発送の各期限 |
| 禁止事項 | 代替品の無断購入、上限超過、未検品発送、虚偽申告 |
| 証拠 | 注文確認、レシート、商品写真、発送控え、追跡番号 |
| 異常時 | 購入停止条件、返品期限、連絡先、判断を待つ状態 |

商品URLだけでは、色違い、サイズ違い、別販売者への切替を防げません。品番、色、サイズ、価格上限を文章で固定し、変更があれば購入前に再承認します。

### 3-5 店舗買付の標準フロー

店舗買付は、パートナーが現地店舗へ在庫確認し、店舗で商品を購入する方式です。

1. **出品者：在庫確認を依頼する**
   管理番号、店舗、商品URL、品番、色、サイズ、数量、価格上限、回答期限を伝えます。

2. **パートナー：店舗在庫を確認する**
   店舗名、確認日時、在庫、税込価格、購入条件、取置期限、担当者名を分かる範囲で記録します。

3. **出品者：買付可否と上限を最終承認する**
   注文状況、利益、発送期限を再確認します。承認前にパートナーへ購入させません。

4. **出品者：承認済み費用を準備する**
   商品代と合意した交通費等を、契約した方法で前払いします。パートナーの自己資金や個人カード利用を当然の前提にしません。

5. **パートナー：店舗で買付する**
   品番、色、サイズ、価格、付属品に加え、購入前に確認できる傷、汚れ、破損を照合します。異常がある場合、上限を超える場合、代替提案がある場合は購入を止めて確認します。

6. **パートナー：購入完了を報告する**
   購入日時、支払額、通貨、レシート等の購入証明、返品・交換期限を共有します。

7. **パートナー：現物を検品する**
   商品、付属品、汚れ・傷・ほつれ・破損を確認し、管理番号が分かる状態で写真を撮ります。

8. **出品者：検品証拠を確認する**
   指示内容と画像・報告が一致するか確認します。不鮮明、撮影不足、異常ありの場合は発送を止めます。

9. **出品者：発送を承認する**
   配送方法、補償、申告内容、発送先を確定します。購入者直送の場合は、この段階で必要最小限の配送先情報だけを共有します。

10. **パートナー：梱包して発送する**
    商品特性に合う梱包を行い、追跡可能な方法で確定した発送先へ発送します。

11. **パートナー／出品者：発送を記録・通知する**
    パートナーは発送控え、追跡番号、発送日時を共有します。出品者は購入者宛ての発送を確認してからBUYMAで発送通知します。

12. **出品者：経費と報酬を精算する**
    前払金、実費証憑、為替、追加費用、契約報酬を管理番号単位で照合し、差額を精算します。

#### 店舗スタッフとの関係づくり

継続して同じ店舗へ問い合わせる場合は、丁寧で正確なやり取りを積み重ねます。

- 商品名だけでなく品番、色、サイズを伝える
- 営業時間、連絡手段、取置条件を守る
- 買わない場合も放置せず、不要になったことを連絡する
- 在庫の優遇や取り寄せを当然の権利として要求しない
- 店舗スタッフへBUYMAの検品責任を負わせない

良好な関係は在庫確認を円滑にする可能性がありますが、在庫確保や特別対応を保証するものではありません。

### 3-6 オンライン買付の標準フロー

オンライン買付は、海外パートナーが現地オンラインストアで注文し、自宅等で受け取る方式です。

1. **出品者：商品と買付先を指定する**
   URL、実際の販売者、品番、色、サイズ、価格上限、配送先、期限を指示します。

2. **パートナー：注文条件を確認して報告する**
   在庫、総額、送料、返品条件、注文名義、決済名義、配送可能地域を確認して出品者へ報告します。マーケットプレイスではサイト名だけでなく実際の販売者も確認し、出品者が内容を照合して最終承認します。

3. **出品者：承認済み費用を準備する**
   契約に従って必要額を前払いし、通貨、送金額、着金予定額、手数料を記録します。

4. **パートナー：指定商品を注文する**
   品番、色、サイズ、数量、価格を注文確定画面で再確認し、無断で類似品や別販売者へ変更しません。

5. **パートナー：注文情報を報告する**
   注文番号、注文日時、総額、到着予定日を共有し、買付先の発送後に配送会社と買付先→パートナー追跡番号を追記します。

6. **パートナー：商品を受け取る**
   外装の破損や個数を確認し、受取日時を記録します。異常があれば開封前後の状態を撮影します。

7. **パートナー：現物を検品する**
   商品、付属品、汚れ・傷・ほつれ・破損を確認し、管理番号が分かる状態で写真を撮ります。

8. **出品者：検品証拠を確認する**
   指示との一致、画像、返品期限を確認します。問題があれば購入者へ送らず、買付先への連絡方法を決めます。この時点ではまだ発送を指示しません。

9. **出品者：発送条件と発送先を確定し、発送を承認する**
   配送方法、補償、申告内容を決め、購入者直送の場合は必要最小限の配送先情報を共有してから発送を承認します。

10. **パートナー：梱包して発送する**
    商品に合う梱包を行い、追跡可能な方法で発送します。

11. **パートナー／出品者：発送を記録・通知する**
    発送控え、パートナー→購入者追跡番号、発送日時を共有し、出品者が実際の発送を確認してからBUYMAで発送通知します。

12. **出品者：経費と報酬を精算する**
    前払金、実際の注文額、送料、為替差、証憑、契約報酬を照合して差額を精算します。

### 3-7 転送の標準フロー

転送は、出品者が商品を注文し、海外パートナーが受取、検品、梱包、発送を担当する方式です。

1. **出品者／パートナー：受取可否を確認する**
   商品、サイズ、数量、予定日、保管期間を伝え、パートナーが受け取れることを確認します。

2. **出品者：買付先と注文条件を確認する**
   BUYMAで利用できる買付先か、買付先規約上の注文名義、決済名義、配送先名義、転送可否に問題がないか確認します。

3. **出品者：商品を注文する**
   パートナーの承諾を得た配送先へ注文し、商品URL、品番、色、サイズ、数量、注文番号、到着予定日を記録します。

4. **出品者：配送情報を共有する**
   配送会社、買付先→パートナー追跡番号、送り主名、荷物数を共有します。BUYMAのログイン情報や決済情報は共有しません。

5. **パートナー：商品を受け取る**
   受取日時、荷物数、外装状態を記録し、異常があれば写真で報告します。

6. **パートナー：現物を検品する**
   商品、付属品、汚れ・傷・ほつれ・破損を確認し、管理番号が分かる状態で写真を撮ります。

7. **出品者：検品証拠を確認する**
   指示内容と画像・報告を照合し、返品期限も確認して発送可否を判断します。

8. **出品者：発送を承認する**
   配送方法、補償、申告内容、発送先を確定し、必要最小限の配送先情報を共有します。

9. **パートナー：梱包して発送する**
   商品に合う梱包を行い、追跡可能な方法で発送します。

10. **パートナー／出品者：発送を記録・通知する**
    発送控え、パートナー→購入者追跡番号、発送日時を共有し、出品者が実際の発送を確認してからBUYMAで発送通知します。

11. **出品者：経費と報酬を精算する**
    現地で発生した送料、資材費等の証憑と契約報酬を照合し、管理番号単位で精算します。

### 3-8 相互検品・梱包・追跡付き発送

#### 相互検品の役割

相互検品は、次の2段階で行います。

1. **パートナーの現物検品**: 実物を手に取り、商品・付属品・状態を確認する
2. **出品者の証拠確認**: 指示書と報告・写真が一致しているか確認し、発送可否を決める

#### 現物検品の3項目

| 項目 | 確認内容 |
|---|---|
| 商品 | ブランド、品番、色、サイズ、数量、左右・セット内容 |
| 付属品 | 商品ページに記載した箱、保存袋、説明書、保証書等 |
| 状態 | 汚れ、傷、シミ、穴、ほつれ、破損、金具、ファスナー、におい等 |

商品特性に応じて、靴の左右とサイズ、衣類の縫製、バッグの内外、時計の動作等を追加します。真贋判断を写真だけで断定せず、まず買付先と購入証明を確実に管理します。

#### 検品写真

- 商品全体の前面・背面
- 品番、サイズ、色を確認できるタグや表示
- 付属品一式
- 傷や汚れが疑われる部分の近接写真
- 外箱や包装に破損がある場合の状態
- 管理番号または撮影日が分かる状態

レシート画像を共有する時は、検品や精算に不要なカード番号、会員番号、住所等を隠します。

#### 梱包

- 商品と付属品が擦れないよう個別に保護する
- 水濡れ、潰れ、衝撃に合わせて資材を選ぶ
- 箱の中で商品が動かないよう隙間を埋める
- 外箱やブランド箱を商品の一部として案内した場合は、その状態も保護する
- 梱包後の重量と寸法を確認し、送料見積との差を記録する

基本の順序は次のとおりです。

1. 衣料品は形を整えて畳み、薄葉紙や保護袋等で包む
2. 商品特性に応じて気泡緩衝材で保護する
3. 適切な大きさの段ボールへ緩衝紙を敷いて商品を入れる
4. 商品が動かないよう隙間へ緩衝材を入れる
5. 段ボールの合わせ目と両端をH字になるようテープで閉じる

#### 国際発送

- 配送先の氏名、住所、郵便番号、電話番号を送り状と照合する
- 発送人の氏名、住所、電話番号も正しく記載し、偽名や架空情報を使わない
- 追跡できる配送方法を使い、商品価値とリスクに応じて補償を検討する
- 購入者が選んだ配送方法を変更する場合は、発送前に購入者の了承を得る
- 海外発送での郵便局留めは避け、購入者が指定した住所へ発送する
- 配送会社の禁制品、危険物、素材・容量制限を確認する
- BUYMAで禁止されている船便を使わない
- 税関申告では品名、数量、価格等を事実どおり記載し、過少申告をしない
- 発送控え、追跡番号、申告書類の控えを管理番号へ紐付け、発送伝票は6か月間保管する
- 発送通知は、購入者宛ての荷物を配送会社へ実際に引き渡した後に行う

BUYMA上の発送期限は、受注リストに表示される注文確定日時から18日後の同じ時刻です。買付先からパートナーへの配送やパートナーの受取が遅れ、購入者宛ての発送が期限に間に合わない見込みの場合は、期限前に購入者へ状況を説明し、同意を得たうえで延長申請します。期限経過後は延長申請や発送通知ができないため、日付だけでなく時刻まで管理してください。

### 3-9 送金・前払・精算の管理

特定の送金サービスを教材の固定推奨にはしません。利用国、受取国、通貨、取引目的、本人確認、上限、手数料、為替、着金時間、返金条件は変わるため、取引時点の公式条件を比較します。

#### 送金前の確認項目

- 送金者と受取人の契約上の氏名
- 送金元国、受取国、通貨、業務上の送金目的
- 送金額、表示手数料、為替レート、相手の着金予定額
- 着金予定日と買付・取置期限
- キャンセル、組戻し、返金時の扱い
- 利用規約上、業務委託費や商品代の送金に使えるか
- 現地の法令・税務・外為規制上の確認事項

#### 管理表へ残す記録

| 項目 | 記録内容 |
|---|---|
| 送金 | サービス名、送金日時、送金額、通貨、手数料、取引番号 |
| 為替 | 適用レート、レート確認日時、実際の円換算額 |
| 着金 | 着金日時、受取額、通貨、差額 |
| 支出 | 商品代、交通費、送料、資材費、その他承認済み費用 |
| 証憑 | 注文確認、レシート、明細、発送控え |
| 精算 | 前払金、実費、返金、契約報酬、最終支払額、支払日 |

```text
最終精算額
= 承認済み実費
- 前払金
+ 契約した報酬
```

パートナーの立替を認める場合は、上限、対象費目、事前承認、証憑、返済日を契約で決めます。高額商品を個人の自己資金やカードで先に買わせる運用を標準にしないでください。

本人確認が必要な場合も、パスポート画像をお互いに送り合うことを標準手順にはしません。契約・決済・本人確認サービスの正規手続を使い、取得目的、必要項目、保管場所、閲覧者、削除時期を決めます。

### 3-10 購入者情報を海外パートナーへ渡す時

海外パートナーから購入者へ発送する場合、氏名、住所、郵便番号、電話番号等を海外へ渡すことがあります。これは単なる発送作業だけでなく、個人データの外国にある第三者への提供に関する確認が必要になり得ます。

個人情報保護委員会の現行ガイドラインを確認し、少なくとも次を決めてから運用します。

1. 提供先の国・地域と受取人
2. 提供する項目と発送に必要な理由
3. 本人同意、同等水準と指定された国・地域、提供先が相当措置を継続的に講ずる体制、法定例外のうち、どの要件に基づくか
4. 再提供の禁止、目的外利用の禁止、秘密保持
5. 安全な共有方法と閲覧できる人
6. 保管期限と削除確認
7. 誤送信、紛失、不正利用が起きた時の連絡手順

相当措置を継続的に講ずる体制を根拠にする場合は、契約書や確認書を作るだけで終わりではありません。措置の実施状況や提供先国の制度を継続的に確認し、支障が生じた時の是正・提供停止、本人への情報提供等、ガイドライン上の義務を確認します。

外国にある第三者への提供は、提供先や安全管理措置等により扱いが変わります。「業務委託だから必ず同意不要」「契約書があれば必ず適法」と自己判断せず、運用開始前に公式ガイドラインを確認し、判断が難しい場合は個人情報保護委員会の相談窓口や専門家へ確認してください。

#### 共有を最小限にする

発送時に通常必要となるのは、氏名、郵便番号、住所、配送会社が求める電話番号、管理用の注文識別子です。

次の情報は発送担当者へ共有しません。

- BUYMAのID・パスワード・認証コード
- 購入者とのメッセージ履歴全体
- クレジットカード・銀行口座情報
- パスポート等の本人確認書類
- 発送に不要な生年月日、購入履歴、他の取引情報

公開リンクの表計算ファイルへ配送先を置かず、対象案件の担当者だけが期限付きで閲覧できる方法を使います。発送完了、返品可能期間、問い合わせ対応等で必要な期間を定め、不要になった情報は削除またはアクセス不能にします。

### 3-11 関税・輸入消費税・発送通知

#### 購入者へ海外から直接発送する場合

BUYMAでは、海外発送の商品に関税等が発生した場合、通常は購入者が輸入者として支払います。ただし、商品ページで「関税込み」または関税返金対象として案内した取引は、最終的に出品者が負担します。

関税等は必ず発生するものではなく、発生の有無と金額は税関が判断します。原産国、品目、素材、価格、数量等で変わるため、教材内の一律税率で購入者へ確約しません。輸入消費税等を含め、必要に応じて税関の品目分類・税率・課税価格の公式情報を確認します。

商品ページと取引メッセージでは、次を曖昧にしません。

- 発送地
- 配送方法と追跡の有無
- 関税等の通常の負担者
- 関税込み・返金対応を行う場合の条件と手順
- 配送遅延や通関照会が起きた時の連絡方法

#### 発送通知の条件

パートナーが現物を検品し、購入者宛ての荷物を配送会社へ引き渡し、発送控えと追跡番号を出品者が確認してから発送通知します。

次は発送通知をしてはいけない状態です。

- 店舗またはオンラインショップがパートナーへ発送しただけ
- パートナーが商品をまだ受け取っていない
- 現物検品が終わっていない
- 梱包しただけで配送会社へ引き渡していない
- 追跡番号を仮入力している

原管理表は、海外買付の依頼から精算までを同じ管理番号で追える構成です。

原xlsxは、そのまま配布しません。

買付先からパートナーまでと、パートナーから購入者までの配送は別区間です。2つの発送日・追跡番号を同じセルへ入れず、**BUYMAへ登録する発送日・追跡番号は、パートナーから購入者へ発送した区間だけ**にします。

- 空行に大量の`#N/A`が出る数式を修正する
- 行を増やすとずれる参照範囲を固定する
- 買付先→パートナーと、パートナー→購入者の発送日・追跡番号を別列にする
- BUYMA発送通知日時を追加し、購入者宛て発送との前後関係を検証する
- 店舗・オンラインの固定報酬を削除し、契約値を入力する
- 報酬の通貨、換算日、支払日を追加する

- 送金額、手数料、着金額、取引番号を追加する

- 代表入力値と期待結果で数式テストを行う

`経費立替`シートは、購入日、費目、現地通貨額、為替、円換算額、証憑、支払完了日を管理します。`報酬価格`シートは固定単価表ではなく、契約した業務、単価、通貨、適用開始日、改定履歴を参照する台帳へ変更します。

### 3-13 パートナー採用との境界

本章は、契約済みの海外パートナーへ業務を依頼する手順です。募集、選考、契約、教育、評価の詳細は第8章で扱います。

海外パートナーを探す時も、利用サービスの禁止事項を回避する募集は行いません。クラウドワークスは、無在庫販売に関係するEC出品・商品登録・在庫管理、代理注文・代理購入・代理受取・買付、アカウント共有等を禁止対象として案内しています。そのため、本教材では海外買付パートナーの募集先として推奨しません。

また、採用時に次を標準条件にしません。

- 業務遂行と無関係な性別・年齢で候補者を限定・優先する
- パスポート画像を相互交換する
- BUYMAアカウントを共有する
- パートナーの個人カード・自己資金を前提にする
- 特定ツールでの顔出し面談を一律必須にする
- 根拠のない固定報酬を「相場」として提示する

必要なのは属性ではなく、買付環境、言語、連絡可能時間、検品精度、配送環境、資金取扱い、個人情報管理、異常時対応を業務要件として確認することです。

旧教材では、直営店のスタッフが検品し、店舗から購入者へ直接発送する方法を「直営店直送」として解説していました。しかし、BUYMAでは、買付先から購入者へ未検品で直接送ることは原則禁止されています。店舗スタッフは、出品者と共に買付を行うパートナーとしても認められていません。

そこで本章では、旧教材の強みである「直営店の在庫・担当者・購入証明・輸出条件へ直接確認できること」を残し、発送経路だけを現行ルールへ合わせます。

```text
旧教材の経路
直営店 → 購入者
店舗スタッフの確認だけで発送

本章の経路
直営店 → 出品者または正式な買付パートナー
→ 現物検品 → 購入者へ発送
```

本章では、この現行経路を**直営店連携仕入れ**と呼びます。

本章のゴールは、候補店舗1件について次を完了できることです。

1. ブランド公式の店舗・連絡先を確認する
2. 輸出、支払、配送、購入証明、返品の条件を確認する
3. 出品者または買付パートナーが現物検品する経路を決める
4. 店舗から検品者までと、検品者から購入者までの配送を分けて管理する
5. 回答日と根拠を管理表へ残し、受注時に再確認する

### 4-1 直営店連携仕入れの仕組み

利用できる基本経路は次の2つです。

| 経路 | 店舗の配送先 | 現物検品 | 購入者への発送 |
|---|---|---|---|
| 出品者受取 | 出品者本人 | 出品者本人 | 出品者本人 |
| パートナー受取 | 正式な買付パートナー | 買付パートナー | 買付パートナー |

どちらも、店舗から届いた商品を一度受け取り、注文内容と品質を確認してから購入者へ発送します。

#### 店舗スタッフができること

- 在庫、価格、品番、色、サイズを確認する
- 店舗方針の範囲で商品写真や動画を送る
- 取り置き、他店舗からの取り寄せ、輸出条件を案内する
- 公式の決済方法で注文を受ける
- 出品者または買付パートナーへ商品を発送する
- レシート、インボイス、注文確認等を発行する

#### 店舗スタッフへ任せられないこと

- BUYMA上の出品者責任を負う現物検品
- 出品者の買付パートナーとしての購入者直送
- BUYMAの発送通知に使う最終発送判断
- 購入者クレーム時の出品者責任

店舗から届く写真は事前確認の補助です。最終的には、出品者または正式な買付パートナーが実物を手に取って検品します。

### 4-2 メリットとリスク

#### メリット

- ブランド公式の実店舗在庫を確認できる
- 公式オンラインに表示されない店舗在庫が見つかる場合がある
- 他店舗からの取り寄せ可否を直接確認できる
- 購入証明、輸出条件、配送条件を店舗へ確認できる
- 継続取引により、同じ担当者へ正確に問い合わせやすくなる場合がある
- 店舗の許可があれば、実物写真や動画を商品確認に使える場合がある

#### リスク

- 同じブランドでも国、店舗、担当者、時期で対応が異なる
- 国外配送、輸出時の税、取り置き、取り寄せは保証されない
- 店舗から検品者までの配送分だけ日数と送料が増える
- 返品・交換期限が短く、購入者発送前に対応できない場合がある
- 決済リンク、送金先、連絡先が本物か確認する必要がある
- 店舗から受け取った画像の利用には、著作権や利用許可の確認が必要になる
- 購入者情報を海外パートナーへ渡す場合は、第3章の外国第三者提供の確認が必要になる

直営店との連絡ルートができても、在庫確保、国外配送、税の免除、割引、高値販売を保証するものではありません。候補商品ごとに、仕入れ総額と発送期限まで成立するかを確認します。

### 4-3 候補店舗の探し方

候補店舗は、ブランド公式サイトのストアロケーターから探します。検索結果、地図サービス、SNSプロフィールだけで公式店舗と決めません。

#### 候補登録の手順

1. ブランド公式サイトを開く
2. 公式のストアロケーターへ移動する
3. 国、都市、店舗名、住所、電話番号、公式メール等を確認する
4. 公式サイト上のURLを保存する
5. 確認日と対応言語を記録する
6. 連絡前の状態を「未確認」として登録する

競合出品者の商品画像、サイズ表示、発送地、配送会社から「このブランドは直営店直送できる」と断定しません。それらは候補を探す仮説にはなっても、店舗との現行条件を証明する情報ではありません。

### 4-4 問い合わせ前の準備

問い合わせ前に、買付と検品の設計を決めます。

| 項目 | 決める内容 |
|---|---|
| 商品 | 商品名、品番、色、サイズ、数量、商品URL |
| 店舗 | ブランド、国、都市、店舗名、公式連絡先 |
| 注文者 | 出品者本人または契約済みパートナー |
| 店舗の配送先 | 出品者または現物検品を行う買付パートナー |
| 検品者 | 誰が実物を確認するか |
| 購入者発送 | 誰がどの配送方法で発送するか |
| 支払 | 通貨、上限、公式決済経路、送金の要否 |
| 期限 | 在庫回答、取置、決済、店舗発送、検品、購入者発送 |
| 証拠 | 回答記録、注文確認、購入証明、検品写真、配送伝票 |

連絡ツールは店舗が公式に案内する手段を使います。特定のメッセージアプリ、通話サービス、翻訳サービスへ固定しません。

翻訳を使う場合も、商品、価格、税、配送先、返品等の重要条件は原文と日本語を照合し、曖昧な回答を推測で確定しません。交渉担当へ任せる場合の採用・契約・教育は第8章で扱います。

### 4-5 店舗へ確認する10項目

旧管理表の7項目を残しつつ、未検品直送を避けるための検品経路、返品、回答根拠を追加します。

#### 1. 店舗の基本情報

- ブランド公式の直営店または公式に確認できる店舗か
- 店舗名、住所、国、電話番号、公式メール
- 回答者名または部署、回答日時、対応言語

#### 2. 出品者または買付パートナーへの配送

- 指定した国・住所へ発送できるか
- 配送先名義と決済名義に条件があるか
- 私書箱、転送先、ホテル等に制限があるか
- 商品を店舗から購入者へ未検品で直送する提案になっていないか

#### 3. 輸出時の税の扱い

- 表示価格に含まれる税の種類と金額
- 店舗が国外へ発送する時の税の扱い
- 免税または輸出扱いに必要な配送証明・書類
- 返金方式の場合の条件、時期、手数料
- 税を除いた最終請求額と根拠

EUではEU域外への輸出にVAT免除の枠組みがありますが、実際の適用には輸出方法や証拠等の条件があります。店舗が必ずVATを差し引くとは限りません。EU以外の国も含め、「VAT OFFできますか」だけで終わらせず、店舗が適用する税務処理と最終請求額を書面で確認します。

#### 4. 支払方法

- 店舗公式の決済リンクまたは公式オンライン決済が使えるか
- 対応カード、銀行送金等の方法と通貨
- 決済名義、請求先住所、本人確認の条件
- 追加手数料、返金方法、決済期限

カード番号、セキュリティコード、認証コードを、チャット、SMS、メールへ送信しません。電話注文を使う場合も、店舗の正式な決済手順、通話録音の扱い、カード情報を保存しない運用を確認します。

#### 5. 配送方法・送料

- 配送会社、サービス名、追跡、補償
- 商品サイズ・重量別の送料
- 発送までの日数と到着目安
- 通常便と速達便の違い
- 輸出申告、インボイス、禁制品の扱い
- 店舗発送後に受け取る追跡番号

#### 6. 他店舗からの取り寄せ

- 取り寄せ可能な地域・店舗
- 取り寄せ費用、所要日数、キャンセル条件
- 取り寄せ後の購入期限

取り寄せ可否を店舗スタッフの能力評価には使いません。ブランド方針、在庫、国境、店舗区分等で変わる業務条件として記録します。

#### 7. 連絡先

- 継続連絡に使ってよい公式連絡先
- 担当者名、部署、対応時間、使用言語
- 個人連絡先を取得する場合の店舗方針と利用目的
- 担当変更時の公式な引継ぎ先

#### 8. 購入証明・インボイス

- 店舗名、住所、購入日、商品、価格が分かる証憑を発行できるか
- 紙、PDF、メール等の受取方法
- 商品同梱書類と電子送付書類の違い
- BUYMAから買付先確認があった時に提出できる内容か

購入証明は価格を隠す目的で受け取らないのではなく、出品者が必ず保管します。購入者へ価格部分を見せない場合も、出品者の証拠原本まで削除・加工しません。

#### 9. 返品・交換

- 返品・交換できる条件と期限
- 返品先、送料、返金方法、必要書類
- セール品、取り寄せ品、国外発送の例外
- 不良、誤商品、配送破損時の連絡手順

#### 10. 検品補助と画像利用

- 店舗方針の範囲で、発送前に品番・色・サイズ・状態を確認できるか
- 商品写真や動画を送付できるか
- 画像を商品確認以外、特に出品画像として使ってよいか
- 利用許可の範囲、期限、クレジット表記

店舗の事前確認や写真は、正式な買付パートナーによる現物検品の代わりにはなりません。利用許可が確認できない画像はBUYMAへ掲載しません。

### 4-6 誠実な問い合わせ文

次は、旧教材の挨拶文と7確認項目を、現行経路へ合わせた問い合わせ例です。自分の氏名、事業、配送先、商品、実績に合わせて書き換えます。

```text
Subject: Inquiry about purchase, export, and shipping arrangements

Hello [Boutique / Client Advisor name],

My name is [your name], and I operate an online retail business in Japan.
I am interested in purchasing the item below from your boutique.

Product: [product name]
Reference / SKU: [reference]
Color: [color]
Size: [size]

Before placing the order, could you please confirm the following?

1. Is the item currently available, and until when can it be reserved?
2. Can you ship it to my designated receiving partner in [city, country]?
   My partner will physically inspect the item before it is sent to the customer.
3. What is the final price and the VAT or local-tax treatment
   when the item is exported?
4. Which official and secure payment methods are available?
5. Which carrier will you use, and what are the shipping cost,
   estimated delivery time, tracking, and insurance conditions?
6. If unavailable, can the item be transferred from another boutique?
7. Can you issue a detailed invoice or receipt showing the boutique,
   purchase date, product, and price?
8. What are the return or exchange conditions for an international order?
9. If your policy allows, could you share photos of the item before shipment?
   Please also let me know whether those photos may be used on a product page.

I understand that all services and conditions are subject to your boutique policy.
Thank you for your time.

Kind regards,
[your name]
[business contact]
```

相手が回答しやすいよう、最初から全店舗へ同一文面を大量送信せず、商品と店舗に合わせて必要項目を絞ります。回答が届いたら当日または次の営業日を目安にお礼を伝え、回答内容を管理表へ記録します。

### 4-7 問い合わせから購入者発送までの12工程

1. **出品者：公式店舗を確認する**
   ブランド公式のストアロケーターで店舗、国、住所、公式連絡先を確認し、確認日を残します。

2. **出品者または交渉担当：条件を問い合わせる**
   4-5の項目から必要な質問を選び、本人の役割と目的を偽らず問い合わせます。

3. **出品者：回答を管理表へ記録する**
   回答者、日時、原文、税、支払、配送、購入証明、返品、連絡先を保存します。

4. **出品者：検品経路と仕入れ総額を決める**
   店舗から出品者または正式な買付パートナーへ送り、誰が現物検品するかを確定します。商品代、現地税、2区間の送料、報酬に加え、関税込み・関税返金等で出品者負担となる関税等を原価へ含めます。通常の海外発送で購入者負担となる関税等は原価へ混ぜず、商品ページの表示条件として別に管理します。

5. **出品者：受注後に在庫と条件を再確認する**
   品番、色、サイズ、数量、価格、取置期限、返品、店舗から検品者までの発送予定を再確認します。購入者宛ての発送が、注文確定日時から18日後の同じ時刻までに間に合うかも確認します。

6. **出品者：公式の安全な方法で決済する**
   店舗の公式決済リンク、公式オンライン決済、確認済み口座等を使い、決済額、通貨、手数料、取引番号を保存します。

7. **出品者：購入証明を受け取る**
   注文確認、レシート、インボイス等を受け取り、店舗名、住所、日時、商品、価格を確認します。

8. **店舗：出品者または買付パートナーへ発送する**
   店舗→検品者の発送日、配送会社、追跡番号を記録します。この時点ではBUYMAの発送通知を行いません。

9. **出品者または買付パートナー：受取・現物検品する**
   商品、品番、色、サイズ、付属品、傷、汚れ、破損を確認し、日付または取引IDが分かる写真を残します。

10. **出品者：検品証拠を確認して発送を承認する**
    指示内容、購入証明、検品報告を照合します。異常がある場合は購入者へ送らず、返品・交換の判断をします。

11. **出品者または買付パートナー：購入者へ発送する**
    購入者が選んだ条件に従い、追跡可能な方法で発送します。配送方法を変更する場合は発送前に購入者の了承を得て、海外発送では郵便局留めを指定しません。海外パートナーへ購入者情報を渡す場合は、発送に必要な最小限だけを共有し、第3章3-10で確認した外国提供の条件、保管期限、削除手順に従います。検品者→購入者の発送控え、発送日、追跡番号を保存します。

12. **出品者：BUYMAで発送通知し、取引を管理する**
    購入者宛ての発送を確認してから、検品者→購入者の追跡番号をBUYMAへ入力します。受取、問い合わせ、関税、返品等を取引完了まで管理します。

店舗→検品者と、検品者→購入者は別の配送です。2区間の発送日・追跡番号を同じセルへ入れず、BUYMAへ登録するのは購入者宛ての区間だけにします。

店舗発送や検品者の受取が遅れ、購入者宛て発送が期限に間に合わない見込みの場合は、期限前に購入者へ状況を説明し、同意を得たうえで発送期限の延長を申請します。

### 4-8 決済・輸出時の税・購入証明

#### 決済

優先するのは、店舗公式ドメインの決済画面、店舗が発行した検証可能な決済リンク、店舗名義を確認できる送金先です。

- リンク元、ドメイン、店舗名、金額、通貨を確認する
- 短縮URLや個人名義口座は、店舗公式窓口で真偽を再確認する
- カード情報を平文のチャット、SMS、メールで送らない
- セキュリティコードや認証コードを保存しない
- 交渉担当者へカード情報、銀行認証、決済アカウントを共有しない
- 特定の送金サービスを固定推奨せず、国・通貨・規約・手数料を取引時に確認する

電話決済では、カード情報を含む通話録音やメモが残る可能性があります。店舗の公式なカード取扱手順を確認できない場合は、決済リンク等の別手段を依頼します。

#### 輸出時の税

EU域外への輸出はEU VAT上の免除対象になり得ますが、店舗が輸出者となるか、輸出証明を保持できるか等で実際の請求が変わります。国名や表示税率だけから値引額を計算しません。

管理表には次を残します。

- 表示価格と通貨
- 税込み・税抜きの別
- 税の名称と店舗回答
- 最終請求額
- 適用条件と必要書類
- 回答日、回答者、原文または根拠URL
- 返金方式の場合の予定日と実際の返金額

購入者へ海外発送する場合の関税等は第3章のルールに従い、通常取引か関税込み取引かを商品ページで明確にします。

#### 購入証明

BUYMAから買付先確認を求められた時に備え、次が分かる注文確認、レシート、インボイス等を保管します。

- 購入先店舗名と住所
- 買付日時
- 商品名、品番、数量
- 商品価格、税、送料、通貨
- 注文番号または取引番号

検品写真、店舗から検品者への発送伝票、購入者への発送伝票と同じ管理番号へ紐付けます。

### 4-9 店舗写真と現物検品

店舗が発送前の写真を送ってくれる場合、次の順に扱います。

1. 商品確認用として受け取る
2. 品番、色、サイズ、付属品を画像で事前確認する
3. 出品画像に使う場合は、店舗または権利者の利用許可を確認する
4. 許可の範囲、回答者、回答日を保存する
5. 出品者または買付パートナーが受取後に現物検品する

店舗写真に問題がなくても、配送中の破損、誤同梱、付属品不足は受取後に判明することがあります。店舗写真だけで「検品済み」と判断せず、購入者発送前の現物検品を必須にします。

現物検品と梱包の詳細は第3章の3-8を使います。

### 4-11 問い合わせ業務を任せる場合

- 不通なら無報酬等、一方だけにリスクを負わせる条件にしない

## 第5章 BUYMA オンラインVIP仕入れ完全解説マニュアル

本章でいう「オンラインVIP仕入れ」は、海外ショップと誠実な取引関係を作り、店舗が公式に提供する会員、常連顧客、取引、購入量等の条件を確認して仕入れる方法です。

条件は、個別クーポン、会員価格、アカウント別価格、期間限定オファー、購入量に応じた提案等、ショップによって異なります。すべてのショップに「VIP制度」があるわけではなく、割引率や承認を保証するものでもありません。

本章のゴールは、候補ショップ1件について次を完了できることです。

1. 現在営業している正規のショップか確認する
2. 割引がなくても成立する通常条件を計算する
3. 自分の実績だけを使って問い合わせる
4. 対象、除外、期限、最低購入額、併用、返品を確認する
5. 決済直前の総額で利益を再計算する
6. 条件と証拠を管理表へ記録する

### 5-1 オンラインVIP仕入れとは

| 項目 | 通常仕入れ | VIP・会員・取引条件あり |
|---|---|---|
| 価格 | 公開価格 | 店舗が承認した条件による価格 |
| 利用者 | 一般購入者 | 会員、常連、取引先等の条件を満たす人 |
| 適用 | 公開条件 | 対象ブランド、商品、期間、回数等に制限がある場合 |
| 管理 | 決済時の価格確認 | 承認証拠、条件、期限、除外、コード等の管理が必要 |
| 継続 | 都度確認 | 条件変更・停止を前提に再確認 |

旧教材では「専用クーポンで通常価格より10〜30%安くなり、免税でもっと安くなる」と説明していました。現行版では、この固定幅を使いません。

- 割引率はゼロを含めてショップ判断
- 税の扱いは販売国、配送先、店舗の輸出処理で変わる
- セール、ブランド、商品、配送先によって適用外になる場合がある
- 割引があっても送料、為替、関税、返品条件で利益が残らない場合がある
- 承認済み条件でも、期限切れや店舗判断で変更・終了する場合がある

「VIP」という名前より、**今回の注文へ適用できる条件が書面と決済画面で確認できるか**を重視します。

### 5-4 交渉するタイミング

交渉は、相手へ伝えられる事実がある時に行います。

#### 購入前

購入したい具体的な商品があり、通常条件でも利益と買付意思がある場合に、会員・常連・取引条件の有無を問い合わせます。割引をもらうためだけに、買う予定のない商品や架空の月間購入額を伝えません。

#### 購入時

実際に注文する直前に、今回使える公開キャンペーン、会員条件、個別条件があるか確認します。条件確認を待つ間に在庫や発送期限へ影響する場合は、注文を優先するかキャンセルするかを利益計算と購入者対応から判断します。

#### 購入後

実在する注文番号、購入商品、対応への感謝を伝え、今後も利用したい場合に継続条件を相談します。購入していない商品や存在しない注文を実績として使いません。

#### 再交渉

旧教材の「1か月後・2か月後に再交渉」を固定ルールにしません。次のような理由がある時だけ再連絡します。

- 実際の購入実績が増えた
- 以前案内された再連絡時期になった
- 条件の期限が近い
- 新しい取引・会員プログラムが公開された
- 対象ブランドや配送国を変更したい

同じ内容を短期間に繰り返し送らず、ショップの営業時間、連絡方針、配信停止意思を尊重します。

### 5-5 候補選定から継続確認までの10工程

1. **候補ショップを登録する**
   候補名、URL、国、発見元、登録日を管理表へ入れます。

2. **正規性とBUYMA利用可否を確認する**
   運営会社、販売元、ブランド公式の根拠、BUYMA禁止・要注意買付先、購入証明を確認します。

3. **通常条件の利益を計算する**
   割引ゼロの価格、送料、税、為替、決済に加え、出品者負担となる関税等を原価へ入れて利益を試算します。通常の海外発送で購入者負担となる関税等は原価へ混ぜず、商品ページの表示条件として別に管理します。割引がないと赤字になる商品を、未確定の交渉結果だけで出品しません。

4. **自分の事実を整理する**
   実際の購入履歴、購入したい商品、取り扱いブランド、今後の現実的な購入意向を整理します。

5. **公式窓口へ問い合わせる**
   公式メール、問い合わせフォーム、店舗が案内する連絡先を使い、役割や実績を偽らず連絡します。

6. **回答と証拠を保存する**
   回答日時、送信元、原文、添付、条件、担当者を管理表へ紐付けます。

7. **適用条件を確認する**
   対象、適用外、最低額、併用、期限、利用回数、アカウント、配送国、返品を確認します。

8. **決済直前に適用と総額を確認する**
   コードや会員価格を適用し、商品価格、送料、税、手数料を含む決済総額を確認して利益を再計算します。

9. **実際の注文と結果を記録する**
   注文番号、購入額、割引額、対象商品、利用日、購入証明を保存します。出品者または買付パートナーが現物検品してから購入者へ発送します。

10. **継続・再交渉・停止を判断する**
    返品、配送、利益、対応品質、条件変更を振り返り、継続、再確認、停止の状態を更新します。

### 5-6 誠実な交渉文テンプレート

#### 初回問い合わせ

次のテンプレートは、角括弧内を自分の事実へ置き換えてから使います。事業を運営している場合だけ`I operate...`を使い、準備中なら`I am preparing...`を使います。どちらも事実でなければその行を削除し、角括弧が残ったまま送信しません。

```text
Subject: Inquiry about a regular-customer or trade program

Dear Customer Support Team,

My name is [your name].
[If true: I operate an online retail business in Japan.]
[If true: I am preparing to start an online retail business in Japan.]
I am interested in purchasing authentic fashion items from your store,
including [brand / category / specific item].

Could you please let me know whether you offer any regular-customer,
trade, loyalty, or volume-purchase program?

If a program is available, I would appreciate details about:
- eligible and excluded brands or products,
- minimum purchase requirements,
- validity and usage limits,
- combination with sale prices or other promotions,
- shipping countries, taxes, and return conditions.

I understand that approval and all conditions are at your discretion.
Thank you for your time.

Kind regards,
[your name]
[business contact]
```

#### 購入後の問い合わせ

```text
Subject: Inquiry after order [order number]
```

原教材にあった「月1万〜2万USD以上購入する」「架空の俳優アカウントを作る」等は使いません。購入予定額、購入履歴、肩書、顧客、SNSは、確認できる事実だけを伝えます。

### 5-7 条件をもらった後の確認

「20%コードをもらった」だけでは、仕入れ条件は確定していません。次を確認します。

| 確認項目 | 記録する内容 |
|---|---|
| 対象 | ブランド、商品、カテゴリ、価格帯 |
| 適用外 | 新作、定番、セール、限定、特定ブランド等 |
| 率・価格 | 割引率、会員価格、通貨、小数・端数処理 |
| 最低条件 | 最低注文額、累計額、注文回数、数量 |
| 併用 | セール、公開コード、ポイント、送料無料との併用 |
| 回数 | 1回、期間内回数、数量上限 |
| 期限 | 開始日時、終了日時、タイムゾーン |
| アカウント | 対象メール、会員ID、地域サイト、譲渡可否 |
| 配送 | 対象国、送料、追跡、補償、到着目安 |
| 税・関税 | 表示税、輸出時の処理、関税負担、手数料 |
| 返品 | 返品可否、返金額、期限、送料、割引失効 |
| 証拠 | メール、規約URL、画面、回答者、確認日 |

担当者の口頭回答だけで出品価格を決めず、可能な範囲でメール、規約、決済画面等の証拠を残します。利用条件に守秘や個人利用の指定がある場合は従います。

### 5-8 割引後の利益計算

```text
決済前の見込仕入れ総額
= 店舗が確定した割引・税処理後の商品請求額の円換算額
+ 送料・補償の円換算額
+ 決済・送金手数料
+ 出品者負担となる関税・輸入消費税等
+ 検品・梱包・パートナー報酬等

決済前の見込利益
= 販売価格
- BUYMA成約手数料等
- 決済前の見込仕入れ総額
- その他の販売関連費用

決済後の実績仕入れ総額
= カード・送金明細で確定した商品代・送料等の円請求額
+ その円請求額に含まれていない決済・送金手数料
+ 出品者負担として確定した関税・輸入消費税等
+ 検品・梱包・パートナー報酬等

決済後の実績利益
= 販売価格
- BUYMA成約手数料等
- 決済後の実績仕入れ総額
- その他の販売関連費用
```

決済前は店舗表示、決済条件、利用するカード・送金サービスの条件から見込額を作ります。決済後はカード・送金明細と関税等の確定額へ置き換えます。カードや送金サービスの為替スプレッドは、商品代へ別に重ねず、明細で確定した実際の円請求額または手数料として反映します。送料、税、手数料等が注文総額や円請求額に含まれている場合は再加算しません。「20%OFFだから利益が20%増える」とは限りません。

#### 決済前の2つのシナリオ

| シナリオ | 使う価格 | 目的 |
|---|---|---|
| 通常条件 | 割引なしの決済総額 | 条件が使えない場合の撤退判断 |
| 承認条件 | 決済画面で実際に適用された総額 | 今回の見込利益更新 |

決済完了後も、カード・送金明細、関税等、返品・返金が確定するまでは実績利益と呼びません。

コードが無効、対象外、期限切れ、併用不可になった場合は、無断で通常価格購入へ切り替えません。受注価格、購入者への案内、在庫、発送期限を確認し、買付またはキャンセルを判断します。

### 5-9 コード・会員アカウントの管理

- ショップから許可されたアカウント・注文だけに使う
- 第三者の個人コードや会員アカウントを借りない
- コードや会員価格を公開、販売、特典配布しない
- 担当者の個人連絡先や条件を無断で共有しない
- パスワードと認証コードを交渉担当へ渡さない
- アカウントのメール、地域、通貨、配送先を注文前に確認する
- 期限切れ、停止、利用済みの状態を更新する
- 条件変更の連絡を保存する
- 管理表には証拠ファイルそのものではなく、アクセスを限定した保管先リンクと確認日を記録する
- パスワード、認証コード、カード番号、本人確認書類を管理表へ保存しない
- 収集する担当者情報は取引に必要な範囲へ絞り、保存期限を決めて不要になったら削除する
- 外注担当の契約・業務終了時は、管理表と証拠保管先へのアクセスを停止する

店舗が本人確認や事業情報を求める場合は、必要性、提出先、利用目的、保管方法を公式窓口で確認します。交渉のために架空のSNS、俳優、インフルエンサー、会社、購入実績を作りません。

### 5-10 交渉を外部へ任せる場合

翻訳や問い合わせの下書きを外部へ任せても、ショップの選定、実績の正しさ、条件の採用、決済、BUYMAでの責任はアカウント登録者に残ります。

- 本人ではなく許可を得た担当・アシスタント・通訳であることを正しく伝える
- 架空の購入額や成功しやすい肩書を作らない
- ショップのアカウント、決済、カード情報を共有しない
- 1件電話＋成功200円等の旧固定報酬を相場として使わない
- 回答原文と条件を出品者が再確認する
- 利用サービスの禁止事項を回避する募集をしない

クラウドワークスは、無在庫販売、EC運営代行、代理注文・購入・受取・買付等を禁止対象として案内しています。本教材ではVIP交渉担当の募集先として固定推奨しません。採用、契約、報酬、教育、品質評価は第8章で扱います。
