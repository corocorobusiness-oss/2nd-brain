# Codexに投げるプロンプト集

## 1. 2nd-brainの構成確認

```text
この `2nd-brain` の構成を見て、Codexに読ませるAI秘書用の記憶置き場として分かりやすいか確認してください。

確認してほしいこと:
- フォルダ構成が複雑すぎないか
- `00_system/` に必要なファイルがあるか
- メモ、日誌、成果物の置き場所が分かるか

ルール:
- まだファイルは編集しない
- 不足しているものだけ提案する
```

## 2. メモ整理

```text
`00_system/profile.md`
`00_system/goals.md`
`00_system/rules.md`
`02_daily/YYYY-MM-DD.md`

この4つを読んで、今日のメモを整理してください。

出力:
1. 今日やること
2. タスク
3. アイデア
4. 学び
5. 確認待ち
6. 明日の一手

ルール:
- 今日やることは3つまで
- 元メモにない内容を勝手に足さない
- 不明点は確認待ちに入れる
```

## 3. タスク整理

```text
`00_system/goals.md`
`02_daily/YYYY-MM-DD.md`

この2つを読んで、今の目標に合うようにタスクを整理してください。

出力:
- 今日やること
- 確認待ち
- 後で
- 最初にやること

ルール:
- 今日やることは3つまで
- 確認待ちは今日やることに入れない
- 目標に関係が薄いものは後でに入れる
```

## 4. 企画出し

```text
`00_system/profile.md`
`00_system/goals.md`
`03_memos/`

この3つを読んで、今の目標に合う企画案を出してください。

出口は「___」です。

以下の形式で出してください。

1. 企画タイトル
2. 誰向けか
3. 読者の悩み
4. 伝える内容
5. なぜ今の目標に合うか
6. 最初に作るべき一手

ルール:
- 企画は5案まで
- 元メモにある内容を優先する
- 最後に一番おすすめの案を1つ選ぶ
```

## 5. 週次メンテ

```text
`00_system/profile.md`
`00_system/goals.md`
`00_system/rules.md`
`00_system/workflows.md`
`02_daily/`
`03_memos/`

この範囲を見て、今週の振り返りを作ってください。

やってほしいこと:
1. 今週できたことをまとめる
2. 残っているタスクを整理する
3. 来週やることを3つに絞る
4. `rules.md` に追加した方がいいルールを提案する
5. `workflows.md` に追加した方がいい作業手順を提案する

ルール:
- まだファイルは編集しない
- まず提案だけ出す
- 削除はしない
- 不明点は確認する
```

## 6. 販売ページの壁打ち

```text
`00_system/profile.md`
`00_system/goals.md`
`03_memos/`
`04_outputs/`

この内容を読んで、Brain販売ページの下書きを改善してください。

見てほしいこと:
- 冒頭で読者の悩みが伝わるか
- 誇張した約束になっていないか
- これから始めたい人向けになっているか
- 有料部分の価値が伝わるか

ルール:
- すぐに書き換えず、まず改善案だけ出す
- 売上保証や過剰な表現は入れない
```
