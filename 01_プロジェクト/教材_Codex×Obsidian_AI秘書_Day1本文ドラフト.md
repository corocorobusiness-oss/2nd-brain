# Day 1: Obsidianの土台を作る

作成日: 2026-07-04
参照: [[教材_Codex×Obsidian_AI秘書_商品設計]] / [[教材_Codex×Obsidian_AI秘書_Day0本文ドラフト]]

## 今日のゴール

Day 1では、Codexに読ませるためのObsidianの土台を作ります。

今日のゴールは、次の3つです。

- ObsidianのVault `2nd-brain` を作る
- AI秘書用の最小フォルダ構成を作る
- 自分の情報を置く場所を決める

ここでは、いきなり複雑なSecond Brainは作りません。

最初から完璧を目指すと、ほぼ確実に続きません。

まずは、Codexが読みやすく、あなたも迷わない最小構成を作ります。

## まず作るのは「きれいな情報整理」ではない

Obsidianと聞くと、きれいなノート術やリンク構造を作らないといけない気がするかもしれません。

でも、この教材の目的は、情報整理マニアになることではありません。

目的は、Codexがあなたの文脈を読める状態を作ることです。

つまり大事なのは、

- どこにプロフィールがあるか
- どこに目標があるか
- どこにルールがあるか
- どこに日々のメモがあるか
- どこにタスクがあるか

が分かること。

見た目の美しさより、AIと自分が迷わないことを優先します。

## Vault名は `2nd-brain`

Obsidianでは、メモをまとめて置く場所をVaultと呼びます。

Vaultは「AI秘書の作業部屋」だと思ってください。

この教材では、専用のVaultを1つ作ります。

名前は次で統一します。

```text
2nd-brain
```

`2nd-brain` は「第2の脳」という意味です。

この教材では、Codexに読ませるための小さなAI秘書用Second Brainとして扱います。

日本語名でも使えますが、AIツールやPC環境によっては、パスの扱いで少し面倒になることがあります。

そのため、教材内では半角英数字の `2nd-brain` で統一します。

## 最小フォルダ構成

Day 1で作るフォルダは、これだけです。

```text
2nd-brain/
├── 00_system/
├── 01_projects/
├── 02_daily/
├── 03_memos/
├── 04_outputs/
└── 90_archive/
```

これで十分です。

最初から大量のフォルダを作る必要はありません。

それぞれの役割はこうです。

| フォルダ | 役割 |
|---|---|
| `00_system/` | AIに読ませるプロフィール、ルール、目標、手順 |
| `01_projects/` | 進行中の企画や仕事 |
| `02_daily/` | 日々のデイリーノート |
| `03_memos/` | 思いつき、アイデア、メモ |
| `04_outputs/` | AIと作った文章、企画、投稿案 |
| `90_archive/` | 使わなくなったものの退避場所 |

一番大事なのは `00_system/` です。

ここに、Codexがあなたを理解するための情報を置いていきます。

## 00_systemに作るファイル

`00_system/` の中には、最初にこの4つを作ります。

```text
00_system/
├── profile.md
├── goals.md
├── rules.md
└── workflows.md
```

それぞれの役割はこうです。

### profile.md

あなたが何をしている人なのかを書くファイルです。

例:

```markdown
# Profile

私はこれから発信活動とコンテンツ販売を始めたい個人です。

主な活動:
- YouTube
- note
- Brain教材制作
- SNS発信

AIには、メモ整理、タスク整理、企画出し、文章作成の補助をしてほしいです。
```

### goals.md

今の目標を書くファイルです。

例:

```markdown
# Goals

## 今月の目標

- Brain教材を1本完成させる
- Xで毎日1投稿する
- メモを毎日Obsidianに残す

## 判断基準

迷ったら、発信とコンテンツ販売の準備が続きやすくなる方を選ぶ。
```

### rules.md

AIに守ってほしいルールを書くファイルです。

例:

```markdown
# Rules

- 勝手に外部投稿しない
- ファイルを削除しない
- 個人情報や秘密情報は扱いに注意する
- 分からないことは決めつけずに確認する
- 文章は初心者にも分かるように書く
```

### workflows.md

よくやる作業の手順を書くファイルです。

例:

```markdown
# Workflows

## メモ整理

1. 今日のメモを読む
2. アイデア、タスク、学びに分ける
3. タスクはチェックリストにする
4. 企画になりそうなものは候補として残す

## 企画出し

1. 関連メモを読む
2. 読者の悩みを決める
3. タイトル案を出す
4. 目次案を作る
```

この4つがあるだけで、AIに毎回説明する量はかなり減ります。

## 02_dailyにデイリーノートを作る

次に、`02_daily/` に今日のデイリーノートを作ります。

ファイル名は日付にします。

例:

```text
02_daily/2026-07-04.md
```

中身はシンプルでOKです。

```markdown
# 2026-07-04

## 今日やること

- 

## メモ

- 

## タスク

- [ ] 

## 学び

- 
```

デイリーノートは、AIにとって「今日の現在地」になります。

日々のメモがここにあると、Codexにこう頼めます。

> 今日のデイリーノートを読んで、タスクと企画案を整理して

最初は1日1行でも大丈夫です。

大事なのは、完璧な日記を書くことではなく、AIが読める形で今日の情報を残すことです。

## 03_memosに思いつきを置く

`03_memos/` には、まだ整理されていない思いつきを置きます。

例:

```text
03_memos/brain-ideas.md
03_memos/youtube-ideas.md
03_memos/random-notes.md
```

思いついたことを全部きれいに整理しようとすると疲れます。

最初は、雑に置ける場所がある方が続きます。

ただし、あとでCodexに整理してもらうために、1つのメモには日付を入れておくと便利です。

例:

```markdown
## 2026-07-04

- Codex × Obsidian教材の表紙画像を作りたい
- 初心者向けにBefore/After図を入れたい
- 価格は完成版ローンチで980円から
```

## 01_projectsに進行中のものを置く

`01_projects/` には、進行中の仕事や企画を置きます。

例:

```text
01_projects/brain-course/
01_projects/youtube/
01_projects/newsletter/
```

Brain教材を作るなら、こういうフォルダを作ります。

```text
01_projects/brain-course/
├── concept.md
├── sales-page.md
├── day0.md
└── templates.md
```

プロジェクトごとにファイルがまとまっていると、Codexに相談しやすくなります。

例:

> `01_projects/brain-course/` を読んで、販売ページの弱いところを指摘して

> `day0.md` を初心者向けにもっと分かりやすくして

## 04_outputsに完成物を置く

`04_outputs/` には、AIと作った完成物や公開候補を置きます。

例:

```text
04_outputs/x-posts.md
04_outputs/note-drafts.md
04_outputs/brain-sales-page.md
```

メモと完成物を分けておくと、あとで見返しやすくなります。

`03_memos/` は素材。

`04_outputs/` は出荷物。

この違いだけ覚えておけば大丈夫です。

## 90_archiveは削除の代わり

いらなくなったファイルは、すぐ削除しなくてOKです。

初心者のうちは、削除よりも退避がおすすめです。

`90_archive/` に移動しておけば、あとで戻せます。

AIにファイル操作を頼む時も、

> 消さずにarchiveへ移動して

と指示すると安全です。

## 今日作る完成形

Day 1が終わると、あなたのVaultはこうなります。

```text
2nd-brain/
├── 00_system/
│   ├── profile.md
│   ├── goals.md
│   ├── rules.md
│   └── workflows.md
├── 01_projects/
├── 02_daily/
│   └── 2026-07-04.md
├── 03_memos/
├── 04_outputs/
└── 90_archive/
```

これで、AI秘書の作業部屋ができました。

まだ中身は少なくて大丈夫です。

大事なのは、

- 自分の情報を置く場所
- 今日のメモを置く場所
- 企画を置く場所
- 完成物を置く場所
- 退避する場所

が決まったことです。

## Codexに投げる確認プロンプト

フォルダとファイルを作ったら、Codexにこう聞いてみてください。

```text
このフォルダ構成を見て、AI秘書用の最小構成として分かりにくい点や足りない点があれば指摘してください。
ただし、初心者向けなので複雑にしすぎないでください。
```

最初から完璧にする必要はありません。

Codexに見てもらいながら、少しずつ直していけばOKです。

## よくある失敗

### 失敗1: 最初からフォルダを増やしすぎる

最初から細かく分けすぎると、どこに何を置くか迷います。

まずは6フォルダで十分です。

### 失敗2: 日記を書こうとして続かない

デイリーノートは立派な日記でなくてOKです。

今日やること、メモ、タスク、学びが1行ずつあれば十分です。

### 失敗3: 秘密情報を置きすぎる

AIに読ませる可能性がある場所に、パスワードやトークンを置かないでください。

便利さより安全が優先です。

### 失敗4: 見た目にこだわりすぎる

最初はリンクやタグを完璧にしなくて大丈夫です。

AIが読める場所に、必要な情報があることの方が大切です。
