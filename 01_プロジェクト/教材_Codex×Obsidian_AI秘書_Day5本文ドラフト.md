# Day 5: タスクと目標をAI秘書に追わせる

作成日: 2026-07-04
参照: [[教材_Codex×Obsidian_AI秘書_商品設計]] / [[教材_Codex×Obsidian_AI秘書_Day4本文ドラフト]]

## 今日のゴール

Day 5では、`goals.md` とデイリーノートをつなげて、Codexに今日やることを整理してもらいます。

今日のゴールは、次の3つです。

- 目標と今日のメモをつなげる
- タスクを「今日やる」「確認待ち」「後で」に分ける
- Codexに今日の一手を絞ってもらう

Day 4では、雑なメモを整理しました。

Day 5では、整理したメモを「実際に動けるタスク」に変えていきます。

## タスク整理で一番大事なこと

タスク整理で一番大事なのは、たくさん並べることではありません。

今日やることを絞ることです。

AIに頼むと、気を利かせてたくさんタスクを出してくれることがあります。

でも、タスクが増えすぎると逆に動けません。

だから、この教材では次のルールにします。

> 今日やることは3つまで

3つに絞ると、AI秘書がただのアイデア出し係ではなく、行動を前に進める相棒になります。

## goals.mdを読ませる理由

タスク整理では、`goals.md` が重要です。

同じメモでも、目標によって優先順位が変わるからです。

たとえば、今日のメモにこう書いてあったとします。

```markdown
- X投稿を作る
- 教材Day5を書く
- 表紙画像を作る
- 部屋を片付ける
- 新しいAIツールを調べる
```

今の目標が「教材を完成させる」なら、優先順位はこうなります。

1. 教材Day5を書く
2. 表紙画像を作る
3. X投稿を作る

でも、今の目標が「SNSを伸ばす」なら、優先順位は変わります。

1. X投稿を作る
2. 表紙画像を作る
3. 教材Day5を書く

つまり、タスクは単体では判断できません。

目標とセットで見る必要があります。

## タスク管理の置き場所

タスクは、まずデイリーノートに置きます。

例:

```markdown
# 2026-07-04

## 今日やること

- [ ] 

## タスク

- [ ] Day5本文を書く
- [ ] 表紙画像を作る
- [ ] X投稿を作る
- [ ] 販売ページを見直す

## 確認待ち

- 

## 後で

- 
```

初心者のうちは、複雑なタスク管理ツールを使わなくて大丈夫です。

まずは、その日のデイリーノートにまとめるだけで十分です。

## タスクを3つに分ける

Codexには、タスクを次の3つに分けてもらいます。

| 分類 | 意味 |
|---|---|
| 今日やる | 今日やるべきもの。3つまで |
| 確認待ち | 自分だけでは判断できないもの |
| 後で | 今すぐやらなくていいもの |

この3つに分けるだけで、かなり動きやすくなります。

タスクが全部同じ場所に並んでいると、どれからやるか迷います。

でも、「今日やる」が3つだけなら、まずそこから動けばいい。

## Codexにタスク整理を頼むプロンプト

基本プロンプトはこれです。

```text
`00_system/goals.md` と今日のデイリーノートを読んで、タスクを整理してください。

以下の3つに分けてください。

1. 今日やること
2. 確認待ち
3. 後で

ルール:
- 今日やることは3つまで
- goals.md の目標に近づくものを優先する
- 元メモにないタスクを勝手に増やしすぎない
- 判断が必要なものは「確認待ち」に入れる
- 最後に、最初にやるべき1つを提案する
```

これで、目標に沿ったタスク整理ができます。

## Before / After

Before:

```markdown
## メモ

- Day5本文を書く
- 表紙画像を作る
- Xで制作過程を投稿したい
- 販売ページを見直す
- 画像生成のプロンプトも考える
- 価格は980円からでいいか少し迷う
```

After:

```markdown
## 今日やること

- [ ] Day5本文を書く
- [ ] 販売ページを見直す
- [ ] Xで制作過程を投稿する

## 確認待ち

- [ ] 価格を980円スタートで確定するか確認する

## 後で

- [ ] 表紙画像を作る
- [ ] 画像生成プロンプトを考える

## 最初にやること

Day5本文を書く。
教材本編が進むと、販売ページや発信の説得力も上がるため。
```

このように、AI秘書に優先順位をつけてもらいます。

## 今日やることは3つまで

何度も言いますが、今日やることは3つまでです。

AIはたくさん提案できます。

でも、人間が1日に集中して進められることは限られています。

特に、これから発信やコンテンツ販売を始めたい人は、時間も体力も限られています。

だから、AI秘書にはこう頼みます。

```text
今日やることは3つまでに絞ってください。
```

この一文を入れておくだけで、かなり実用的になります。

## 確認待ちを作る

タスク整理では、「確認待ち」も大事です。

確認待ちとは、今すぐ作業できないものです。

たとえば、

- 価格を本当にこれでいくか
- 商品名を確定するか
- 外部に投稿していいか
- 誰かに確認が必要か
- まだ情報が足りないか

こういうものは、今日やることに混ぜない方がいいです。

混ぜると、手が止まります。

だから、AI秘書には判断が必要なものを「確認待ち」に分けてもらいます。

## 後でを作る

「後で」も大事です。

今すぐやらなくていいものまで今日やることに入れると、1日が重くなります。

たとえば、

- いつか作りたい画像
- あとで見直したい文章
- まだ急がないアイデア
- 今週でなくてもいい改善

こういうものは、後でに逃がしてOKです。

後でに入れるのは、捨てることではありません。

今やることを守るための退避です。

## workflows.mdに追記する

Day 5の手順も、`workflows.md` に追記します。

```markdown
## タスク整理

1. `00_system/goals.md` を読む
2. 今日のデイリーノートを読む
3. タスクを「今日やること」「確認待ち」「後で」に分ける
4. 今日やることは3つまでにする
5. goals.md の目標に近づくものを優先する
6. 最初にやるべき1つを提案する
```

次回からは、こう頼めます。

```text
workflows.md の「タスク整理」に沿って、今日のデイリーノートを整理してください。
```

## タスク整理後のデイリーノート例

整理後のデイリーノートは、こうなります。

```markdown
# 2026-07-04

## 今日やること

- [ ] Day5本文を書く
- [ ] 販売ページを見直す
- [ ] Xで制作過程を投稿する

## 確認待ち

- [ ] 価格を980円スタートで確定するか確認する

## 後で

- [ ] 表紙画像を作る
- [ ] 画像生成プロンプトを考える

## 最初にやること

Day5本文を書く。
```

これくらいシンプルで大丈夫です。

タスク管理は、凝れば凝るほど続かなくなります。

まずは「今日やる」「確認待ち」「後で」だけで始めましょう。

## よくある失敗

### 失敗1: 今日やることが多すぎる

10個も並ぶと、だいたい動けません。

今日やることは3つまでにします。

### 失敗2: 確認待ちを今日やることに入れる

判断が必要なものは、作業タスクとは分けます。

確認待ちに逃がしてOKです。

### 失敗3: 後でに入れるのを怖がる

後でに入れるのは、諦めることではありません。

今日の集中を守るためです。

### 失敗4: 目標を読ませずに整理する

goals.mdを読ませないと、AIは一般的な優先順位で判断します。

必ず目標とセットで整理してもらいましょう。

## Day 5 完了チェック

- `goals.md` とデイリーノートをCodexに読ませた
- タスクを「今日やること」「確認待ち」「後で」に分けた
- 今日やることを3つ以内に絞った
- 最初にやるべき1つを決めた
- `workflows.md` にタスク整理の手順を追記した

ここまでできたら、Day 6でメモと目標をもとに企画出しへ進みます。
