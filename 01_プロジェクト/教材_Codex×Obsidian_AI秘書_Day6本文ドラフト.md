## 企画はゼロから作らなくていい

企画出しというと、白紙からアイデアを出すイメージがあるかもしれません。

でも、これから発信やコンテンツ販売を始めたい人にとって、一番強い企画の種は日々のメモの中にあります。

たとえば、

- 自分がつまずいたこと
- 調べて分かったこと
- 読者に説明したいこと
- 作業中に気づいたこと
- 何度も考えているテーマ
- 人に聞かれたこと

こういうものは、全部企画の素材になります。

`2nd-brain` にメモを残しておくと、Codexにその素材を読ませて企画化できます。

## 企画出しで読むファイル

企画出しでは、最低でもこの3つを読ませます。

```text
00_system/profile.md
00_system/goals.md
03_memos/
```

必要に応じて、今日のデイリーノートも読ませます。

```text
02_daily/YYYY-MM-DD.md
```

それぞれの役割はこうです。

| ファイル・フォルダ | 役割 |
|---|---|
| `profile.md` | あなたが何をしている人か |
| `goals.md` | 今どの方向に進みたいか |
| `03_memos/` | 企画の素材 |
| `02_daily/` | 今日の気づきや作業ログ |

企画は、素材だけでは弱いです。

目標と組み合わせることで、「今出す意味がある企画」になります。

## まずは企画の出口を決める

Codexに企画出しを頼む前に、どこに出す企画なのかを決めます。

出口の例:

- X投稿
- note記事
- Brain教材
- YouTube動画
- メルマガ
- 無料配布PDF

出口が決まっていないと、企画がふわっとします。

たとえば、同じ「Codex × Obsidian」というテーマでも、出口によって形が変わります。

| 出口 | 企画の形 |
|---|---|
| X投稿 | 短く刺さる気づき |
| note記事 | 体験談や解説 |
| Brain教材 | 悩みを解決する手順 |
| YouTube動画 | 視聴者が見たくなる切り口 |

だから最初に、Codexへこう伝えます。

```text
今回はBrain教材の企画として考えてください。
```

または、

```text
今回はX投稿用に、短く刺さる切り口を出してください。
```

出口を先に決めるだけで、提案の質が上がります。

## 企画出しプロンプト

基本プロンプトはこれです。

```text
`00_system/profile.md`
`00_system/goals.md`
`03_memos/`

この3つを読んで、今の目標に合う企画案を出してください。

出口は [X投稿 / note記事 / Brain教材 / YouTube動画] のうち、今回は「___」です。

以下の形式で出してください。

1. 企画タイトル
2. 誰向けか
3. 読者の悩み
4. 伝える内容
5. なぜ今の目標に合うか
6. 最初に作るべき一手

ルール:
- 元メモにある内容を優先する
- 抽象論だけで終わらせない
- 企画は5案まで
- 最後に一番おすすめの案を1つ選ぶ
```

`___` のところに出口を入れます。

例:

```text
今回は「Brain教材」です。
```

```text
今回は「X投稿」です。
```

## 企画案は5つまで

企画出しでありがちな失敗は、案を出しすぎることです。

20案も出ると、見て満足して終わります。

最初は5案までで十分です。

さらに、最後にAI秘書へこう頼みます。

```text
この中で一番おすすめの案を1つ選び、理由も教えてください。
```

企画は数より、次に動けることが大事です。

## 企画を評価する3つの基準

Codexに企画を出してもらったら、次の3つで見ます。

### 1. 自分の経験で語れるか

自分がやっていないことは、説得力が弱くなります。

まずは、自分の経験で語れる企画を選びます。

### 2. 読者の悩みが具体的か

「AIを活用したい人」だけだと広すぎます。

「AIに毎回同じ説明をして疲れている人」くらいまで絞ると、刺さりやすくなります。

### 3. 次の一手が明確か

良い企画は、次に何を作ればいいかが見えます。

- X投稿を書く
- note構成を作る
- Brain目次を作る
- Day 0本文を書く

このように、次の一手が出る企画を選びます。

## X投稿用に変換する

同じ企画を、X投稿に変換することもできます。

プロンプト:

```text
上の企画を、X投稿用に3案へ変換してください。

ルール:
- 1投稿140字以内
- 読者の悩みから入る
- 専門用語を使いすぎない
- 最後に少しだけ続きが気になる形にする
```

出力例:

```text
AIに毎回同じ説明をしてるなら、たぶん足りないのはプロンプト力じゃなくて「記憶の置き場所」。

Obsidianに自分の目標・ルール・メモを置いて、Codexに読ませると、AIが一気に秘書っぽくなる。
```

## note記事用に変換する

noteなら、体験談や解説にします。

プロンプト:

```text
この企画をnote記事にするなら、読者が最後まで読みたくなる構成にしてください。

以下の形式で出してください。

1. タイトル
2. 導入
3. 見出し構成
4. この記事で読者が得られること
5. 最後のまとめ
```

## Brain教材用に変換する

Brain教材なら、悩み解決の手順にします。

プロンプト:

```text
この企画をBrain教材にするなら、販売ページと目次案を作ってください。

以下の形式で出してください。

1. 商品名
2. サブタイトル
3. 誰向けか
4. 読者の悩み
5. 教材で作るもの
6. 章立て
7. 特典案
8. 価格案
```

## YouTube企画用に変換する

YouTubeなら、視聴者がクリックしたくなる切り口にします。

プロンプト:

```text
この企画をYouTube動画にするなら、タイトル案と構成案を出してください。

以下の形式で出してください。

1. タイトル案5つ
2. 冒頭30秒の入り
3. 本編構成
4. 視聴者が得られること
5. サムネイルの文字案
```

## workflows.mdに追記する

Day 6の手順も、`workflows.md` に追記します。

```markdown
## 企画出し

1. `00_system/profile.md` を読む
2. `00_system/goals.md` を読む
3. 関連する `03_memos/` のメモを読む
4. 企画の出口を決める
5. 企画案を5つまで出す
6. 自分の経験で語れるか、読者の悩みが具体的か、次の一手が明確かで評価する
7. 一番おすすめの案を1つ選ぶ
```

次回からは、こう頼めます。

```text
workflows.md の「企画出し」に沿って、03_memos の内容からBrain教材案を出してください。
```

## よくある失敗

### 失敗1: 企画を出しすぎる

20案も出すと、選べなくなります。

最初は5案までで十分です。

### 失敗2: 出口を決めずに企画出しする

X投稿なのか、Brain教材なのか、YouTube動画なのか。

出口を決めないと、企画がぼやけます。

### 失敗3: 自分の経験がないテーマを選ぶ

自分が語れない企画は、文章にしても弱くなりがちです。

まずは自分の経験で語れるものを選びましょう。

### 失敗4: 次の一手を決めない

企画を出して満足して終わるのはもったいないです。

最後に、必ず「最初に作るもの」を決めます。
